# Aurora 课程修订宪章

**适用范围**：L01–L99 全课  
**修订原则**：尽量不删旧内容，优先通过增补、对齐和防呆提升质量。

## 1. 目标

把课程从“能覆盖知识点”提升到“能稳定教学、能独立运行、能清晰验收”。

## 2. 不可动摇的标准

1. 标题、README、正文、下一课预告必须一致。
2. 每课都要能从干净内核运行到结束。
3. 每个练习必须是真练习，不能用完整答案伪装成 TODO。
4. 每课至少有一个明确验证点，最好含边界 case。
5. 每课都要有 Aurora 连接，说明它在 `src/aurora/` 中对应什么。
6. 旧内容可以保留，但必须标明是历史补充，不得和当前主线冲突。

## 3. 增补优先级

### P0 必修

- 修标题/README/下一课预告不一致
- 修会导致 notebook 崩溃的错误
- 修答案泄漏和伪 TODO
- 修 stub 约定不统一

### P1 强烈建议

- 补桥接说明，讲清前后课关系
- 补真练习，不只讲解
- 补验证格，覆盖边界条件和 shape
- 补错误提示，把 Python traceback 翻译成人话

### P2 质量提升

- 补推导密度或为什么
- 补与生产代码的对照
- 补更好的图示、对比表、收束总结

## 4. 每课增补模板

每课只要按下面四块补，不必重写全文。

1. **桥接层**
   - 这一课学什么
   - 为什么放在这里
   - 和上一课、下一课如何衔接

2. **练习层**
   - 至少一个真实 TODO
   - 练习难度小步递进
   - 不预先暴露完整答案

3. **验证层**
   - assert 可靠
   - 覆盖边界 case
   - 发现错误时给出友好提示

4. **收束层**
   - 本课总结
   - 下一课预告
   - Aurora 连接

## 5. 判定规则

如果一条旧意见已经被当前 notebook 修掉，以当前 notebook 为准，不再把旧意见当现状。

如果一条旧意见仍成立，但可通过增补解决，就只增补，不删旧内容。

如果一条旧意见会影响学生通关、运行或验收，优先级一律按 P0 处理。

## 6. 质量底线

- 课题清楚
- 练习真实
- 验证可靠
- 运行稳定
- 前后衔接准确
- 教学深度有变化，不是整套模板复制

## 7. 使用方式

后续修改时先回答三个问题：

1. 这课的当前真实问题是什么？
2. 能否只靠增补解决？
3. 增补后是否同时满足教授意见和审计意见？

能同时满足时再改。

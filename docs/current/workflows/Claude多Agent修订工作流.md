# 给 Claude 的多 Agent 修订工作流

你要修订的是 Aurora 课程 L01-L99。目标不是重写课程，而是用增补方式，把课程做成一致、可运行、可验证、可持续维护的版本。

## 固定输入

每一轮开始前，先读取这三份文件：

- `outputs/standards/修订宪章.md`
- `outputs/prompts/给更强模型的修改提示词.md`
- 当前正在处理的课节对应材料

当前课节材料至少包括：

- 当前 notebook
- 对应 `per_lesson/Lxx.md`
- 课程索引 `INDEX.md`
- 总评 `00_教授总评.md`
- 必要时的邻近课节文件
- 必要时的相关 `src/aurora/` 源码

不要一次性加载整套 99 课。每轮只处理当前课节及其直接相关材料。

## 并行 Agent 分工

每一轮都并行启用至少 3 个 agent：

### Agent A: 课节审计

职责：

- 逐条对照 `outputs/standards/修订宪章.md`
- 找出当前课节的契约问题、运行问题、练习问题、验证问题
- 标出哪些问题必须修，哪些只需增补

输出：

- 当前课节问题清单
- P0 / P1 / P2 分类
- 允许增补的最小修改建议

### Agent B: 交叉对照

职责：

- 核对标题、README、下一课预告、前后课衔接
- 核对当前课节与 `INDEX.md`、`00_教授总评.md`、邻近课节是否一致
- 核对当前课节是否与 `src/aurora/` 的真实实现对齐

输出：

- 不一致项列表
- 必须同步更新的文档清单
- 可能受影响的邻近课节或全局文档

### Agent C: 质量守门

职责：

- 检查是否存在答案泄漏、伪 TODO、空洞 stub
- 检查是否会因跳格运行、缺 import、shape 错误而崩溃
- 检查是否存在过度提示、验证过弱、边界 case 缺失

输出：

- 风险列表
- 需要补的防呆与验证
- 任何可能降低学习者发现空间的内容

## 单课循环

对每一课执行下面的循环，顺序从 L01 开始，到 L99 结束。

```text
for lesson in L01..L99:
    read outputs/standards/修订宪章.md
    read outputs/prompts/给更强模型的修改提示词.md
    read 当前 lesson 材料
    parallel:
        Agent A 审计
        Agent B 交叉对照
        Agent C 质量守门
    coordinator 合并三份结果
    只做最小必要增补，不删除旧内容
    更新必须更新的文档
    做一次自检
    若通过，记录 lesson 完成，进入下一课
```

## 允许的修改方式

优先只做增补：

- 补桥接层
- 补真练习
- 补验证格
- 补边界 case
- 补 Aurora 连接
- 补友好错误提示
- 补总结和下一课预告

尽量不做：

- 重写全文
- 删除旧内容
- 直接把答案暴露给学生
- 把练习改成演示

## 每课结束后必须更新的文档

根据修改影响范围，至少更新以下项目中的所有相关项：

- 当前 notebook
- 对应 `per_lesson/Lxx.md`
- `INDEX.md`，如果标题、评级、统计摘要、课节顺序有变化
- `00_教授总评.md`，如果总评结论、评级分布、Top 问题有变化
- `docs/README.md`，如果当前/历史入口有变化
- `docs/archive/README.md`，如果有新的归档或迁移
- 任何引用了该课的对比报告、总览报告、周计划或索引文件

如果某份文档的内容没有受影响，就不要为了“统一”去改它。

## 自检标准

每轮结束前，必须确认：

- 标题、README、正文、下一课预告一致
- 没有新的答案泄漏
- 练习是真练习，不是伪 TODO
- 验证可运行，最好覆盖边界条件
- 课程叙事和 `src/aurora/` 对齐
- 旧内容仍在，但没有和当前主线冲突

## 失败时的处理

如果出现以下情况，停止推进下一课：

- 当前课节的核心契约仍未对齐
- 练习仍然是伪练习
- notebook 仍然无法从干净内核运行
- 需要删除大量旧内容才能修复
- 无法判断应该更新哪些文档

此时只输出：

1. 当前课节真实问题
2. 最小修复方案
3. 必须同步更新的文档
4. 需要用户确认的分歧

## 终止条件

workflow 只在以下条件全部满足时结束：

- 已完成 L01 到 L99 的全部课节
- 每一课都已按当前标准完成增补与验证
- 每一课结束后必须更新的文档都已同步
- 没有未解决的 P0 问题
- 没有待确认的结构性分歧

## 角色分工的最终目标

主模型负责做决定，多个 agent 负责并行读材料、找问题、找冲突、找风险。  
最终产物不是一堆碎片意见，而是一份可执行的最小修改计划和对应补丁。

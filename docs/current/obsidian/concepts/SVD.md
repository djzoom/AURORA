# SVD · 奇异值分解（Singular Value Decomposition）

[[_lifecycle|← 生命周期总表]] · [[../domains/linear-algebra|← Linear Algebra 领域]] · [[../INDEX|← Master Index]]

> **⚡ 一句话（Layer 8）**：把**任意**一个矩阵按"重要性"排好序拆成一叠方向，前几个方向往往就装下了绝大部分信息——SVD 不是为了"算"，是为了**扔掉不重要的方向**。

---

## 📖 定义
奇异值分解把任意 `m×n` 矩阵写成 `A = U Σ Vᵀ`：U、V 是正交矩阵（左/右奇异向量），Σ 是对角矩阵，对角线是从大到小的奇异值 `σ₁ ≥ σ₂ ≥ …`。等价地 `A = σ₁u₁v₁ᵀ + σ₂u₂v₂ᵀ + …`——一叠秩 1 矩阵的加权和。只留前 k 项就得到最优的**低秩（low-rank）近似**。

## 🤔 为什么工程师要发明它（Layer 9）
> 摘自 L14「🤔 为什么工程师要发明它？」

- **不用它会怎样？** 一张 1000×1000 的图片是一百万个数字；推荐系统的"用户 × 电影"评分表可能上亿格。原样存储、传输、计算都太贵，而且里面大量信息是重复冗余的。工程师需要一种"抓主干、扔细节"的通用办法。
- **它解决了什么真实问题？** SVD 把任意矩阵按重要性排序，只留前 k 个奇异值就得到又小又几乎一样的低秩版本——**压缩、去噪、降维，本质都是这一招**。
- **后面哪里还会再用到？** L17 用[[Eigendecomposition|特征分解]]看系统稳定性，**L84 的 LoRA** 用两个低秩因子微调大模型；推荐系统与 embedding 降维可视化都是同一把钥匙。

## ⚠️ 容易混淆（Layer 4）
| 别搞混 | 它是什么 | 一句话区分 |
|---|---|---|
| **特征分解** | `A=PDP⁻¹`，需方阵 | SVD 对**任意矩阵**都成立；奇异值 ≥ 0，特征值可负/复 |
| **PCA** | 数据协方差的主方向 | PCA 本质是**对中心化数据做 SVD** |
| **低秩近似（truncation）** | 只留前 k 项 | 那是 SVD 的**用法**，不是分解本身 |

## 🧑‍🏫 白板要求（Layer 6）
**★★★★** — 面试高频概念，必须做到：
- ✅ 写出 `A = UΣVᵀ` 与 `Σσᵢuᵢvᵢᵀ`，说清 U、V 正交、σ 降序
- ✅ 解释低秩近似为何是"最优"（Eckart–Young，按 σ 截断）
- ✅ 讲清与特征分解的关系：σᵢ² 是 AᵀA 的特征值
- 🔸 完整数值算法不要求闭卷手推

## 0. 排序原则

1. **断链优先**：夹在已润色课程链条中间的漏网课，前后课桥接文字已更新而它没动，
   衔接错位风险最高，学生按顺序学习时最先撞上。
2. **主线优先**：核心技能链（DSP → 自动微分 → KWS 实战 → ASR）先于支线模块
   （音乐、LLM）；支线先于收尾模块（integration）。
3. **实战课从严**：涉及外部依赖（torch、数据集、音频文件）的项目课（KWS、Whisper
   微调）要额外验证"无外部资源也能跑通"的回退路径。
4. **每批次验收**：每完成一批即跑 `validate_pipeline.py`，并抽查批内相邻课桥接。

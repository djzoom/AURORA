# Aurora 课程 FAQ（常见问题）

> 一门从零开始、拒绝黑盒的音频 AI 课程:99 节课、约 6 个月,从一条正弦波(sine wave)到 Whisper,
> 全部用 NumPy 手写并验证到 `< 1e-10`。**目标不是拿 offer,而是成为真正懂原理的构筑者(builder)。**
>
> 找不到你的问题？欢迎在仓库开一个 Issue,或直接把困惑贴给 AI 助教(见 [答疑一节](#-答疑作业与作品集))。

## 🧭 这门课是什么 / 适合谁

**Q：Aurora 到底是什么？**
一套从零实现的音频 AI 课程 + 一个 from-scratch 的研究系统。你不调 `librosa`、不调 `whisper`,而是**亲手写** FFT、STFT、MFCC、反向传播、注意力、CTC、RAG,再和参考实现逐点比对。

**Q：这门课为谁准备？我适合吗？**
适合**对"引擎盖底下发生了什么"好奇、想真正搞懂原理**的人:学生、转行者、自学者、工程师、研究者、有音频/音乐/广播背景的创作者。只要你有好奇心和耐心,就适合。

**Q：学完的目标是什么？是为了找工作吗？**
目标是**成为基础扎实、真正懂原理的构筑者**。当你需要向别人——面试官、同事,或未来的自己——证明你真的懂时,这套训练顺带让你游刃有余;但那是副产品,**不是目的**。面试只是"证明你懂"的一个场合而已。

**Q：我完全不会编程 / 只会一点 Python，能学吗？**
能。会写 `print`、循环、函数就够起步;不会也能边学边补。真正零基础请先看 [`GETTING_STARTED.md`](GETTING_STARTED.md)。

**Q：我数学很差，能学吗？**
能。数学**从零讲起**——向量、矩阵、导数、概率、复数全部在课里重建,不需要基础。每个难点(复数、反向传播、CTC)前面都有工程动机 + 类比 + 小例子的缓坡。

**Q：我已经会机器学习了，这门课会不会太基础？**
不会太基础——因为它**什么都从零写**。很多做过 ML 的人也未必能在白板上推出 FFT、CTC 前向、或手写反向传播。你可以快速过掉熟悉的部分,重点啃"从零实现"那一层。

**Q：英文不好有影响吗？**
没有。讲解是中文,专业术语带英文标注(`中文（English）`),既读懂直觉,又顺便对接英文论文和交流。

**Q：这门课免费吗 / 有证书吗？**
课程材料在仓库里开放。它是**自学课**,没有官方证书——但你学完手里会有**一份真实的作品集**(你的实现、笔记、demo),那比证书有说服力得多。

## 📚 课程内容与结构

**Q：一共多少课？怎么分模块？**
99 课(L01–L99),11 个模块:基础前导 → 复数三角 → 线性代数 → 微积分 → 概率 → **音频 DSP(主场)** → 深度学习 → 语音识别 → 音乐 AI → 大模型 → 整合收官。完整地图见 [`notebooks/README.md`](../../../notebooks/README.md)。

**Q：为什么全部从零手写，不直接调库？这不是浪费时间吗？**
恰恰相反——**从零写一遍,你才真的懂**。调库你永远不知道内部发生了什么;手写逼你理解每一步数学,而且和 numpy 对齐到 1e-10 证明你写对了。这门课的价值就是**深度**。

**Q：我只想学 LLM / 只想学 DSP，能跳着学吗？**
技术上可以,但**强烈建议按顺序**。后面的模块大量复用前面的(注意力用点积、RAG 用 TF-IDF、Whisper 用你搭的 Mel)。跳课会断层。

**Q：课程会教 TTS（语音合成）吗？**
目前**没有** TTS 模块——课程教了"听"(ASR),"说"(TTS/声码器)是留给未来(Aurora v2)的方向。

**Q：会用到 PyTorch 吗？**
会,但**很晚**。前 53 课(数学 + 音频 DSP)只用 numpy;深度学习模块(L59 起)才引入 torch,后面的音乐嵌入和部分 ASR / 本地 LLM 章节也会用到。

## 🛠️ 环境与工具

**Q：怎么装环境？**
看 [`GETTING_STARTED.md`](GETTING_STARTED.md)——装 Python、建虚拟环境、`pip install -e ".[dev,notebooks]"`、注册 Jupyter 内核、`jupyter lab`,约 30 分钟。

**Q：我用 Windows / Mac，有区别吗？**
基本一致。Windows 装 Python 时**务必勾选 "Add Python to PATH"**;激活虚拟环境的命令略有不同(Mac:`source .venv/bin/activate`;Windows:`.venv\Scripts\Activate.ps1`)。

**Q：`python3: command not found` 怎么办？**
没装好或没加 PATH。重装 Python(Windows 勾 PATH),然后**重开终端**再试。

**Q：pip 装很慢 / 失败？**
网络问题,换国内镜像:`pip install -e ".[dev,notebooks]" -i https://pypi.tuna.tsinghua.edu.cn/simple`。

**Q：Jupyter 里找不到 `Python (AURORA)` 内核？**
补跑一次 `python -m ipykernel install --user --name aurora --display-name "Python (AURORA)"`,再关掉浏览器标签重新 `jupyter lab`。

**Q：运行代码格报 `ModuleNotFoundError: numpy`？**
内核选错了。右上角把内核切成 `Python (AURORA)`。

**Q：需要 GPU 吗？**
前 53 课**纯 CPU**。深度学习模块(L59 起)装 torch,KWS 模型很小、CPU 也能跑。只有"真训练大模型/微调 Whisper"才需要云 GPU。

**Q：能用 Google Colab 吗？**
可以——Colab 自带 Python + Jupyter,把仓库 clone/上传进去,`pip install` 后即可跑;深度学习课还能白嫖它的免费 GPU。

**Q：学到深度学习课报 `No module named 'torch'`？**
到 L59 了才需要 torch。若继续跑音乐嵌入、ASR 或本地 LLM,再按需:`pip install -e ".[speech,music,llm]"`。

**Q：用什么编辑器？VS Code 行吗？**
Jupyter Lab 最直接;VS Code 装 Jupyter 扩展也能开 `.ipynb`。复习知识图谱建议用 Obsidian 打开 `docs/current/obsidian/`。

## 🎧 怎么学 / 学习方法

**Q：每节课怎么学？**
读讲解(科普风)→ 填 `✏️ TODO` → 看 `✅` 检查格 → 卡住看 `solutions/` → 做课末的**白板挑战**(合上电脑手推一遍)。

**Q：一定要按 L01 → L99 顺序吗？**
强烈建议。每课为下一课打地基。

**Q：运行到某格报 `NotImplementedError`，是坏了吗？**
**不是,这是设计好的。** 那是你还没填的 `✏️ TODO`。报错信息通常会告诉你该实现什么。填上就好。

**Q：参考答案在哪？该不该看？**
每个有练习的模块旁边有 `solutions/` 文件夹(参考实现 + 讲解)。**建议先自己认真想 10 分钟再看**——卡住再突破,才记得牢。

**Q：白板挑战一定要做吗？**
强烈建议做。它是检验你**真懂**还是**背过**最直接的方式——合上电脑还能推出来,才是真的会了。

**Q：怎么知道我"真的懂了"，而不是背过？**
三个信号:①能在白板上脱稿推导;②能给别人(或 AI)讲明白;③换个数字/场景你还能做对。做到这三点,就是真懂。

**Q：卡在一课好几天了怎么办？**
①先看 `solutions/`;②把你卡的那一步贴给 AI 让它解释;③睡一觉再看(复数、反向传播这种断崖,常常一觉醒来就通了);④实在不行,先标记跳过、继续往下,回头再补。

**Q：知识图谱（Obsidian）是干嘛的？要用吗？**
`docs/current/obsidian/` 是一张知识网:每个术语的「生命周期 / 依赖 / 易混淆 / 白板要求」都在里面,还有一张[面试冲刺图](../obsidian/interview/Audio-AI.md)。复习时非常好用,但不是必须——用 Obsidian 打开能看到双向链接的网络图。

**Q：学完具体会什么？**

## ❓ 答疑、作业与作品集

**Q：没有老师，我遇到问题问谁？**
你有三个"老师":①每课的 **✅ 检查格**(即时判分);②`solutions/`(参考答案 + 讲解);③**AI 助教**(见下)。

**Q：怎么"交作业"？**
不用交给别人——**检查格就是批改老师**,打 ✅ 就对了。你真正的"作业成果"是把实现和笔记**沉淀成自己的作品集**。

**Q：怎么用 AI 帮我学？（具体做法）**
卡住时,把你的代码 / 报错 / 困惑**贴给 AI**(Claude、ChatGPT 等),让它:
- **解释**你没看懂的那一步(比搜索快一百倍,随叫随到);
- **帮你 debug** 你的实现;
- **当考官考你**:让它出题、你白板作答、它点评你的推导;
- **复核你的白板推导**:把你纸上推的 FFT/CTC 打字给它挑错。
> 问得越具体(贴代码、贴报错、说清卡在哪一步),它帮得越准。

**Q：怎么把学习变成作品集？**
①把你的实现 + 笔记**开源到 GitHub**;②每个硬核实现写一条 thread / 一篇短博客("我手写了 FFT,踩了这三个坑");③上线一个**能点的 demo**;④用 Obsidian 维护你自己的知识图谱。学完你手里就是一个**做出来、讲得清、别人能点开的真东西**。

**Q：我发现课程里有 bug / 错误，怎么办？**
太好了——请直接发邮件到 [dev@0xGarfield.com](mailto:dev@0xGarfield.com)。最好顺手附上这几项:
- 课程号 / notebook 文件名

如果你也方便,可以同时开 Issue 或 PR,但邮件是最快的直达通道。

**Q：能和朋友一起学吗？**
非常推荐。一个人啃 99 节硬核课容易半途而废;有人和你在同一个蝶形图前卡住、又在同一个"啊原来如此"会心一笑,这条路会好走得多。

## 🔧 疑难杂症（其他常见坑）

**Q：notebook 里图画不出来 / matplotlib 卡住？**
确认装了 `notebooks` 附加包;图会内嵌显示,无需 `plt.show()`。

**Q：我改了代码，运行却没生效？**
Jupyter 有状态:改完要**重新运行那一格**(`Shift+Enter`);依赖它的下游格也要重跑。必要时 Kernel → Restart 从头跑。

**Q：`make test` 报错某个包没装（sklearn/torch）？**
少数深度学习课依赖 torch/sklearn。跑课程验收门用 `python scripts/validate_pipeline.py`(不需要 torch);要跑全部单测再按需 `pip install -e ".[speech,music,llm]"`。

**Q：老听到"两态验证"，是什么意思？**
每课有两种状态:**学生态**(你还没填答案,只在 TODO 处报错)和**答案态**(填入 solutions 后从头跑到底 `exit 0`)。这保证课程"真的能跑通",不是"看起来对"。

**Q：为什么有些课没有 `solutions/`？**
可视化课(🎨)和元课程(如白板演练、复盘)通常没有代码 TODO,所以没有参考答案文件——它们的"练习"是纸笔或自我评估。

**Q：术语后面括号里的英文是干嘛的？**
每个专业名词第一次出现都带英文(和缩写),比如 `梅尔频率（mel）`——让你既读懂中文直觉,又能无缝对接英文论文和技术交流。

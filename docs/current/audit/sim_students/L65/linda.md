# 琳达 — L65 学习日志

## 我理解了什么（简要）
1. **DAG 图可视化**：计算图用节点（Value对象）和边（算子）表示，每个节点存 `data` 前向值和 `grad` 反向梯度。反向传播沿 `_prev` 依赖往回走。
2. **Loss 曲线诊断过拟合**：Train loss 应单调下降，Val loss 停止下降甚至上升就是过拟合信号，通过 early stopping 及时停训。
3. **混淆矩阵定位误分类**：对角线 = 正确，非对角线 = 误分类；行归一化 = 各类 recall，能看出哪对词被互相混淆。

## 遇到的困难和问题

在 cell-l65-0539，示例 `L = (a*b) + b**2`，给出 `dL/da=3, dL/db=8`。  
问题：这两个梯度值是怎么从公式推出来的？  
- 是对 L 分别对 a、b 求偏导吗？但代码没有列出公式推导步骤。  
- 课程说梯度"沿着 `_prev` 往回走"，但从 L=15 反向到 dL/da 的具体流程是什么？乘法链式法则？  
需要课程补充：**逐步反向传播示例**（从损失点开始，一步步算梯度）。

### 【理解】Early stopping 的触发条件表述有歧义
题目说"最近连续 patience 个 epoch 的 val_loss 都高于当前最小值时，返回当前最小值对应的 epoch"。  
问题：这个判断条件到底是什么？  
- 是"连续 patience 个 epoch 都**没有新的最小值**"吗？  
- 还是"连续 patience 个 epoch 的 loss 都**严格大于历史最小值**"？  
看示例 `losses_a=[0.9,0.7,0.5,0.6,0.65,0.7,0.72,0.75]`，epoch 3 后再没降过，应该在 epoch 6 才返回 epoch 3（patience=3 时等待 3 个 epoch）。但代码注释的措辞让我不确定精确的逻辑。  
需要课程补充：**状态机描述** 或**伪代码**，明确 `no_improve` 计数器何时递增、何时重置。

### 【理解】Mel 滤波器的三角形设计没有原因说明
Cell-l65-0547 的 `naive_mel()` 用三角滤波器累积能量：每个 mel 滤波器在中心频率达峰，两边线性衰减到 0。  
问题：为什么必须是三角形？  
- 为什么不用方形、高斯或其他形状？  
- 三角形对应人类听觉的哪个特性？  
- 代码段 `if f0 <= f < f1: fb[m-1,k] = (f-f0)/(f1-f0+1e-12)` 是线性插值，但为什么这样做能"模拟人耳感知"？  
需要课程补充：**mel 滤波器的物理或生理背景** —— 为什么选这个形状，参考资料在哪。

需要课程补充：**梯度范数的诊断阈值** 和**示例**（什么值是爆炸、什么值是消失）。

### 【实践】Mel 参数选择的原则不清楚

问题：什么时候该用哪个值？

- HOP=160 对应什么的采样率？怎么计算一个合理的 HOP 值？  
- 这些超参数跟模型容量、样本量、目标精度有什么关系？  
需要课程补充：**超参数选择指南** 或**经验法则**（比如"HOP 通常取 N_FFT 的 1/4"）。

### 【理解】混淆矩阵行 vs 列归一化的用途差异不明确

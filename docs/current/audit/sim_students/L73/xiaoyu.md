### 【理解】DP 回溯逻辑完全看不懂

课程说"回溯时走对角线是替换，向上走是插入，向左走是删除"，但没有图啊！代码里 `dp` 是个二维表，`hyp` 和 `ref` 分别对应行和列吗？为什么向上走就是插入、向左走就是删除？这个几何直观完全缺失，看到代码里那堆 `dp[i][j]` 和 `i-1`, `j-1` 就懵了。

需要：**一张网格图，标上 hyp 词在横轴、ref 词在纵轴，画出几个回溯路径（对角线/向上/向左），标注对应的操作**。

### 【推演】三个回溯条件怎么判断的？

为什么 `dp[i][j] == dp[i-1][j-1] + 1` 就是替换？为什么 `dp[i][j] == dp[i-1][j] + 1` 就是插入？

在初始化的 DP 时，应该选 `min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1]) + 1`，这说明有三个"前一个状态"。回溯时只有三个条件，为什么不需要检查 `dp[i][j-1]`？如果三个值都加 1 后相等怎么办？

需要：**清楚地讲第一次做 DP 时是如何 min 三个方向、第二次回溯时是如何逐个检查这三个条件的**。

### 【理解】'C' 操作和 WER 的关系

代码里有 `ops.append('C')`（匹配），但讲 WER 的公式里只有 S/D/I，没有 C。那 C 算到 WER 里吗？课程说"完全识别正确"，是指 C 操作，还是整句没有 S/D/I？

需要：**明确说 C 和 S/D/I 的关系，以及最后 WER 只用 S/D/I 来算，C 是验证的**。

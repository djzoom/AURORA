# 琳达 — L73 学习日志

## 我理解了什么

**WER = (S+D+I) / N** 把词错误分成三类：替换（说错了个词）、删除（漏说了）、插入（多说了）。通过 DP 编辑距离的回溯就能数出这三类各多少个。整体 WER 数字一样的模型，错误原因可能完全不同，所以得分解开看。

## 遇到的困难和问题

### 【推演】DP 编辑距离回溯的方向对应关系

代码里从 `(i, j)` 回溯，根据 `dp[i][j]` 来自哪里（`dp[i-1][j-1]`、`dp[i-1][j]` 还是 `dp[i][j-1]`）判断走哪个操作。但课程里说"向上走是插入，向左走是删除"，这对应的是矩阵的行列位置。我没理清"向上"(i-1)、"向左"(j-1)、"对角线"(i-1,j-1) 为什么对应 I、D、S 这三个错误。希望能有一张图或者更直白的说明：假设 i 轴是 hyp 长度，j 轴是 ref 长度，那么走哪个方向为什么就表示删除/插入。

### 【计算】代码演示的第一个例子

```
hyp = ['the', 'cat', 'sat']
ref = ['the', 'big', 'cat', 'sat', 'down']
```

代码给出 S、D、I、WER 的最终结果，但 notebook 里没显示输出。我试着手算 DP 表和回溯，DP 表构造倒还好（编辑距离本身），但从 `(3,5)` 反向回溯到 `(0,0)` 的操作序列，我心算了一遍不太确定对不对。希望课程能显示这个例子的详细回溯过程，或者提供一个更简单的例子让我能手工验证。

### 【理解】WER > 100% 在实际中怎么理解

课程说"插入词超过参考词就 WER > 1.0"，数学上没问题，但现实中这意味着什么？是不是说模型"幻觉"得太厉害，输出的词数是参考文本的好几倍？这会不会提示我们需要在后处理阶段加长度约束或置信度过滤？还是说这样的句子本身就应该被标记为"完全失败的识别"单独处理？

### 【实践】`analyze_errors()` 中 `worst_examples` 的返回格式

代码框架里说"找 WER 最高的 5 个样本（含索引）"返回到 `worst_examples`，但没明确数据结构。应该是 `[(idx, wer), ...]` 还是 `[(hyp, ref, wer), ...]` 还是别的格式？这样在后续分析时，是直接用索引查原始数据，还是要把 hyp/ref 也存起来便于看内容？

### 【理解】S/D/I 比率与修复方向的对应

课程表格说"S_rate 高 → 增加多样性，D_rate 高 → 检查 VAD/padding，I_rate 高 → 降低 beam width"。这些修复建议的背后逻辑我理解得不深——为什么替换错误多就需要更多数据多样性，而不是改模型架构？为什么删除错误提示"信道噪声"而不是别的？这些诊断路径好像需要一些语音识别或模型训练的背景知识。

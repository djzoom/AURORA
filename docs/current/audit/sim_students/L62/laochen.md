# 老陈 — L62 学习日志

## 我理解了什么

这课的核心就是把一段音频打成"标准地图"给神经网络吃。具体是：读WAV → 统一长度（补零或截断到16000个采样点） → 用mel频谱表示成时频图 → 把数值归一化（均值=0，方差=1）→ 装进PyTorch DataLoader批量喂模型。关键认识是**mel频谱比原始波形更简洁**（从16000维降到40×T），而且**pad/truncate + per-sample归一化** 是实践细节，确保模型训练时梯度稳定。

## 遇到的困难和问题

### 【理解】Mel频谱为什么是"人耳感知"的表示，而不直接用FFT线性频率？
课程表格对比了三种表示，说Mel频谱对应"人耳感知频率"。但这句话对我来说太魔法了——**人耳怎么就感知成了对数频率而不是线性的？**
- 低频的100Hz和200Hz，人耳能区分吗？  
- 高频的10000Hz和10100Hz，人耳能区分吗？  
是不是说人类对频率的分辨率不是均匀的？这个原理课程没讲清，我只能记住"Mel更好"这个结论，但没真正理解为什么。

### 【推演】`_synth_keyword()` 中相位的计算逻辑
第6节代码有这一行：
```python
phase = 2 * np.pi * np.cumsum(freq) / SR
```
我卡在这里。我知道 `sin(2π × 频率 × 时间)` 能生成正弦波，但：
- **为什么要用 `np.cumsum(freq)` 而不是直接 `freq * t`？**  
- 我理解`cumsum`是累积求和，但频率在时间轴上累积？这和扫频音有什么关系？
- **为什么要除以SR(采样率)？** 单位是怎么约的？

这个推导没看透，感觉缺少了从"频率随时间变化"到"相位"的中间步骤。

### 【计算】从16000采样点到T=32帧，怎么推导？
课程验收标准说 `output.shape == (40, 32)`，后面的注释解释是"n_fft=2048、hop_length=512"时T=32。但：
- **怎么从这两个参数算出来帧数=32？**  
- 我猜是 `(16000 - 2048) / 512 ≈ 32` 之类的公式吗？  
- 为什么选n_fft=2048和hop_length=512？有没有更直观的理由，还是说这就是工程经验值？

### 【理解】`mel_spectrogram()` 返回(T, n_mels)然后要`.T`转置成(n_mels, T)
课程第10节表格说mel_spectrogram输出是(T, n_mels)，然后要转置。但为什么：
- **mel_spectrogram一开始不直接返回(n_mels, T)形状？**  
- 转置后(n_mels, T)是"频率在行、时间在列"。我知道CNN常见的图像格式是(channel, height, width)，所以频率作为"channel"有道理，但为什么mel_spectrogram的设计者当初要返回相反的顺序？是数学上更自然吗？

### 【理解】为什么`KWSDataset.__getitem__`里要检查标签合法性，抛出ValueError？
第17节代码最后有：
```python
if label not in self.label_map:
    raise ValueError(f"未知标签 '{label}'，...")
```
注释说"严格拒绝未知标签，避免OOB索引导致CrossEntropyLoss静默错误"。我有两个疑问：
- **什么是"OOB索引"？** 我猜是out-of-bounds，但为什么标签不在label_map里就会索引越界？
- **CrossEntropyLoss"静默错误"是什么意思？** 为什么不主动抛错，反而"静默"？这种错误有多危险？

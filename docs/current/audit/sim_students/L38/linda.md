# 琳达 — L38 学习日志

## 我理解了什么

FFT 就是用分治把 DFT 算快了：把序列按偶/奇下标分成两部分，各算一个小的 DFT，再用"蝶形合并"公式（`X[k] = E[k] + W·O[k]`）拼成完整结果。这样从 N² 次乘法降到 N·log N，对 4096 点的信号能快 200 多倍。`butterfly()` 函数就是做这个合并的工作——输入两个 N/2 的 DFT，输出一个 N 的 DFT。

## 遇到的困难和问题

### 【理解】旋转因子 W_N^k 的含义和由来
课程直接给公式 `W_N^k = exp(-2πi·k/N)`，但没说这是什么、为什么要这样算。这是从 DFT 定义 `X[k] = Σ x[n]·exp(-2πi·kn/N)` 里来的吗？为什么叫"旋转"因子？复数乘法和旋转有什么几何关系？我能背下来套公式，但不知道背后在干什么。

### 【推演】"W_N^(k+N/2) = -W_N^k"的推导过程
课程说这个等式成立，所以下半部分用减号就对了。但我推不出来：从 `exp(-2πi·(k+N/2)/N)` 怎么变成 `-exp(-2πi·k/N)`？涉及 `e^(iπ) = -1` 的推导，我卡住了。

### 【理解】为什么一定要按"偶数/奇数"下标分？
课程给了一张递归树，显示 `[0..7]` 怎么拆成偶数组 `[0,2,4,6]` 和奇数组 `[1,3,5,7]`。但没解释：是不是任意分法都能用蝶形合并？还是说，"偶奇交替"这个特殊规律有数学必要性？

### 【实践】蝶形网络图（N=8）看不懂
那张信号流图（signal flow graph）有三层彩色的 X 形交叉，标着 `W_8^k` 等。我知道每个 X 代表一个蝶形单元，但为什么是这个连接顺序？"位反转序"是什么意思？什么时候在代码里用到这个图？

### 【理解】递归版和迭代版（位反转排列）怎么对应
课程说 L38 教递归分治树（最容易懂），但生产代码 `aurora.audio.transforms.fft()` 用迭代 + 位反转排列。这两个是同一个算法吗？"位反转"怎么对应递归版的"奇偶拆分"？为什么生产版不用递归？

### 【计算】半步练习的 DFT([1,1]) 手算对不对
我自己算了：`DFT([1,1])` 应该是 `[2, 0]`（用公式 X[k]=Σ x[n]·exp(-2πi·kn/2)），但没法验证，只能靠代码。这种两三项的手算对初学者来说很重要，能不能给个简化的计算步骤说明？

### 【理解】闭卷题"展开 k=log₂ N 层求和式"怎么写
T(N) = 2T(N/2) + O(N) 的递推式，我知道要展开成 `T(N) = O(N) + O(N) + ... + O(1)`，但怎么从递推关系一步步写成求和式？对数的定义我会，但递推展开的步骤没学过。

### 【实践】需要一个 N=2 蝶形的玩具例子
课程讲的最小单位是 N=4（两层递归）。但 N=2 的蝶形（把两个单点 DFT 合成两点 DFT）完全没讲。这样我理解的最底层还是"单点输入"，想看看 1→2 这一步具体怎么算。

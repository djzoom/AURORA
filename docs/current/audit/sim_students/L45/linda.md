# 琳达 — L45 学习日志

## 我理解了什么

STFT 输出复数矩阵，我们只保留幅度（丢掉相位）、平方后变成功率、取对数变成 dB 尺度。用 matplotlib 的 `imshow` 画这个矩阵时，要转置、设 `origin="lower"`、用 `extent` 标好时间和频率坐标。这样就能像 X 光一样看声音的时频分布，440 Hz 和 880 Hz 的纯音会显示成两条清晰的水平线。

## 遇到的困难和问题

### 【理解】功率谱的"单位"是什么
开篇说"幅度谱，单位：任意；功率谱，单位：watts"。但代码中 `pow = A**2` 只是数学平方，没有任何单位转换。如果原始 STFT 是"任意"单位，那 A² 怎么就变成 watts 了？这里的"单位"是物理意义上的瓦特，还是只是打比喻？应该解释一下。

### 【推演】eps 值在两种形式中的等价性
课程说"20·log₁₀(A + 1e-8)（振幅形式）与 10·log₁₀(A² + 1e-10)（功率形式）近似等价，但 eps 楼层略有差异"。我知道 log(x²) = 2·log(x)，所以 10·log(A²) = 20·log(A)。但**加上 eps 之后**，当 A 很小（比如 A=1e-6）时，两个式子的值会差很大啊：前者约 20·log(1e-8)≈ -160 dB，后者约 10·log(1e-10)≈ -100 dB。什么时候它们才算"等价"？需要定量说明 A 的取值范围。

### 【计算】STFT 帧数公式没有明确给出
白板挑战的 Q1 要求计算 `n_frames = 1 + (N - win_len) // hop`，但这个公式在前面没有被明确推导或列出。只有看到对答案的代码才知道。这个公式是怎么来的？帧数的定义（例如是否包含部分帧、是否有边界处理）应该在"概念"部分或之前讲清楚。

### 【理解】extent 参数的顺序为什么是这样
说 extent 的顺序是 `[x_left, x_right, y_bottom, y_top]`，对应 `[0, duration_s, 0, sr/2]`。这个顺序（特别是 `y_bottom` 和 `y_top`）似乎与常见的 `[x_min, x_max, y_min, y_max]` 不同。是 matplotlib 的特殊约定吗？为什么要这样设计？文档中应该说明这是按什么逻辑排列的。

### 【理解】为什么 STFT 的最高频率是 sr/2
"最高频率: {f_bins[-1]:.0f} Hz (= SR/2 = {SR//2} Hz)"，即奈奎斯特频率。这涉及采样定理，但课程（至少这一课）没有解释。为什么不能表示高于 sr/2 的频率？这对初学者来说是"魔法"。建议在前面某一课补充"采样定理快速说明"。

### 【实践】导入中的别名 `_stft` 有点混淆
验证代码中用 `from aurora.audio.stft import stft as _stft`，而前面的导入是 `from aurora.audio.stft import stft`。这两个是同一个函数吗？为什么要起别名？这样做是为了避免命名冲突吗？应该注释说明。

### 【理解】eps 值选择的物理或数学依据
功率形式用 `1e-10`（对应 -100 dB 噪声底），振幅形式用 `1e-8`。为什么选这些特定的值？它们与听觉感知的下限有关吗？还是只是工程上的约定？这会影响我对"安静信号"的理解。

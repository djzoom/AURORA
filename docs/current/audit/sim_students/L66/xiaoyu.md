# 小雨 — L66 学习日志

## 我理解了什么

- **ASR 系统的三层链条**：音频波形→特征编码器（提取 80 维 log-Mel 频谱）→语言解码器（生成词）→文字。现代系统用 Transformer，不需要手工设计发音词典。

- **WER 评估指标的含义**：就是对错的词数除以总词数，错误分三类——替换（说错词）、删除（漏说）、插入（多说）。计算用编辑距离，可以大于 1.0。

## 遇到的困难和问题

### 【理解】编码器的"下采样"和 2× 是什么意思？
> Cell 5 代码注释："Whisper 用 2 层 conv，其中第二层 stride=2 → 2× 下采样"，输出 `(T//2, 512)`

我看不懂"下采样"的直观含义。是删除一半的帧吗？从 300 帧变成 150 帧的这个操作有什么作用？这涉及卷积层的参数 stride，但课程没有解释为什么编码器要做这个。

### 【理解】log-Mel 处理流程的每一步是什么？
> Cell 8："stft → |·|² → mel_filterbank · power → log(max(·, 1e-10))"

每个箭头都看不懂：
- stft 是傅里叶变换吗？

- mel_filterbank 是一个矩阵吗？怎么和 power 乘？
- 为什么要取 log？是数据压缩还是为了让模型更好学？

### 【推演】Levenshtein DP 的核心逻辑推导缺失

- 为什么"相同则不操作"（`dp[i-1][j-1]`），"不同则三选一"（min 三种操作）？这里对应的操作是什么——在词列表上删词、加词、改词吗？
- 边界条件 `dp[i][0]=i, dp[0][j]=j` 是什么意思？为什么是递增？
- 最后 return 的是 `dp[H][R] / R`，这是在计算什么比值？

我没办法从零填出 TODO。

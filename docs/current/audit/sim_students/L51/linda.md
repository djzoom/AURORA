# 琳达 — L51 学习日志

## 我理解了什么

这课教怎么把真实的 WAV 文件从头到尾过一遍完整的特征提取流水线：读 WAV → 算 STFT 频谱 → 转成 Mel 频度 → 提 MFCC 系数，每一步都能画图看。元音的频谱图上会看到平行的亮纹（基频和谐波），辅音是短促的宽带闪光。采样率影响频率分辨率。

## 遇到的困难和问题

### 【理解】Mel 频度是什么及为什么要用
课程说"压缩后的感知频率"，但没有解释：
- 人类听觉为什么是非线性的？
- Mel 频度跟线性频度具体怎么对应（是有公式吗）？  
- 为什么语音处理要用 Mel 而不是直接用 Hz？

这对理解 mel_spectrogram 为什么要存在很关键。

### 【推演】谐波为什么加权 `1.0 / k`
代码 `make_vcv_signal` 里：
```python
vowel_a = sum((1.0 / k) * np.sin(2 * np.pi * 120 * k * t_a) for k in range(1, 9))
```
为什么第 k 个谐波的幅度是 `1/k`？这是从哪个物理或数学公式推导出来的？高中物理没学这个规律。

### 【计算】FFT 输出为什么是 `n_fft//2 + 1` 个频率分量
代码 `mag.shape[1]` = `n_fft//2 + 1`，而不是 `n_fft`。课程说频率分辨率是 `sr/n_fft`，但：
- 为什么实时只返回 n_fft//2 + 1 个值（不是 n_fft）？
- 为什么要 +1？
- 这涉及"实数 FFT 的对称性"吗？

### 【理解】什么是"中心填充"（center=True）
白板题说"两端各反射填充 `n_fft//2`"，然后 `n_frames = 1 + N // hop`。：
- 什么是"反射填充"？为什么要填充而不是直接分帧？
- 如果不填充，n_frames 会怎样变？对最后的 MFCC 有什么影响？

### 【计算】练习 1 的帧数公式为什么是 `duration / 0.01`
练习要求"验证帧数 ≈ `duration / 0.01`"（当 hop=160 时）。我知道 `160 / 16000 = 0.01` 秒，所以这个公式是 `duration / (hop/sr)`，即 `duration * sr / hop`。课程能不能直接写清这个推导，而不是只在提示里说？初学者容易以为 0.01 是个魔数。

### 【理解】C0 系数为什么是"总能量"
白板题说"C0 ∝ sum(log Mel energy)"，代码验证是：
```python
from aurora.audio.mfcc import dct_ii
x_ones = np.ones(40)
c = dct_ii(x_ones)
assert abs(c[0] - np.sum(x_ones) / np.sqrt(40)) < 1e-10
```
这个验证证明了 DCT 的第一个系数等于所有输入的和除以 √N。但课程没解释：
- DCT 为什么第一个系数是这样？（需要 DCT 的数学定义，但我没学过）
- 为什么 log 和 sum 合起来就表示"总能量"？

### 【计算】谐波频率范围为什么到 8000 Hz
白板题答案代码：
```python
harmonics = [f0 * k for k in range(1, int(8000/f0)+1)]
```
8000 Hz 是 sr/2（奈奎斯特频率）。但课程没明说：为什么谐波不能超过奈奎斯特频率？是因为采样定理吗？那为什么在这个语境里要强调这一点？

### 【理解】librosa 对比验证的 2 dB 阈值
代码检查 Aurora 与 librosa 的差距 < 2 dB：
```python
assert diff.max() < 2.0, "Aurora 与 librosa Mel 差距过大..."
```
为什么阈值是 2 dB？这个数字从哪来的？太大了会说明什么问题？

### 【实践】三个练习都是 `NotImplementedError`
练习 1、2、3 都没有样板代码，只有 TODO 和 assert。我知道 Python 怎么写，但：
- 练习 2 的"对比"具体要展示什么？是画两张图还是打印数值？
- 练习 3 要我实现 `make_vcv_f0` 但没说什么地方要改，我必须翻回上面的 `make_vcv_signal` 去对比，不太友好。

# 老陈 — L47 学习日志

## 我理解了什么

log-Mel是语音识别的标准特征，就是一个5步流水线：原始波形 → STFT(拆成频率) → 平方变功率谱 → 用三角滤波器把频率轴压成80个Mel档 → 取log压缩范围。最终输出一个"热力图"，每一行是一帧，每列是一个Mel频率档，数字表示那一帧在那个频率的响度。

eps=1e-8这个很聪明——防止log(0)变成-∞爆掉，把最小值限在约-18.4。

课文说是因为"人耳低频分辨率高、高频分辨率低"和"临界带宽掩蔽效应"。但我高中物理没学过人耳的临界带宽是啥——课文提了这个概念但没有铺垫。我能背下"三角形让相邻滤波器50%重叠"这句话，但不理解为什么恰好50%，这个50%是怎么来的？为什么不是30%或70%？

### 【推演】矩阵乘法 power @ fb.T 的形状推导

课文说 `power(T,F) @ fb.T(F,M) → (T,M)`，这我能验证。但为什么**一定要转置**才对？我重新思考了一下：

- `fb` 是 shape `(n_mels, n_fft//2+1)`，即**每行一个Mel滤波器**
- 如果不转置直接 `power @ fb` 会是 `(T,F) @ (M,F)` → 不能乘
- 转置后 `fb.T` 变成 `(F,M)`，所以 `(T,F) @ (F,M) → (T,M)` ✓

我现在明白了：`power @ fb.T` 的每个数字是"第t帧的功率谱向量" · "第m个Mel滤波器向量"，也就是做**内积**（点乘）——把功率谱投影到每个Mel滤波器上。只是我花了半分钟才推出这个逻辑，课文的表述对我来说有点跳跃。

### 【理解】滤波器"峰值≈1"与"不归一化面积"的区别

最后的推导题说Aurora用"HTK惯例——三角形峰值≈1，**不做**面积归一化"。这段话我需要理清：
- "峰值≈1" 意思是每个三角滤波器的最高点都是1
- "不归一化面积" 意思是：即使最高点都是1，宽的三角（高频）的面积会比窄的三角（低频）大很多
- 这带来的后果是：高频的Mel能量会自动被放大（因为滤波器面积大），低频会被压小

这样设计有意义吗？我没有看到课文解释为什么HTK选择不归一化。如果选择了Slaney归一化（面积都相等），会怎样？

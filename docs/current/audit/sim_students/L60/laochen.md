# 老陈 — L60 学习日志

## 我理解了什么

1. **计算图和叶/非叶节点**：`requires_grad=True`创建的Tensor是"叶子"（用户直接建的），运算结果是"非叶子"（自动生成）。叶子在backward后自动填`.grad`，非叶子默认不保存（省内存）。

2. **backward的核心原理**：跟L56的Value引擎一样——backward()按反向遍历整个计算图，逐个计算梯度。每个运算都有对应的梯度函数（`grad_fn`）记录"我是怎么算出来的"。

3. **梯度累积和清零的必要性**：每次backward梯度会加到`.grad`上而不是覆盖，所以训练循环里必须调`zero_grad()`清零。

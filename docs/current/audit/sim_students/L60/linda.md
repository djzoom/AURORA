# 琳达 — L60 学习日志

## 我理解了什么

这节课让我看到 PyTorch 和手写 Value 引擎其实在做同一件事——建一个计算图，记录每个运算，然后反过来求梯度。核心区别是：叶节点（我创建的张量）的梯度会自动保存到 `.grad` 里，中间运算产生的张量需要 `retain_grad()` 才能查看它们的梯度。梯度累积这个细节很重要，每次 `backward()` 都是加法，所以必须 `zero_grad()` 清零。

## 遇到的困难和问题

### 【理解】叶节点和非叶节点的设计逻辑（第 2 节）

**卡在原文**："叶节点由用户直接创建，非叶节点由运算派生"

**具体困惑**：这个分类的界线有点模糊。为什么 `a = torch.tensor(2.0, requires_grad=True)` 就天然是叶节点，而 `c = a * b` 就必须是非叶节点？感觉只是"哪里来的"这个事实，而不是某种必然的数学区分。

**为什么在意**：因为这决定了 `.grad` 的存储方式，但课程没解释"为什么必须这样分"。是为了显存而人为划分吗？还是有更深的原因？

---

### 【推演】链式法则在梯度计算中的具体应用（第 11 节）

**卡在原文**："`loss1 = z * z` → `dL1/dw = 2z·x = 36.0`"

**具体困惑**：这步从 `dL/dz = 2z` 跳到 `dL/dw = 2z·x` 用的是链式法则吗？
- z = w*x，所以 dz/dw = x
- 然后 dL/dw = dL/dz · dz/dw = 2z · x？

我推出来了，但课程的表述太紧凑，没有明确说是链式法则。如果这确实是链式法则，希望能更清楚地标明。

---

### 【理解】非叶节点梯度默认为 None 的实际含义（第 2 & 6 节）

**卡在原文**："中间激活值可能有数百万个，全存会爆显存"

**具体困惑**：这是说，对于一个大模型，如果我们想保留每个中间层（激活函数输出、注意力分数等）的梯度，显存会瞬间爆炸？那实际训练中，是不是根本不需要看中间梯度，只关心参数（叶节点）的梯度就够了？

**为什么在意**：这关系到我后续调试模型时到底应该看哪些梯度。课程需要补充：什么时候*必须*看中间梯度，什么时候可以忽略。

---

### 【理解】`backward(retain_graph=True)` 和 `retain_grad()` 的区别（第 3.5 节）

**卡在原文**：表格说两个 API"正交"，可同时使用

**具体困惑**：
- `retain_grad()` 保留非叶节点的 `.grad` **值**
- `backward(retain_graph=True)` 保留**计算图结构**

这两个为什么要分开？不能一个 API 搞定吗？课程的"多任务学习"例子太抽象，我看不出为什么一定要在同一个前向图上做两次 backward。

**为什么在意**：这两个概念容易混淆，课程需要一个更具体的场景（比如真实的 GAN 或多任务网络代码）来说明为什么不能合并。

### 【计算】verify_gradients() 的手算验证（第 4 & 12 节）

**卡在原文**：`L = 2*a*b + 3*b` → `∂L/∂a = 2b`，`∂L/∂b = 2a + 3`

**具体困惑**：这些是关于两个变量的偏导数。我按高中数学推了一遍，确实是对的：
```
L = 2ab + 3b
∂L/∂a = 2b ✓
∂L/∂b = 2a + 3 ✓
```
但我是按"把 b 当常数、对 a 求导"的机械方法推的，没有真正*理解*为什么这样推是对的。偏导数这个概念对我来说还是有点陌生。

**为什么在意**：这是后续所有梯度计算的基础，如果我对偏导数只会机械应用，就会遇到更复杂的梯度推导时卡壳。

# 小雨 — L60 学习日志

## 我理解了什么

1. **计算图记录原理**：每个 `requires_grad=True` 的 Tensor 运算都会自动挂上 `grad_fn`（像是一个"做过什么"的标签），PyTorch 用这个标签反向流梯度。这和 L56 手写的 `Value` 类其实是同一个想法。

2. **叶节点梯度有，中间节点梯度没有**：用户直接创建的 Tensor（叶节点）的梯度会自动保存到 `.grad` 里；中间算出来的 Tensor（非叶节点）的梯度默认丢掉，除非我主动调 `retain_grad()` 才能看。

3. **backward 前必须清零**：每次 `backward()` 都是把新梯度**加上去**而不是覆盖，所以必须 `zero_grad()` 清空旧的，不然梯度会莫名其妙变大。

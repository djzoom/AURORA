# 琳达 — L35 学习日志

## 我理解了什么

**欧拉公式是"单位圆上的坐标"的复数写法**：`e^{iθ} = cos θ + i·sin θ`，可以用 Python 代码直接拼出来。**FFT 旋转因子**就是把特定的角度（`-2π·k·n/N`）代入欧拉公式，生成一个模长恒为 1 的复数。代码上两个函数的逻辑很清楚：`euler()` 返回复平面坐标，`twiddle()` 是 `euler()` 的包装。

## 遇到的困难和问题

### 【理解】欧拉公式的推导过程不见了
课程直接说"欧拉公式就是这样"，给出 `e^{iθ} = cos θ + i·sin θ`，还用"秒针旋转"类比。但**为什么实部就是 cos、虚部就是 sin**？这个对应关系是从哪儿来的？有没有更严格的推导（比如泰勒级数）或者几何证明？高中数学里我没学过，课程也没补。

### 【理解】复数乘法"旋转"的几何本质没讲清
代码验证 `z * 1j` 旋转 90°，结论是对的。但**为什么乘以虚数单位就能旋转？** 两个单位圆上的点相乘（比如 `(cos θ₁ + i·sin θ₁) × (cos θ₂ + i·sin θ₂)`），结果的模为什么还是 1？用勾股定理可以验证 `|e^{iθ}| = √(cos²θ + sin²θ) = 1`，但这只是验证，不是理解。

### 【理解】FFT 旋转因子的负号为什么必须是负的
课程说 `twiddle(k, n, N) = e^{-2πi·k·n/N}`，解释是"负号是 DFT 的习惯约定，表示顺时针旋转（分析方向）"。这太抽象了。**为什么频率分析的时候一定要顺时针？为什么不能逆时针？** 背后有没有更深层的原因（比如某个求和公式里的性质）？

### 【理解】twiddle 函数在 FFT 全流程中的位置不明确
课程说 "L37-L39 重写 FFT 全靠今天这两个函数"。但本课没有给出 FFT 的公式或伪代码，我看不出 `twiddle` 是怎么嵌入 DFT 求和 `X[k] = Σ x[n] · e^{-2πikn/N}` 的。只是看到代码里有 TODO 和验证，没有流程图或例子。

### 【实践】"手动拼 cos+sin" 和 "直接用 np.exp(1j*theta)" 的区别是什么
实现提示其实就是答案：`return np.cos(theta) + 1j * np.sin(theta)`。但课程说"手动拼可以让你亲眼看到欧拉公式成立"——如果我直接用 `np.exp(1j*theta)` 替代，数值结果是完全一样的吗？两种写法有性能差异吗？如果一样，为什么非要手动拼？如果不一样，为什么课程不说？

## 老陈（laochen）

1. **【理解】矩阵乘法定义为什么必须是"逐行点积"？高中的行列式是这样乘的吗？**
   → 在"## 1. 矩阵×向量：逐行点积"标题 cell 之后，新增插播 markdown cell："这个『逐行点积』的规则，
   到底是从哪冒出来的？"——用"配方书"类比说明动机，并说明这和高中行列式是完全不同的对象，
   点出定义是为了让矩阵乘法能表示"函数复合"（呼应第 5 节）。

2. **【理解】"逐行点积"和"列的线性组合"两个视角为什么等价，没有推导。**
   → 在"🔗 Aurora 连接"cell 之后、"🎨 图示：矩阵×向量=列的线性组合"标题之前，新增代数推导 markdown +
   可运行代码 cell：把 `y[i]=Σⱼ A[i,j]x[j]` 按列重新分组，推出 `A@x = Σⱼ x[j]·A[:,j]`，并用具体数字核对，
   代码里同时跑两种算法断言相等。

3. **【理解】为什么 (m,n)@(n,)=(m,)，而不是列数决定输出维度？**
   → 与第 1 条合并在同一张插播 cell 中，用"配方书有 m 页 ⟹ m 个输出；每页要读完 n 个输入 ⟹ 输入 n 维"
   直接解释行数/列数与输入/输出维度的对应关系。

4. **【推演】旋转矩阵 cos/sin 的来源，高中三角函数怎么突然出现在矩阵里？**
   → 在"## 2. 旋转矩阵"标题之后、绘图代码之前，新增插播 markdown："cos、sin 为什么会跑到旋转矩阵里？"
   从单位圆上一点 (cosφ, sinφ) 出发，用高中两角和公式推出旋转后坐标 `(x cosθ − y sinθ, x sinθ + y cosθ)`，
   写成矩阵形式，并说明与"列的线性组合"视角吻合（R 的两列分别是 R@[1,0]、R@[0,1]）。附带复数视角的可选说明。

5. **【理解】(A@B)@x = A@(B@x) 为什么总是成立，没有证明。**
   → 在"5. 乘法=函数复合"代码验证 cell 之后，新增插播 markdown："(A@B)@x == A@(B@x)——这为什么总是成立？"
   给出从"逐行点积"定义出发、交换双重求和顺序的完整代数证明。

6. **【理解】对角矩阵为什么会有"逐分量缩放"这个性质，是巧合还是必然？**
   → 在"参数实验：对角矩阵与全1方阵"cell 中追加说明：非对角元素为 0 ⟹ 点积中对应项自动归零 ⟹
   只剩对角项，直接从"逐行点积"定义推出，非"看结果反推"。

7. **【实践】`laviz` 可视化在本机无法运行，希望有文字描述。**
   → 在"🎨 图示"标题 cell 中追加一段⚠️提示：说明如果导入失败可跳过，并用文字描述这张图画的是
   矩阵列向量被输入分量拉伸后首尾相接求和的过程。

8. **【计算】白板挑战里的三角函数值容易混淆，希望有速查表。**
   → 在旋转矩阵插播 cell 中新增"特殊角度速查表"（0°/30°/45°/60°/90°/180°/270° 的 cos/sin），
   并附一个用 45° 走一遍的手算示范（不涉及白板挑战本身的 90° 答案），供学生套用到白板挑战。

# 小雨 — L48 学习日志

## 我理解了什么

音频信号被切成重叠的小块（分帧），每块做 DFT 得到一个频谱。把所有频谱横向排列就成了时频热力图。然后用 Mel 滤波器组在频率轴上做加权平均，把高频细节压缩掉，这样同样维度就能装进更多人耳在乎的低频信息。线性频谱图上升频信号在高频占很多行，但在 Mel 图上就平均分布了。

## 遇到的困难和问题

### 【理解】Mel 转换公式从天而降
"mel = 2595 * log10(1 + f / 700)" 这个公式为什么长这样？为什么是 2595 和 700 这两个神奇数字？课程只说"转换公式"，但没有解释这是人耳感知特性总结出来的，还是怎么设计的。能否补充：这个公式是怎么来的，或者给一张图对比人类听觉和 Mel 刻度的对应关系？

### 【理解】"反射填充"是什么黑魔法
center=True 时会在信号两端"反射填充 n_fft // 2 个样本"。反射填充是什么意思？为什么不是补零？为什么要填充？为什么填充量恰好是 n_fft // 2？这一段涉及窗函数的泄漏问题，但课程没有给图或直观解释。

### 【理解】窗函数公式的参数细节
"w[n] = 0.5 * (1 - cos(2π n / N))" 中，n 的范围是 0 到 N-1 吗？为什么分母写的是 N 不是 N-1？后面提到"对称窗的 N-1"和"周期窗的 N"，这两者波形"极相近"但"严格意义不同"。作为初学者，这是太细微的差别了，看不出区别。能否用图对比周期窗和对称窗在实际应用中的效果差异？

### 【推演】帧数公式怎么推出来的
"帧数 = 1 + (N - n_fft) // hop_length"，这个公式为什么是这样？假设信号长 N=1000，n_fft=256，hop=64，我想象第一帧在 [0:256]，第二帧在 [64:320]，……最后一帧什么时候停？怎么能推出上面这个公式？center=True 时变成 "1 + (N + 2·(n_fft//2) - n_fft) // hop_length"，这更加看不懂了。

### 【理解】为什么矩阵乘法就是 Mel 能量
代码里 `power_check @ fb.T` 计算 Mel 能量。前面说"Mel 滤波器组在频率轴上做三角形加权平均"，但为什么变成矩阵乘法就能实现加权平均？fb 的每一行是什么？为什么要转置？能否用一个小例子展开：比如只有 3 个频率 bin 和 2 个 Mel 滤波器，手工演算一下怎么从频谱得到 Mel 能量？

### 【理解】Chirp 信号在 Mel 轴上为什么"均匀上升"
chirp 频率在线性上升（200 → 8000 Hz），但在 Mel 图上轨迹看起来是均匀的。这是为什么？是因为 Mel 公式把线性频率拉伸了，所以同样的线性频率跨度在 Mel 轴上被压缩了？能否补充一张图：标出 chirp 在几个时刻点的线性频率和对应的 Mel 值？

### 【理解】HTK vs Slaney 为什么有两套标准
课程说 aurora 用 HTK 惯例（三角形峰值都是 1.0），Slaney 是除以带宽。为什么有两种选择？选 HTK 对"高频滤波器会攒到更多能量"是"故意设计"还是"历史遗留"？这个选择对后续的 log-Mel 和 DCT 有什么影响吗？

### 【理解】n_mels=80 这个数字从哪来
"通常取 n_mels=80 个滤波器"。80 是音频 AI 的工业标准吗？还是实验得出来的最优值？为什么不是 64 或 128？这涉及采样率、频率范围、人耳分辨率，但课程没有给任何理由。

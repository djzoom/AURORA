# 小雨 — L44 学习日志

## 我理解了什么

STFT 其实就是把音乐分成一小段一小段，对每小段做频率分析（FFT）。核心三步很清晰：**切帧** → **每帧乘窗** → **每帧做 FFT**。相比之前在 L43 学的原理，这课是真实代码实现，用 `my_stft()` 把三步串起来，最后得到一个时间×频率的复数矩阵，这就是声谱图的原始数据。`center=True` 的反射补零很聪明，让帧中心能对齐信号开头。

## 遇到的困难和问题

### 【理解】STFT 核心公式中的数学符号太密集

> 出现位置：L44-0435 "## 概念 1：STFT 流水线" 中的公式
> 
> ```
> STFT[t, f] = sum_{n=0}^{N-1}  x[t*hop + n] * w[n] * exp(-j*2*pi*f*n/N)
> ```

我一看到求和符号 Σ 、三层下标（[t,f]、w[n]、x[]）、虚数单位 j、指数函数 exp()，就害怕了。我知道这个公式对应"切帧→乘窗→FFT"这三步，但中间的跳跃逻辑我推不出来。比如：
- 为什么 n 要从 0 到 N-1？这对应什么？
- `t*hop + n` 这个下标是什么意思，怎么对应"第 t 帧"的？
- `exp(-j*2*pi*f*n/N)` 这一项为什么就能把信号转成频率？

希望课程能**用画图或类比**逐步拆解这个公式，而不是直接说"实现上等价于……"跳过。比如，用一个具体的例子（比如 N=8 的小 FFT）展开计算，看实际数字是怎样从公式里生出来的。

### 【推演】帧数公式 n_frames = 1 + L // hop 的推导

> 出现位置：L44-0447 "## ✏️ 闭卷推导检查格 — STFT 帧数公式"

代码验证中说 n_frames = 1 + len(signal) // hop（当 center=True 时）。这个公式怎么推出来的？

我尝试推：
- 补零后信号长度 L + N_FFT（左右各补 N_FFT/2）
- 第 0 帧从下标 0 到 N_FFT-1
- 第 t 帧从下标 t*hop 到 t*hop+N_FFT-1
- 最后一帧要求 t*hop+N_FFT-1 ≤ (L+N_FFT-1)，即 t*hop ≤ L，所以 t ≤ L//hop
- 帧数 n = (L//hop) + 1

但这里有几个我不确定的地方：
- 为什么右边界是 (L+N_FFT-1) 而不是 (L+N_FFT)？
- 如果 L 不能整除 hop，最后剩的那点数据怎么处理？
- center=True 时，左边的反射补零怎么算进去？

希望课程给出**逐步的推导过程**，或者至少说明"补零后数据范围是多少，第 t 帧取哪些下标"。

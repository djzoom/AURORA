# 老陈 — L44 学习日志

## 我理解了什么

STFT 就三个步骤串起来：先把长信号切成一段段（`frame_signal`），每段乘一个钟形窗函数（消边界跳变），再对每段做FFT求频谱。最后堆成一个表格，行是时间（第几帧），列是频率（哪个频率成分），格子里填的是那一时刻那个频率有多强。这就是音乐软件里看到的频谱图背后的东西。实现时关键是要用手写的FFT（L37-39），不能偷懒用numpy，为的是从零理解算法。

### 问题 3【推演】center=True 时帧数 n_frames 的推导
notebook 最后要求"闭卷推导"帧数公式。给定的是：
- 补零后长度 `L + N_fft`
- 跳步 `H`

我试着推：第一帧起点是 0，第 t 帧起点是 `t*H`；最后一帧的起点 + 窗长必须 ≤ 补零后长度，即 `t*H + N_fft ≤ L + N_fft`，所以 `t ≤ L/H`，帧数是 `1 + ⌊L/H⌋`。

但问题是：**为什么补零两端各 N_fft//2 个零后，帧数公式没变**？直观地说，补零应该让我多切几帧啊？notebook 没解释补零如何影响帧计数。

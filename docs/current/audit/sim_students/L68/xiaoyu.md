# 小雨 — L68 学习日志

## 我理解了什么

CTC 是一个"多帧压缩成少字"的算法，核心是引入 **blank 符号** ∅ 来填补帧数和字符数的差异。压缩分两步：先去掉相邻重复的字符（`a-a → a`），再去掉 blank。一个目标序列对应很多条不同的路径（例如 `a ∅ b` 和 `a a b` 都压缩成 `ab`），**贪婪解码** 每帧取最大概率字符，然后压缩，是一个快速但不保证最优的解码方法。

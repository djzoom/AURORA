# 琳达 — L42 学习日志

## 我理解了什么

- FFT 是把 DFT 的递推计算分成多层蝶形，从 O(N²) 降到 O(N log N)，通过"折叠冗余"加快
- 实数信号的频谱对 bin = N/2 镜像对称，所以只用看前半段就完了
- 纯音是一根尖峰，和弦是几个峰，噪声是平的噪点——听音乐就能"看见"不同信号的频谱长相

## 遇到的困难和问题

### 【理解】DFT 矩阵定义的数学含义 {cell 5}
"DFT 把时域信号投影到 N 个不同频率的复指数基上"，然后直接写出 F[k,n] = W^(k·n)，其中 W = exp(-2πi/N)。但没有解释为什么用这个 exp(-2πi·k·n/N) 的形式就能提取频率。我高中没学过"基"和"投影"这些线性代数，也不懂复指数 exp(-2πi·...) 怎么就代表某个频率了。

**希望补充**：复指数在复平面上的几何意义（为什么是旋转？）；为什么不同的参数 k·n/N 能"选中"不同频率分量（可以用一个具体的数值例子）。

### 【理解】酉矩阵和验证代码的含义 {cell 5 末尾}
代码验证 `F @ conj(F).T = N·I`，说这证明了"酉性"，但完全没解释酉矩阵是什么东西、为什么这个条件重要。conj() 和转置我懂，但为什么把这两个操作组合起来验证一个特殊的矩阵乘法关系？

**希望补充**：酉矩阵的直观解释（比如"保持向量长度不变的线性变换"）；这个性质在 FFT 能量守恒或计算中有什么用。

### 【推演】Cooley-Tukey 递推公式的推导 {cell 6}
直接甩出公式 `X[k] = E[k] + W^k·O[k]` 和 `X[k+N/2] = E[k] - W^k·O[k]`，一点推导都没有。不知道为什么能把原来的 N 点求和分成偶数和奇数两部分，也看不出怎么从 `sum x[n]·exp(...)` 的定义推到这个递推形式。

**希望补充**：从原始 DFT 求和公式出发，分离偶奇项的完整步骤（最好再简化到 N=4 或 N=8 的小例子逐步演示）；说明为什么 E[k] 和 O[k] 各自就是 N/2 点的小 DFT。

### 【理解/实践】位逆序置换是干什么用的 {cell 8 练习}
代码给出 `bit_rev = [int(f'{i:03b}'[::-1], 2) for i in range(8)]`（3位二进制反转），但没说为什么迭代式 FFT 一定要这样打乱输入顺序。蝶形网络的图里也看不出这个重排和分层结构的关系。

**希望补充**：为什么位逆序是必要的（迭代式 FFT 的层结构和顺序有什么关系）；用 N=8 的具体例子，先排原序 [0,1,2,3,4,5,6,7]，变成位逆序后的样子，再对着图看一遍，说明这样重排后第一层蝶形怎么工作。

### 【推演】共轭对称性 X[N-k] = conj(X[k]) 的数学原因 {cell 7}
说实数输入导致这个对称性，代码也验证了，但没有任何推导。完全不知道这个结论怎么来的，只能死记硬背"实数就是对称的"。

**希望补充**：从 DFT 定义（或复指数的性质）出发，一步步说明为什么实数输入一定满足这个关系；给一个 N=4 的小例子，手工算出全部 4 个 DFT 值，验证左右对称。

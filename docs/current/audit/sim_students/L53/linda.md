# 琳达 — L53 学习日志

## 我理解了什么

**L53 的核心故事**：40 维 Mel 频谱不是平等的——相邻 bin 高度相关（频谱包络平滑），所以 DCT-II 这个"找光滑基函数的工具"把能量集中到前 13 个系数，剩下 27 个系数几乎是零。这就是为什么音色可以用 13 维向量描述。代码完整展示了：`mel_spectrogram → power_to_db → dct_ii → 截取前13维 = MFCC`。

## 遇到的困难和问题

### 问题 1：DCT-II 公式的「直观动机」【理解】
课程给出的公式：
```
y[k] = sum_{n=0}^{N-1}  S[n] * cos( pi/N * (n + 0.5) * k )
```
没有解释**为什么**用余弦基函数。我知道这是 FFT 的"实数版"，但为什么在频谱分析这个具体场景下，余弦就能自动把光滑信息集中到低频系数？是否与信号的"连续性"或"能量分布"有某种对偶关系？

### 问题 2：「ortho 正规化」的具体含义【理解】
代码写 `norm="ortho"`，课程说"正交归一化后 `k=0` 对应常数基"。但附录里验证 `c[0] = mean(x) * sqrt(N)` 时，这个 `sqrt(N)` 是 ortho 加上的吗？ortho 具体改了 DCT 公式的哪一部分？为什么 k=0 时的系数不是直接的 `sum(S[n])` 而要乘个 `sqrt(N)`？

### 问题 3：DCT 正交性 `D @ D.T = I` 有什么用【推演】
附录验证了这个性质，然后 TODO 练习说 "用 `dct_truncated @ B` 做逆变换"。但我不懂：
- 为什么正交性能推出逆变换？
- `B` 应该是 `D` 本身还是 `D` 的某种变形？
- "B 行正交 → 近似逆变换" 这句话怎么理解？

### 问题 4：「倒谱 = 对数谱的逆傅里叶变换」怎么转化成代码【理解】
课程说倒谱能"把卷积拆成加法"（声道 × 激励）。这涉及卷积定理吗？为什么一定要在对数域做逆变换？代码里为什么是用 DCT-II 而不是真的做 IFFT？

### 问题 5：`power_to_db` 为什么必需【理解】
代码中：
```python
log_mel_frame = power_to_db(mel_spec[10], top_db=None)
dct_all = dct_ii(log_mel_frame, norm="ortho")
```
为什么直接 `dct_ii(mel_spec)` 不行？是因为 DCT 对数值范围有要求，还是因为人耳感知遵循对数律，所以必须先取 log？

### 问题 6：DCT 为什么优于 DFT/IFFT【理解】
课程说 "MFCC 用 DCT-II 近似 IDFT"。但为什么不直接用 IFFT？是计算速度快，还是 DCT 对实数信号更合适，还是两者都有？

### 问题 7：TODO 逆变换练习具体怎么做【实践】
练习说要"构建 DCT-II 正交基矩阵 B"，然后 `log_mel_recon = dct_truncated @ B`。但：
- B[k, n] 的矩阵形状是 (40, 40) 还是 (13, 40)？
- 为什么是右乘（@ B）而不是左乘？
- 这样重构出来的是否是某种"伪逆"？

### 问题 8：c[1] 和 c[2] 的"倾斜"和"弯曲" physically 什么意思【理解】
表格说 c[1] 是"频谱包络的低频倾斜"，代码对比低频/高频音色时确实 c[1] 有正负。但这个"倾斜"和人听到的 "低音" vs "高音" 有什么对应关系？是否 c[1] > 0 就一定是低音主导？

### 问题 9：为什么前 13 个系数能保留 >95% 的方差【推演】
代码验证了这个现象，但没有证明**为什么** DCT 在 Mel 频谱上就自动有这个性质。是否与 Mel 尺度的自然特性（声学基础）有关？还是任何平滑信号做 DCT 都会这样？

### 问题 10：L54 为什么要学「Value 计算图」【理解】
附录 A 说 L53 是"前向特征"，L54 开始要学"反向梯度"。但我想知道：计算图和 MFCC 这些手工特征有什么接口？是说神经网络要学会"自己提特征"而不是用人工的 MFCC 吗？

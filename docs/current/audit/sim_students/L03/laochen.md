## 遇到的困难和问题

### 问题 1【推演】扫频公式里的 `t**2 / (2 * t[-1])` 我推不出来，照直觉写还写错了

位置：扫频 cell，`sweep = np.sin(2 * np.pi * (f0 * t + (f1 - f0) * t**2 / (2 * t[-1])))`，注释只有一句"积分形式，瞬时频率 f(t)=f0+(f1-f0)t/T"。

我的直觉是：既然频率随时间从 200 升到 3000，那就把 f(t) 直接代进正弦——`np.sin(2*np.pi*(f0+(f1-f0)*t/T)*t)`。我真的跑了一下对比：我的写法结尾峰值扫到了 **5750 Hz**，几乎是 3000 的两倍；课程写法才是对的。也就是说"把 f(t) 代进 sin"这个最自然的想法是错的，但课程完全没提醒这个坑，只丢了"积分形式"四个字——积分是微积分的内容，我高中就没学明白，早忘光了。

希望补充：哪怕本课"不推导公式"，也请加一个直白的警示框："你可能想直接写 sin(2π·f(t)·t)，这是错的（试试看，它会扫到 5800 Hz）；正确公式为什么长这样，L37 之前给个链接或一句白话解释（比如'相位是频率一点点攒出来的，不是当前频率乘总时间'）。"现在这行代码对我就是黑盒里的黑盒。

### 问题 2【理解】"bin"是什么？为什么峰值是 437.5 Hz 而不是 440 Hz？

位置：show_spec 的注释"频率分辨率 = sr/NFFT = 62.5 Hz/bin"，以及后面验证 cell 打印"峰值 bin 频率"。

"bin"这个词全课没有任何定义，直接出现在注释里。而且我运行验证 cell，打印出来的峰值是 **437.5 Hz**，不是 440 Hz——课文前面明明说"一条水平亮线，位于 440 Hz"。我一开始以为是我跑错了，后来算了下 437.5 = 62.5 × 7，好像跟那个"62.5 Hz/bin"有关系：是不是频率轴只能取 62.5 的整数倍，440 落不到格子上，只能显示成最近的 437.5？如果是这样，assert 里"< 20"的容差是不是就是为这个留的？

希望补充：用一句话+一个类比解释"bin"（比如"频率轴像尺子上的刻度，NFFT=256 时刻度间隔 62.5 Hz，440 Hz 会被归到最近的刻度 437.5"），并且验证 cell 的期望值直接写明"你会看到 437.5 而不是 440，这是正常的"，免得初学者以为自己跑错。

### 问题 3【实践】`spectrum.mean(axis=1)`——行和列哪个是频率哪个是时间？

位置：验证 cell，"# 找峰值所在频率 bin  peak_bin = spectrum.mean(axis=1).argmax()"。

注释说 specgram 返回 (spectrum, freqs, t_bins, im)，但没说 spectrum 这个二维数组的形状是什么。我打印了一下是 (513, 17)：513 行对应频率、17 列对应时间。可 axis=1 到底是"对行平均"还是"对列平均"我一直分不清，只能靠试。如果这里 axis 写错成 0，argmax 出来的就是某个时间 bin 而不是频率 bin，结果完全是另一个意思，代码却不会报错。

希望补充：加一行注释写明 `spectrum.shape = (频率数, 时间帧数)`，并说明"mean(axis=1) = 沿时间方向取平均，把 17 帧压成一条平均频谱"。对分不清行列的人，这一行注释能省半小时。

### 问题 4【理解】白噪声：每个时间点取随机数，为什么就"所有频率都有能量"？

位置：白噪声 cell，"白噪声：所有频率同时随机响"，代码只有 `rng.standard_normal(len(t))`。

代码做的事我看得懂：每个采样点取一个随机数。但这是"时间上随机"，课文直接跳到"所有频率、所有时刻都有能量"——时间上的随机为什么等价于频率上的"全都有"？这中间对我是断层的。另外 `standard_normal`（标准正态）也没有半句解释，换成 `rng.uniform` 行不行？图会不会不一样？

希望补充：不要求推导，给个类比即可（比如"随机抖动没有偏爱任何节奏，快的慢的成分都掺在里面，所以每个频率都能分到一点能量"），并注一句"用别的随机分布画出来也差不多，L37 学了 DFT 就能解释为什么"。

### 问题 5【理解】§3 里"谐波"没定义就用了，和"共振峰"的关系也没说清

位置：§3 "浊音……有清晰的谐波（harmonics）结构"，以及元音代码里 700/1200 Hz 被叫做"共振峰"。

"谐波"这个词是第一次出现，没有任何解释（基频、共振峰好歹各给了一句话）。而且我糊涂的是：代码里元音就是三个正弦相加（130 + 700 + 1200 Hz），那 700 和 1200 算"谐波"还是"共振峰"？700 不是 130 的整数倍，好像不是谐波？课文说共振峰是"水平亮带"，可代码画出来是三条细线不是带。这两个概念在我脑子里现在是缠在一起的。

希望补充：给"谐波"一句定义（如"基频的整数倍：130、260、390……"），并在元音代码注释里说明"真实语音是很多条谐波、被共振峰包络加亮；这里为了简化只用三个正弦近似亮带的位置"。

## 遇到的困难和问题

### 问题 1【推演】扫频那一行公式，注释里的"积分形式"三个字等于没解释

位置：扫频 cell，`sweep = np.sin(2 * np.pi * (f0 * t + (f1 - f0) * t**2 / (2 * t[-1])))`，
注释写"积分形式，瞬时频率 f(t)=f0+(f1-f0)t/T"。

我卡住的地方：我第一反应是"频率随时间线性变化，那不就是把 f(t) 乘上 t 吗"，于是自己写了
`np.sin(2*np.pi*(f0+(f1-f0)*t/T)*t)` 跑了一下对比——结尾实际频率约 5655 Hz，几乎是 3000 Hz
的两倍！课程的写法（结尾约 2925 Hz）才是对的。可是**为什么**我的直觉写法错了、为什么要出现
`t**2/2` 这个东西，课程一个字没说。"积分"是我高中没学扎实的内容，"积分形式"四个字对我就是黑话。
开头明明承诺"本课不推导任何公式"，但这行代码本身就藏着一个我推不出来的公式。

希望课程补充：要么换成一句人话类比（比如"相位像里程表，频率像时速表——车速一直在变时，
里程不能用『当前时速×总时间』来算"），要么明确写"这行为什么长这样会在 LXX 课解释，现在照抄即可"。
现在这种"给公式但不给台阶"的状态最难受。

### 问题 2【理解】"bin""hop""频率分辨率"三个新词说了"不用理解"却反复出现

位置：`show_spec` 工具 cell 的注释："频率分辨率 = sr/NFFT = 62.5 Hz/bin"、
"每步跳 128 点（hop）= 8 ms，显示更连续（分辨率由 hop 决定，不是 overlap 本身）"。

我卡住的地方：注释开头说"两个参数现在不用理解"，紧接着却塞进 bin、hop、频率分辨率三个
没解释过的词，还附送一个纠错括号"分辨率由 hop 决定，不是 overlap 本身"——我连 overlap 决定
分辨率这个"错误说法"都还没听过，就先被纠正了一遍，完全不知道它在跟谁说话。而且 "bin" 不是
说说就算了：最后的验证 cell 里 `peak_bin`、"峰值 bin 频率"又出现了，全课没有一处告诉我
bin 是什么。

希望课程补充：要么把这些注释删到只剩"这两个参数 L43 细讲，先照抄"；要么用一句话定义
bin："频率轴被切成的一个个小格子，每格宽 sr/NFFT Hz"。二选一都行，别卡在中间。

### 问题 3【理解】§3 一段话抛出四个语音学术语，c10 的一堆数字来路不明

位置：§3 "语音谱图的特征"（共振峰/基频/清音浊音/谐波）以及 c10 的
`vowel_approx`（`0.5 * sin(f0) + 0.3 * sin(f1) + 0.2 * sin(f2)`，参数如 `130, 700, 1200`）。

我卡住的地方："共振峰：声道的谐振频率"——用一个我不懂的词（谐振频率）解释另一个我不懂的词
（共振峰），我读完只知道"有这么个东西"。然后代码里 0.5/0.3/0.2 三个系数、a 元音是 700+1200、
i 元音是 300+3000，这些数字哪来的？为什么把三个正弦加起来就"近似元音"了？注释只说"非精确
语音合成，仅用于谱图演示"，但演示的逻辑本身没讲。

希望课程补充：一个类比即可，比如"喉咙和嘴像一个可变形的音箱，嘴型不同就放大不同频段，
被放大的频段叫共振峰；表格里的 700/1200 这类数字是语音学家量出来的典型值，记不住没关系"。
再加一句"系数 0.5/0.3/0.2 只是让低频响一点，随便改也不影响结论"，我就踏实了。

### 问题 4【实践】验证 cell 里 spectrum 的形状没交代，NFFT 还悄悄从 256 换成了 1024

位置：验证 cell `spectrum, freqs, _, _ = ax.specgram(...)` 和
`peak_bin = spectrum.mean(axis=1).argmax()`。

我卡住的地方：`spectrum.mean(axis=1)` 我看得懂语法，但不知道 axis=1 平均掉的是什么——
自己跑了一遍打印 shape 才知道 spectrum 是 (513, 17)，即 513 个频率格 × 17 个时间帧，
axis=1 是把时间平均掉。这层含义注释里没写（只写了返回四个值）。另外这里 NFFT 突然从前面
一直用的 256 变成 1024，没有任何说明；我打印发现峰值其实在 437.5 Hz 而不是 440 Hz——
这恰好是"bin 宽 15.625 Hz、440 落不到格线上"的活例子，但课程没点破，我是自己撞明白的。

希望课程补充：给 spectrum 加一行注释 "形状 = (频率格数, 时间帧数)，mean(axis=1) 是对
时间取平均"；顺便说一句"打印出来是 437.5 而非 440，因为频率轴一格 15.625 Hz"，这比
assert 容差 20 Hz 的写法更能教到东西。

### 问题 5【解答】读谱图挑战按关键词判卷，意思对但用词不同就被判"⚠️"

位置：读谱图挑战验证 cell，`correct = {"图B": ["和弦", "chord", "多频", "三和弦"], ...}`。

我卡住的地方：图B 我第一次填的是"Do-Mi-Sol 三个音同时响"——意思完全对（题目里那三个频率
就是本课的 C-E-G），但不含任何关键词，被打了 ⚠️ 让我以为自己理解错了，回去重读 §2 才发现
是判卷太死。图D 同理，"声音越来越小的纯音"也过不了（关键词只认"衰减/decay/envelope/减弱"）。

希望课程补充：要么放宽关键词（图B 加 "同时/do/mi/sol/三个"，图D 加 "变小/变弱/越来越"），
要么在 ⚠️ 提示里写明"本格只做关键词粗查，若你确认语义正确可自行通过"，别让答对的人自我怀疑。

# 小雨 — L34 学习日志

## 我理解了什么

采样率太低就会把高频信号看错成低频——这叫混叠。比如用8kHz采样6kHz的正弦波，采样点落在2kHz的反相位上，完全看不出区别。所以CD采样率44100Hz是为了能容纳人耳听到的最高20kHz，Nyquist定理说采样率必须超过频率的两倍才行。Aurora在16kHz采样率下只用0-8kHz的频率，就是为了避免这种混叠污染。

## 遇到的困难和问题

### 【理解】Python `round()` 函数在公式中的含义
在 cell 中看到 `round(freq / sample_rate)` 这个计算，课程说它"找到最近的整数倍采样率"，但没有解释为什么。比如 `round(6000/8000) = round(0.75) = ?`——Python 会舍入成 1 还是 0？课程只在变量表里提到"Python `round(0.5)=0`（银行家舍入）"，但 0.75 不是 0.5，这个规则怎么应用？希望补充一句：round() 找的是离频率最近的"sr 的倍数"的序号（k），所以 round(0.75) 的结果代表第几圈。

### 【推演】混叠公式 `abs(freq - sr·round(freq/sr))` 的推导
看到手算表格（sr=8000, 6000Hz → 2000Hz），但跳过了推导过程。具体是：为什么要"先乘 sr，再取差"？从直觉上讲，6kHz 怎样一步步变成 2kHz？能不能补充一个中间推导，比如"6kHz 相当于 8000×0.75Hz，即多走了 0.75 圈；0.75 圈等价于向下走 0.25 圈；0.25 圈对应的频率是……"这样的链条？

### 【理解】两个混叠公式的等价性
代码里出现过两个不同的写法：
- 第一种：`abs(freq - sample_rate * round(freq / sample_rate))`
- 第二种：`abs(((f + nyquist) % sample_rate) - nyquist)`

这两个是一样的吗？能看出来吗？初学者看不出，希望课程说明一下为什么换个写法还是对的，或者就用一个式子。

### 【理解】sin 的相位翻转关系
验证码中的注释："正弦向下折叠会附带相位翻转，所以 `sin(2π(fs−f)n/fs) = −sin(2πfn/fs)`"。这是三角恒等式，但怎么证？为什么向下折叠时 sin 前面多了个负号？初学者没学过这个变换，单靠打印对比 `diff_minus` 能验证相等，但看不懂为什么相等。希望补充一行或一个 hint："（提示：用 sin(A−B) 展开，或者想象正弦波镜像翻转）"。

### 【计算】边界条件中 Nyquist 频率本身的返回值困惑
断言说 `predict_alias_freq(4000, 8000) == 4000.0`，即 Nyquist 频率返回自己。但"混叠是高频折叠成低频"的直觉上，4000Hz（刚好在折叠边界）为什么不产生混叠、还是返回 4000？能否补充一句解释：Nyquist 频率是能无混叠表示的最高频率，所以它本身不折叠？还是有其他数学理由？

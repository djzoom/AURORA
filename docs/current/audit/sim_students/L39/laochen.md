# 老陈 — L39 学习日志

## 我理解了什么

FFT 是把一个长 N 的信号**按偶数位和奇数位拆成两个长 N/2 的小问题**，各自递归做 FFT，再用"旋转因子"（某个复指数）把两个小答案**蝶形合并**成完整答案。递归到长度为 1 时直接返回（基础情形），这样一路往上层层合并，就能把 O(N²) 的 DFT 加速到 O(N log N)。代码上就是五个 TODO：递归基、拆分、旋转因子、蝶形、拼接。

## 遇到的困难和问题

### 【推演】Cooley-Tukey 拆分公式从何而来
课程直接给了 `X[k] = E[k] + exp(-2πi·k/N)·O[k]` 这个公式，但没解释**为什么**偶奇拆分能这样合并。DFT 定义是 `X[k] = Σ x[n]·exp(-2πi·k·n/N)`，怎么通过 n 的奇偶性拆成这个形式？这一步推导过程要补充。

### 【理解】X[k+N/2] 为什么是减号
讲义说 `X[k+N/2] = E[k] - exp(-2πi·k/N)·O[k]`，只一句"不需要额外乘法"就交代了。但**为什么后半部分是减号而不是加号？** 是在 k 替换成 k+N/2 时自动推导出来的，还是有更深层的对称性？这个反问题一定得补充推导或至少说明来源。

### 【理解】旋转因子 W^k 与蝶形的具体关系
上一课（L38）讲了蝶形图，现在这课用"旋转因子"这个新名字，但 `exp(-2πi·k/N)` 和蝶形图里画的箭头、权重具体是什么对应？这个名词转换没有说清楚，容易混。

### 【实践】旋转因子在后半频段（k >= N/2）的计算
代码里写 `k = np.arange(N // 2)` 只算前半，然后用这个 twiddle 同时处理 `top` 和 `bottom`——也就是说，**前 N/2 个频点用 k=0..N/2-1 的 twiddle，后 N/2 个频点的 twiddle 就是什么？是用同一个数组吗？** 代码框架没有明确这一步，"拼接返回"那行我猜测是 `np.concatenate([top, bottom])`，但为什么 bottom 用的还是同一套 twiddle？

### 【计算】白板问题 3 的手算细节
题目要求手算 `x = [1, 0, -1, 0]` 的 `X[1]`：
- E = DFT([1, -1]) = ?  我知道 DFT 的定义，但对一个长度为 2 的小信号手算 DFT，要分别算 `X[0]` 和 `X[1]`，这两个分量各是多少？
- O = DFT([0, 0]) = [0, 0]，这个显然。
- `W_4^1 = e^{-πi/2} = ?`  我知道 e^(ix) = cos(x) + i·sin(x)，代入 x = -π/2，得 cos(-π/2) + i·sin(-π/2) = 0 + i·(-1) = **-i**。这个对吗？
- 最后 `X[1] = E[1] + (-i)·O[1]`，代入后算什么？这个具体的数值没有演示。

### 【理解】递归基 N=1 为什么直接返回原值
讲义说"N==1 时 DFT 就是原值本身"，为什么？DFT 公式 `X[k] = Σ_n x[n]·exp(...)` 在 N=1, k=0 时退化成 `X[0] = x[0]·exp(0) = x[0]`，这我理解。但如果有人给出 N=1，k=任意，这是什么意思？频域也只有一个点 k=0，对不对？这个细节需要确认。

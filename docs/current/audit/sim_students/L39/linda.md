## 我理解了什么

**分治+递归**：把 N 点 DFT 分成偶/奇两个 N/2 点 DFT，然后用一个叫"蝶形"的公式 `E[k] ± twiddle·O[k]` 合并回来。递归一路钻到 N=1（基础情形就是返回原值），然后一层层往上合并。这样从 O(N²) 变成 O(N log N)，对 N=1024 快 200 倍。

**递归执行顺序**：调用栈可视化那一节特别清楚——程序深度优先地钻到 N=1，然后才开始往回收线。我理解了为什么必须有 `if N==1: return` 这一行，不然递归永远停不下来。

**代码框架**：5 个 TODO 项（基础情形、偶奇拆分、旋转因子、蝶形合并、拼接返回）的逻辑都懂了，就是 Python 代码怎么写。

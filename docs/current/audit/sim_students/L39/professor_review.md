### 问题 1：Cooley-Tukey 拆分公式从何而来（老陈、琳达、小雨共同困扰）
**学生问题**：课程直接给了公式，但没解释**为什么**偶奇拆分能这样合并。从 DFT 定义到 FFT 公式的推导过程缺失。

**改动内容与位置**：
- **新增 markdown cell**（在原"1. 递归结构"之前）："预备：从 DFT 定义推导 Cooley-Tukey 公式"
  - 从 DFT 定义 $X[k] = \sum x[n] e^{-2\pi i k n / N}$ 开始
  - 按 $n=2m$（偶数）和 $n=2m+1$（奇数）拆分求和
  - 展示指数如何重写成两个 N/2 点 DFT 的形式
  - 推导出 $X[k] = E[k] + e^{-2\pi i k/N} O[k]$ 和 $X[k+N/2] = E[k] - e^{-2\pi i k/N} O[k]$
  - 解释减号来自周期性 $e^{-2\pi i n/2}$，对奇数项自动给出 -1

### 问题 7：为什么必须是 2 的幂（小雨困惑）
**学生问题**：为什么不能是 3、5 或其他底数？补零到底是什么意思、补多少？"物理分辨率"是什么？

**改动内容与位置**：
- **修改 section 2 markdown cell**（"2. 必须是 2 的幂"）
  - 原文很简洁，补充新内容：
    - 解释这是"算法设计要求"而非编程限制
    - 为什么：递归每层必须**恰好对半** N→N/2→...→1
    - 如果 N=5，无法对半分下去（5→2.5✗）
  - 补充"非 2 幂怎么办"小节：
    - 补零公式 $N_{\text{padded}} = 2^{\lceil \log_2 L \rceil}$
    - 举例：5→8, 100→128
  - 新增"物理分辨率"解释：
    - 补零只改变**频率网格密度**，不改变**分辨能力**
    - 频率分辨率 = 采样率 / 原始时长（由原始信号决定）
    - 补零后网格分布更密，但两个接近频率的区分能力不变
- **修改 code cell 7fa120ea**（原"演示 N=5 补零"）
  - 升级为全面演示：多个补零示例 + 真实信号对比
  - 生成 440+880 Hz 混合信号（两个音符）
  - 示意补零不改变"物理分辨率"的图表
  - 打印对比说明："补零改变网格**密度**，不改变**物理分辨率**"

### 问题 8：递归的下标对应与拼接逻辑（小雨困惑）
**学生问题**：看公式 $X[k] = E[k] + W^k O[k]$ 和 $X[k+N/2] = E[k] - W^k O[k]$，不清楚：
- $E[k]$ 的 k 范围是什么（N/2 只有那么多元素）
- 最后怎么拼接成长度 N 的数组
- 为什么上半用 `+`，下半用 `-`（同一套 $W^k$）

**改动内容与位置**：
- 在"具体手工示例 N=4"中已详细展示：
  - `k=[0,1]`（范围 0..N/2-1）
  - `E=[0,2]` 有 2 个元素，`O=[0,0]` 有 2 个元素，都是长度 N/2
  - 对 k=0,1 分别算 top 和 bottom（各得到 2 个值）
  - 拼接：`[top[0], top[1], bottom[0], bottom[1]]` = `[0,2,0,2]`
  - 为什么上下用同一套 `twiddle`：这是公式设计，减号自动处理相位差

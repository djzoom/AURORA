# 小雨 — L36 学习日志

## 我理解了什么
- **窗函数的作用**：在截取一段音频时，两端突然从有声音跳到无声会产生"假的高频"。通过逐点乘以一个两端接近 0、中间接近 1 的权重数组，可以温柔地把边缘压低，减少这种虚假频率污染（旁瓣泄漏）。
- **三种常用窗对比**：矩形窗最简单但旁瓣最强，Hann 和 Hamming 的边界更平滑所以旁瓣弱，Blackman 最激进但代价是频率分辨率下降（主瓣更宽）。
- **形状规律**：Hann 和 Blackman 首尾都逼近 0，Hamming 的首尾约 0.08（保留一点能量），矩形窗首尾都是 1（完全不处理）。

## 遇到的困难和问题

### 【理解】Periodic 与 Symmetric 两种约定分不清
> 原文位置："Periodic 与 Symmetric Hann" 小节

为什么有两个 Hann 公式？分母是 N 还是 N-1，结果差别这么大。课程说"Aurora 用 periodic，NumPy 用 symmetric"，但初学者怎么记住用哪个？更关键的是，为什么 Aurora 选 periodic 而不跟 NumPy 一样？这个选择在 STFT 中有什么具体影响？表格对比了两个版本，但没解释 periodic 为什么叫"DFT-even"，也没说这对无缝拼帧有什么帮助。

### 【理解】频谱泄漏原理跳得太快
> 原文位置：引言中"FFT 不知道'这是截取边界'，会把这个跳变当成真实的高频成分" 以及附录中"矩形截断 = 乘以矩形函数 → FFT 得到 sinc 旁瓣"

"旁瓣（sidelobe）"长什么样？为什么矩形截断会产生高频？课程说会得到"sinc 旁瓣"，但 sinc 函数是什么没有介绍过，我只能相信结论。建议在这个关键点补一个直观的图或更详细的推导。

### 【实践】`np.argmax()` 函数突然冒出来
> 原文位置：第一个代码框 cell 10 "关键步骤"和 describe_window 的实现提示

代码提示中写 `int(np.argmax(w))`，但这课之前没有介绍过这个 NumPy 函数。虽然名字有点自解释（arg=argument，max=最大值，合起来=找最大值的位置），但对只会 print、变量、循环、列表的初学者，这还是陌生的。可以在代码上方简单说明一句："`argmax` 返回最大值所在的下标位置"。

### 📌 问题3：Dropout 和 BatchNorm 的作用及 train()/eval() 模式区别
**涉及学生**：老陈、琳达、小雨（三人都卡在这）

**修订内容**：
- **新增 markdown cell**：`## 3.5 model.train() 和 model.eval()：神经网络的两种工作模式`
  - **Dropout 部分**：
    - 问题：过拟合（过度依赖某些神经元）
    - 解决思路：随机丢弃（类比"随机换笔记工具"使人学会多条路线）
    - train() 模式：以概率 p 丢弃 → eval() 模式：全保留（自动缩放）
  
  - **BatchNorm 部分**：
    - 问题：内部协变量偏移（数据分布不稳定）
    - 解决思路：每层都对 batch 数据做归一化
    - train() 模式：用当前 batch 的 mean/std
    - eval() 模式：用全局 running mean/std（训练过程积累的）
  
  - **既用 eval() 又用 no_grad() 的原因**：
    - eval() 改变网络行为（Dropout/BN 工作方式）
    - no_grad() 改变计算模式（禁用自动求导）
    - 两者各司其职，缺一不可

- **新增 code cell**：`为什么必须 zero_grad()？反面教材`
  - 对比：没有 zero_grad()（梯度累积，权重跳跃）vs 正确做法（独立计算）
  - 演示梯度确实在累积（第1轮 g1，第2轮 g2+g1，第3轮 g3+g2+g1）
  - 直观展示正确做法的稳定性

### 📌 问题5：过拟合的定义和"泛化间隔"的概念
**涉及学生**：老陈、琳达、小雨（三人都需要故事说明）

**修订内容**：
- **新增 markdown cell**：`## 5.5 过拟合（Overfitting）与泛化间隔`
  - **什么是过拟合？** 用高考故事类比：
    - 学生 A：背往年真题（死记硬背）→ 真题 100 分，新题 60 分 → 过拟合
    - 学生 B：理解概念（深度学习）→ 真题 95 分，新题 85 分 → 泛化好
  
  - **如何诊断过拟合？** 需要两条曲线：
    - 训练损失曲线（Training Loss）
    - 验证准确率曲线（Validation Accuracy）
    - 过拟合症状：训练继续改进但验证停滞（间隔出现）
  
  - **为什么要用独立验证集？**
    - 若训练集 = 验证集 → 模型"记住"数据 → 训练/验证准确率都 100% 但无法泛化
    - 独立验证集迫使模型依赖真实学到的规律
  
  - **"泛化间隔"的定义**：
    - 泛化间隔 = 验证性能 - 训练性能
    - 间隔 < 0.1：优秀
    - 间隔 > 0.2：明显过拟合
  
  - **与 L65 的连接**：本课学会用独立验证集观察过拟合，L65 将学到用可视化和 early stopping 诊断与抑制

**位置**：L64-0531（目视验证 eval_accuracy）→ L64-0532（参数实验）之间

# 琳达 — L64 学习日志

## 我理解了什么

- 训练一个模型需要两个循环函数：`train_epoch`（在训练集上一轮一轮地喂数据、算损失、反向传播、更新参数）和 `eval_accuracy`（在验证集上测试模型准确率）。这两个函数就是完整训练闭环的骨架。
- 混淆矩阵就是一个表格，`C[i][j]` 表示真实是第 i 类、预测错成第 j 类的样本有多少个。看对角线（都是对的）和非对角线（各种错误类型）就能诊断模型在哪里搞混。
- 为了看出模型是否真的在学东西而不是巧合，必须用独立的验证集（而不是拿训练数据来测试自己），这样才能看出"训练集准确率"和"验证集准确率"的差距，从而发现过拟合。

## 遇到的困难和问题

### 【理解】CrossEntropyLoss 的初始值为什么是 log(num_classes)？
notebook 在 L64-0518 说"初始值 ≈ log(num_classes) = log(10) ≈ 2.3 ← 随机猜测的期望损失"，白板挑战试图解释"softmax输出≈均匀分布，每类概率≈0.1，-log(0.1)=2.303"。我能算出数字对得上（log(10)≈2.303），但**为什么损失函数要设计成 -log(正确类概率)？** 为什么不是别的形式？这背后的数学逻辑课程没有展开。

### 【理解】optimizer.zero_grad() 为什么必须每个 batch 都清一次？
L64-0526 的表格说"清零梯度，每 batch 必须执行"，但**没解释为什么。** PyTorch 梯度默认是累积的吗？如果我忘了清零会怎么样——损失会变成什么鬼样子？这是一个容易踩坑的地方，但课程只给了规则，没给反面教材。

### 【理解】Softmax 到底是什么？
公式 `softmax(logit)[j] = exp(logit[j]) / sum_k exp(logit[k])` 在课程中被默认理解，但从没单独定义过。我知道这是把 logit 转成概率（加起来等于 1），**但为什么要用指数函数 exp？** 为什么不直接用 `logit / sum(logit)` 这样的归一化？这背后有数学原理吗？

### 【理解】过拟合（overfitting）到底指什么？为什么需要独立验证集？
L64-0533 的注释说"若 val_loader 和 train_loader 使用同一份数据，模型永远无法展示泛化间隔，过拟合诊断也就无从谈起"。我理解**结论**（要用独立数据集），但**为什么会"永远无法展示泛化间隔"？** 什么是"泛化间隔"？过拟合的症状是验证准确率比训练准确率低，但课程没有举例说明这个现象是怎么出现的。

### 【理解】model.train() 和 model.eval() 的魔法：Dropout 和 BatchNorm 是什么？
L64-0526 表格说这两个函数"对 Dropout/BN 训练模式"有影响，白板挑战问 3 才稍微解释了一下（"Dropout: train时随机丢弃，eval时全部保留"），但**为什么要随机丢弃参数？** 这不会破坏模型吗？BatchNorm 里的"batch统计"和"running mean/var"是什么鬼？这些都是课程之外的黑盒。

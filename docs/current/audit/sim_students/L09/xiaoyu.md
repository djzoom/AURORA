# 小雨 — L09 学习日志

## 我理解了什么

向量就是一个等长的数字列表，NumPy 里用 array 存，形状是 `(n,)`。向量的两个基本操作很简单：加法是逐个数字相加（像混音），缩放是全部数字乘同一个倍数（像调音量）。任意二维向量都能用两个基向量加权叠加表示出来，这就是"线性组合"。

## 遇到的困难和问题

### 【理解】基向量和线性组合的必然性
课程说"任意 2D 向量可由基向量 e₁、e₂ 加权叠加表示"，并在代码里演示了 `a*e1 + b*e2` 能得到 `[a, b]` 这样的向量。我跟着例子明白了**怎样做**，但没看懂**为什么一定是这两个向量**。为什么选 `[1,0]` 和 `[0,1]` 作为基向量？能不能用其他向量作为基？这个逻辑跳跃需要补充说明。

### 【理解】矩阵与向量作用的原理
代码展示了矩阵 `A @ v` 会改变向量的值和长度，但没有解释这是**怎样**改变的、改变的**规则**是什么。题目提到"矩阵把向量拉伸"和"特征向量被拉伸但不旋转"，这些都很抽象。虽然说"将在 L12 中详细展开"，但现在看着有点悬空。

### 【推演】负数缩放的几何含义
课程提到 `c < 0` 时"方向反转"，但只用箭头图示展示了正缩放的情况。`scale([3,4], -1)` 会得到 `[-3,-4]`，这在二维坐标系上是什么样子？相当于转过 180 度吗？需要一张对比图。

### 【计算】符号 ‖v‖ 和 `np.linalg.norm()` 的对应关系
notebook 用 `||v||` 或 `‖v‖` 表示向量长度，代码里用 `np.linalg.norm(v)` 计算。"norm" 这个词对初学者不友好——能否在第一次出现时加个注解，说"norm 就是长度的意思，用函数 `np.linalg.norm()`"？这样比要去猜测或查词典省心。

### 【解答】浮点数精度和 `np.allclose()` 的必要性
测试代码用 `np.allclose(result, expected, atol=1e-10)` 检查答案，而不是直接 `result == expected`。我不理解为什么计算 `[0.3, -0.6, 0.9]` 这样的结果还会有误差。是不是计算机的浮点数存储问题？如果是，能给个简短说明吗（比如"浮点数四舍五入可能产生微小偏差，所以用 allclose 检查是否在误差范围内"）？

### 【实践】高维向量的直觉建立
MFCC 特征是 13 维向量，音频用 5 维向量 `[0.0, 0.5, 1.0, 0.5, 0.0]` 做例子。这些都超过三维了，怎样在脑子里想象它们的加法和缩放？二维时有"向右向上的箭头"这种直觉，高维就没有了。是否有非可视化的直觉帮助（比如"就是把每个位置的数字分别操作"），或者一定要靠代码验证？

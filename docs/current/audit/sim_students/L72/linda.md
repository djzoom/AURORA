# 琳达 — L72 学习日志

## 我理解了什么

1. **LoRA 的核心思想**：Whisper 有 244M 参数，但只需在 q_proj/v_proj 这两个矩阵上冻结原权重、额外加一条"低秩补丁" ΔW = B·A（两个瘦矩阵相乘），就能改动整个模型的行为。参数量从 d×k 降到 2×d×r，比如 768×768 的矩阵用 r=8 只需 24K 参数而非 590K。

2. **训练的步骤流程**：加载模型 → 用 LoraConfig 注入低秩分支 → 用 WhisperProcessor 预处理音频和文本 → 用 Seq2SeqTrainer 训练（只更新 A/B 矩阵）→ 用 WER 评估效果。

3. **WER 的含义**：把模型输出和真实文本都分词，算词级编辑距离除以参考词数，用替换(S) / 删除(D) / 插入(I) 三种错误类型来衡量。可以 > 100%（当插入太多词时）。

## 遇到的困难和问题

### 【参数/推演】lora_alpha 缩放因子的意义不明
LoraConfig 中设置 `lora_alpha=32`，代码注释说"缩放因子：ΔW 乘以 lora_alpha / r"，即 32/8 = 4。**但为什么要这样做？** 直接改学习率行不行？ΔW 被乘以 4 会不会破坏权重初始化平衡？课程没有解释这个参数为什么存在、为什么这样设置。

### 【理解】q_proj/v_proj 为什么比 k_proj 和其他层更关键
课程提示"query/value 决定'关注什么'，是任务适配的关键"，但没有深入解释。我想知道：k_proj(key) 不也决定了自注意力的计算吗？为什么只注入 q 和 v，不注入 k？在微调中它们的作用有什么本质区别吗？

### 【推演】为什么必须用两个矩阵 A 和 B，而不是一个低秩矩阵 C
讲解说"B=0 初始化，保证 ΔW=B·A=0 不影响预训练"，但一个单独的矩阵 C 也可以初始化为 0 啊。**用两个矩阵是否有数值稳定性、梯度流、或者其他我没看出来的优势？** 这个细节没有说透。

### 【实践】simple_wer 的编辑距离算法需要依赖前置知识
练习 2 要求实现 simple_wer，需要"用编辑距离"。课程说"与 L67/L68 的 edit_distance() 相同"，但这是另一课的函数，当前 notebook 没有给出具体的动态规划代码或伪码。**作为初学者，我需要编辑距离的具体实现思路或公式。**

### 【理解】WER > 1.0 在实践中代表什么问题
课程给了例子："good" vs "this is very very good" 的 WER = 4.0。这说明模型输出了 4 个多余的词。**这种情况意味着什么？是过拟合表现、还是数据集标注错误、还是训练参数选得太激进？** 这对我们判断模型好坏的实际含义不清楚。

# 琳达 — L46 学习日志

## 我理解了什么

Mel 尺度把人耳的对数感知（频率翻倍听起来"等距"）转成数学公式 `mel = 2595 * log10(1 + f/700)`。用这个公式在 Mel 域均匀取点、再反投影回 Hz，就自然形成了低频滤波器窄密、高频滤波器宽疏的分布——正好贴合耳蜗结构。三角滤波器组矩阵和功率谱的 `@` 乘法一步到位产出 n_mels 个 Mel 能量值。

## 遇到的困难和问题

### 【理解】Mel 公式的魔数从何而来
课程直接给出 `mel = 2595 * log10(1 + f/700)` 和验收数值（1000 Hz → 1000 mel），但没说明**为什么是 2595 和 700 这两个数字**。对高中生来说，这就像看"对数底数为 e"突然出现一样，缺少 motivation。要不要加点背景：这是 Stevens 1937 年拟合人耳实验数据得出的近似式，还是有其他物理来源？

### 【理解】整数 bin vs 分数 bin 的区别没讲
课程最后一段明确说"本课用的是**整数 bin 索引**版...生产版 Aurora 用**连续频率（分数 bin）**构造，思路相同但边缘 bin 会有细微差异"。但完全没解释**什么叫分数 bin**，为什么要分数，两版的输出差在哪。这让我有点摸不透"整数版是近似还是标准版"。

### 【推演】FFT bin 索引公式怎么推
代码写 `bin_idx = np.floor((n_fft + 1) * hz_points / sr).astype(int)`，课程只说"把 Hz 映射到 FFT bin 索引"，没给**推导过程**。比如：
- 为什么是 `(n_fft + 1)` 而不是 `n_fft`？
- FFT 的 bin 间距（Hz/bin）是多少？这个公式是从哪个基础公式变形来的？

我能跟着代码计算，但不知道**为什么这样选**。

### 【理解】三角滤波器形状为什么是这样定义的
课程说"第 m 个三角滤波器在 [left, center] 线性升至 1，在 [center, right] 线性降至 0"——这个**定义有标准吗**？还是说人耳听觉实验验证过三角形比矩形或其他形状更好？为什么是对称的（升降斜率不一样是可以的吗），为什么峰值固定为 1？

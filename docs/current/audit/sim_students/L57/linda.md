# 琳达 — L57 学习日志

## 我理解了什么

核心就是三个嵌套的 class：`Neuron` 做一个加权求和+激活、`Layer` 把 nout 个 Neuron 并排放一起接收同一个输入、`MLP` 把多个 Layer 首尾连起来。最后所有参数（w 和 b）都拍成一个平坦列表给优化器。`MLP(3,[4,4,1])` 总共 41 个参数，就是把每层的权重和偏置加起来：16+20+5=41。

## 遇到的困难和问题

### 【理解】最后一层为什么要 `nonlin=False`？

课程在第 57 课标题下说"输出落在 (-1, 1)，便于观察"，但这根本没说清楚啊。后面白板挑战问 3 的"答案格"里才解释"回归输出或 logit，不需要 relu 截断负值"——但课程正文里没有这句话。

**问题**：为什么回归任务一定要是线性的、不能过 tanh？如果我想让输出也在 (-1, 1) 范围内，直接 tanh 不行吗？课程应该明确说：回归输出需要**无限制的实数范围**，一旦过激活函数就被压缩了，所以最后一层必须线性。

### 【理解】Layer 返回值的类型不一致

Layer 的说明写着："若 len(outs)==1 返回 outs[0]，否则返回 outs"。

这意味着：
- `Layer(2, 1)([x1, x2])` 返回一个 `Value`
- `Layer(2, 3)([x1, x2])` 返回一个 `list[Value]`

代码里有注释说"单输出层应返回 Value 而非列表"，但没解释**为什么**要这样设计。我猜是为了方便后面 MLP 的串联？但这种隐式的类型转换对初学者来说很容易出 bug——有时候期望列表，有时候期望标量。

### 【理解】MLP 的"统一包装为列表"是什么意思？

Cell 55 的 MLP.__call__ 说明："统一返回列表（若结果不是 list 则 [x]）"。

但最后一层如果只有 1 个神经元（回归任务常见），它返回的是 `Value`（因为 Layer 的规则），所以 MLP 需要把它包装成 `[Value]`。这个逻辑对吗？代码里实际怎么实现？课程没有明确的代码示例。

### 【推演】参数数量公式的推导

表格里给了每层的参数数（16/20/5），也给了"权重数 nin × nout + 偏置数 nout"的公式。但课程没有推一遍为什么是这个公式。我能推出来：每个 Neuron 有 nin 个权重 + 1 个偏置 = nin+1，一层有 nout 个 Neuron，所以 nout×(nin+1)。但课程应该明确写出这个推导，不要让学生自己想。

### 【实践】前向传播的计算图有多大？

白板挑战问 5 说"每个 Neuron 约 nin+1 次加法+1 次激活，...合计约 45 个节点"。但怎么从"nin+1 次加法"推出 45 个节点？每次加法/乘法都是一个计算图节点吗？"约"是什么意思——能算精确值吗？对理解反向传播有什么用？课程应该明确说明计算图节点的定义，或者直接告诉我们不用关心这个细节。

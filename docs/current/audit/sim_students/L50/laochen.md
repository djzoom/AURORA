# 老陈 — L50 学习日志

## 我理解了什么

MFCC 就是一条流水线：**原始音频 → STFT 求频率 → Mel 滤波（模拟人耳） → 取对数（压缩） → DCT-II 去相关**，最后截取前 13 个系数。整个过程把 1 秒钟 16000 个采样点压缩成 (帧数, 13) 的矩阵，喂给后续的语音识别模型。实现就是四个函数的串联，每步都是矩阵操作，没有新的算法。

## 遇到的困难和问题

### 1【理解】mel_filterbank 的矩阵乘法方向  
流程图第 3 部分写"mel_filterbank @ |STFT|²"，但第 7a7385b6 的演示代码实际写的是"power @ fb.T"。这两个形式不一样。我理解矩阵 @ 需要维度匹配，但"为什么要转置 mel_filterbank"没有明确说。power 是 (n_frames, 513)，fb 是 (40, 513)，那 fb.T 变成 (513, 40)，power @ fb.T 才能得 (n_frames, 40)。问题是：为什么 mel_filterbank 的行列关系是这样？能不能给一句"mel 矩阵的形状约定是..."的说明？

### 2【理解】power_to_db 的 1e-10 下限保护  
第 4 部分说"power_to_db 内置 1e-10 下限保护"。我知道 log(0) 是无穷小，所以需要一个下限。但具体做法是什么？是 log(max(power, 1e-10)) 吗？还是直接 log(power + 1e-10)？这两种结果不同啊。另外"10·log10"是什么意思——log10 就是以 10 为底的对数，为什么再乘以 10？分贝（dB）是什么单位？

### 3【理解】dct_ii 对二维矩阵的操作方式  
演示代码 7a7385b6 中，log_mel 是 (n_frames, n_mels) 的矩阵，直接传给 dct_ii，输出仍是 (n_frames, n_mels)。课程说"对每帧做 DCT-II"，那应该是对每一行（每一帧）单独做 DCT-II，还是 dct_ii 函数内部自己处理了二维的情况？如果是前者，my_mfcc() 的实现中要不要加个循环？

### 4【理解】低阶 DCT 系数 vs 高阶系数的物理意义  
第 579cac48 部分说"低阶 DCT 系数对应声道形状（元音特征），高阶系数对应细节（声门激励、噪声）"。这个对应关系是怎么建立的？为什么 DCT 变换后，自动就能分离出这两种信息？这个解释太跳跃了，能补充一句说明吗？另外"声门激励"是什么？高中物理里没学过。

### 5【推演】n_frames = 1 + L // hop 的推导  
维度检查格说这是公式，但没说从哪来。前面 L44（STFT 课）应该讲过，但我这里需要回溯。为什么不是 ceil(L / hop)？"center=True"（两端反射填充）怎么影响这个公式？能在这里给一句快速提示吗？

### 6【理解】对数压缩的物理直觉  
课程说"幅度相差 100 倍 → dB 只差 20"。数学上 log10(100) = 2，×10 = 20，我能算。但"为什么压缩到 dB 后，语音识别模型效果就更好了"——这个因果关系没有建立。是因为人耳对响度感知本身就是对数的吗？还是仅仅是为了让大幅度信号和小幅度信号的重要性更平衡？

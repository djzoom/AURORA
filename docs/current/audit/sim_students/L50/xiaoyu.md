# 小雨 — L50 学习日志

## 我理解了什么

MFCC 是把一帧声音（480个数字）压缩成13个数字的五步流水线：STFT 拿出频率成分 → Mel 滤波器按人耳方式重新组织 → 取对数压缩大的数值 → DCT 变换让这13个数字各自代表不同粗细度的声道形状 → 取前13维作为特征输入语音模型。每一步都是矩阵乘法或逐元素操作，没有学习参数。

## 遇到的困难和问题

### 【理解】Mel 频率为什么能"模拟人耳"  
> 原文："人耳感知频率"，"模拟人耳"  

课程说 Mel 滤波器组能模拟人耳，但**没有解释人耳的生理原理**。人耳真的是按"Mel 频率"来感知声音的吗？还是说只是实验发现两者相关？比如为什么不用其他的频率变换？不明白 Mel 相比线性频率的优势在哪。

### 【推演】DCT-II 公式中的 w[k] 为什么要分 k=0 和 k>0
> 原文："`w[0] = sqrt(1/N)`，`w[k>0] = sqrt(2/N)`"  

为什么 k=0 系数和其他系数的权重不同？为什么一个是 $\sqrt{1/N}$，其他的是 $\sqrt{2/N}$？这看起来是"正交归一化"的规则，但**没有直观解释为什么要这样设计**。如果我要推导一个新的变换，怎么判断权重应该设成多少？

### 【推演】矩阵乘法方向：为什么是 `power @ fb.T` 
> 原文："mel_filterbank @ |STFT|²"，代码：`mel = power @ fb.T`  

图里写的是数学描述，代码写的是 numpy 实现。
- power 是 `(n_frames, 513)`，fb 是 `(40, 513)`
- 代码用 `power @ fb.T`，即 `(n_frames, 513) @ (513, 40)` 得到 `(n_frames, 40)`
- 但数学公式"mel_filterbank @ |STFT|²"听起来是"(40, 513) @ (513, ?)"

**这两种写法是等价的吗？** 如果 power 和 STFT 的关系是 power = |STFT|²（逐元素平方），那从维度来看：
- 数学写法 `(40, 513) @ (513, ?)` 需要右边是列向量或矩阵，每一列对应一帧
- 所以应该是 `(40, 513) @ (513, n_frames)` 还是 `(40, 513) @ (513, n_frames).T`？

感觉这里**行列顺序**容易搞混，希望课程画个"这是 40 个滤波器，每个作用在 513 个频率上，一个数据点"的图来澄清。

### 【理解】对数压缩（power_to_db）为什么用 10·log10 而不是自然对数
> 原文："log 压缩"，"动态范围（幅度相差 100 倍 → dB 只差 20）"  

课程举例说幅度相差 100 倍时，dB 只差 20，这说明对数变换压缩了范围。但为什么是 **10·log10** 而不是 **ln**（自然对数）？
- 如果用 log10：100 倍 → 10·log10(100) = 10·2 = 20 ✓（符合例子）
- 如果用 ln：100 倍 → ln(100) ≈ 4.6（数值不同）

这说明必须用 **log10**，但**为什么这个因子 10 和底数 10 被选中？** 这是分贝的定义吗？还是有信号处理的物理原因？课程没有解释。

### 【计算】n_frames 公式 `1 + L // hop` 为什么要加 1
> 原文："n_frames = 1 + L // hop"  

给定 L=16000, hop=256：
- n_frames = 1 + 16000 // 256 = 1 + 62 = 63

**为什么多加 1？** 我猜可能和 STFT 的填充机制有关（课程提到"center=True"，两端反射填充），但**具体推导没看到**。如果没有这个 +1 会怎样？前面几课（L44 STFT）有讲过吗？

## 遇到的困难和问题

### 【理解】`np.arange(1, n+1)` 为什么不能写成 `np.arange(n)`？

在"手写LLN收敛曲线"那块代码：
```python
running = np.cumsum(flips) / np.arange(1, n + 1)
```
题目解释说"除以当前次数即得运行平均值"，但为什么是 `np.arange(1, n+1)` 而不是 `np.arange(n)`？

我试过心算：如果 `flips = [1, 0, 1]`，那么 `cumsum = [1, 1, 2]`。
- 用 `np.arange(1, 4)` 得 [1, 1, 2] / [1, 2, 3] = [1.0, 0.5, 0.67] ✓
- 用 `np.arange(3)` 得 [1, 1, 2] / [0, 1, 2]，第一个数是 1/0 爆了 ✗

所以必须从1开始，但题目没解释这是为什么——这对Python初学的人其实不显然。

### 【推演】Shannon熵的0 log 0定义为什么是0？

练习要求实现 `shannon_entropy(p)`，公式是 H(p) = −Σ pᵢ log₂ pᵢ。题目说"用 `np.where` 或布尔掩码跳过 pᵢ = 0 的项（0 log 0 定义为 0）"。

**我的疑惑**：为什么数学里定义 0 log 0 = 0？这不是通常的数学运算。我理解 0 log 0 是一个"不定式"（像 0/0 一样），但为什么在这里取极限是0？

我试过极限：当 p → 0⁺ 时，p log₂p → 0 吗？直觉上 log₂p → −∞，但 p 衰减得更快？没推过，这里有个逻辑空洞。

### 【理解】softmax防溢出：减去最大值为什么能救命？

代码是：
```python
e = np.exp(np.asarray(z, float) - np.max(z))
```
评论说"避免 `np.exp(logit)` 在大数值下溢出"。

**但为什么减 `np.max(z)` 就能避免？** 假如 z = [100, 101, 102]：
- 直接算：exp(100), exp(101), exp(102) 都炸了（超过浮点数上限）
- 减 max 后：z' = [-2, -1, 0]，exp(-2), exp(-1), exp(0) 都很安全

我看到了数值上的好处，但这个技巧的数学原理是什么？是不是跟对数性质有关？（我高中时学过 log(a/b) = log a - log b，但这里是 exp，反向的关系？）

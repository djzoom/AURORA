# 小雨 — L31 学习日志

## 我理解了什么

- **大数定律**：通过 `np.cumsum(flips) / np.arange(1, n+1)` 一行代码就能看出频率如何从乱跳逐步稳定，最后贴近理论概率 0.5，样本越多波动越小。
- **分布的形状**：正态分布是个钟形，标准差越大钟越平铺；用 z-score 把不同量级的数据都变成 −3~3 的范围（68-95-99.7 法则），这样模型梯度才不会忽大忽小。
- **Softmax + 交叉熵**：logit 分数先过 softmax 压成 0~1 的合法概率，再取 −log 计算损失——置信度越高（概率越接近 1）损失越小，这样模型就能学着变得更自信且正确。

## 遇到的困难和问题

### 【理解】标准差（σ）为什么是衡量"宽窄"的量 
引用："用 `(x - μ) / σ` 把不同尺度的特征统一量纲"。看到公式和对比图，但没理解**为什么要除以 σ 而不是别的**。σ 在几何上代表什么？为什么这样变换后所有分布都变成"标准"的一样形状？

### 【推演】Shannon 熵公式 H(p) = −Σ pᵢ log₂ pᵢ 的由来  
课程给了公式，做练习也能调试通过，但这个公式**为什么是这个形式**？为什么要用负号？为什么用 log₂ 而不是 ln？这不是数据拟合出来的——肯定有数学上的深层逻辑，但课程没有给出推导或直觉解释。

### 【理解】为什么"0 log 0 定义为 0"  
练习代码里说要跳过 pᵢ = 0 的项，令其贡献为 0，这就对。但**为什么这样定义是对的**？是纯数学约定还是因为 lim_{p→0⁺} p log p = 0 这个极限？前者的话，有没有其他定义方式？后者的话，这个极限怎么推？

### 【计算】梯形数值积分 `np.trapezoid` 的原理和精度  
代码用梯形法算 CDF，验证了"68-95-99.7 规则"成立。但：
- 梯形法**怎样**把曲线下的面积分割成梯形来计算？
- 用多少个点（这里代码里 `n=4000`）就"足够精准"了？有没有误差估计的方法？
- 要是被积函数在某处不光滑，梯形法还能用吗？

### 【理解】温度参数（Temperature）为什么除以 T  
代码示例 `softmax(logits / T)` 展示了 T 的效果：T < 1 时分布集中（自信），T > 1 时平滑（随机）。但**为什么这样缩放就有这个效果**？是因为分子 exp 对输入的敏感度变化吗？这个参数的数学基础是什么（比如是从某个分布的"温度"概念借来的吗）？

### 【理解】log 空间为什么能避免数值溢出  
softmax 实现里用 `e = np.exp(z - np.max(z))` 来稳定计算。直觉上"减去最大值能让数字小一些"，但：
- 这样**真的不会改变 softmax 的结果**吗？为什么数学上成立？
- 为什么不是减去最小值或者除以某个常数？为什么**必须**是减去 max？

### 【理解】为什么 68-95-99.7 是这三个数字  
验证代码显示 ±1σ → 68%、±2σ → 95%、±3σ → 99.7%，这是正态分布的"经验法则"。但这三个数字怎么来的？是不是可以从正态 PDF 的公式 $\frac{1}{\sigma\sqrt{2\pi}} e^{-x^2/2\sigma^2}$ 直接推出来？或者需要查表？

### 【理解】最小化交叉熵为什么等价于最小化 KL 散度  
最后提到 $H(p, q) = H(p) + KL(p||q)$，其中 H(p) 是**真实标签的**自熵（固定的），所以最小化 H(p,q) 就是最小化 KL 散度。逻辑清楚，但：
- KL 散度 $KL(p||q)$ 的定义和几何含义是什么？
- 为什么说 KL 散度是"概率分布之间的距离"（虽然它不满足距离的对称性）？

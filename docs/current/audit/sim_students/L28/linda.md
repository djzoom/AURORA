# 琳达 — L28 学习日志

## 我理解了什么

- **均值、方差、标准差**都是用一个数来描述一组数据的"集中趋势"和"散程度"，公式背后的逻辑是 take sum / divide by count（高中就学过）
- **z-score 标准化** `(x - μ) / σ` 就是把数据"挪到以 0 为中心、1 为半径"的参考系，这样不同量纲的特征就可以直接比较
- **NumPy 的 axis 参数** 控制求和/求均值是沿哪个维度压缩，`keepdims=True` 是为了让广播对齐，代码我看得懂

## 遇到的困难和问题

### 【理解】为什么方差定义是平方而不是绝对值？ 
课程在第 1 节说"距离越大，分布越散"就用 `(x_i - μ)²` 定义方差。但为什么一定要平方？用绝对值 `|x_i - μ|` 不也能衡量散程度吗？这两个有什么本质区别吗，还是只是数学家的习惯写法？

### 【理解】"Bessel 校正"（n-1）这个修正是什么意思？
第 1 节代码注解提到 `x.std(ddof=1)` 是"样本标准差"的 n-1 修正。课程没解释为什么样本要除以 n-1 而不是 n。这和"样本偏差"有关吗？什么时候用 ddof=0，什么时候用 ddof=1？我猜可能要看数据量或应用场景，但没有原理解释。

### 【理解】标准化后为什么均值**一定**是 0？
我理解 `(x - μ) / σ` 这个公式，但为什么一减去均值，结果的均值就自动变成 0 了？是数学规律还是巧合？感觉需要一个简短的证明或直观说明。

### 【理解】axis=1 标准化为什么比 axis=0 更合理？
第 3 部分代码展示了两种标准化方向。课程说 axis=1 是"消除各频段的绝对能量差异，保留频段间的相对关系"，但我不懂为什么这对音频特征更有用。换句话说，**为什么我们不想要"各频段的绝对能量差异"**？是因为不同麦克风录同一个音频时绝对能量差很大，所以要抹掉？还是因为模型不关心绝对响度只关心音色形状？

### 【理解】梯度爆炸/消失与标准化的关系
Aurora 连接段说"跳过这步梯度容易爆炸或消失"。我不懂深度学习的梯度概念，但这个说法暗示特征不标准化会导致某些数值问题。能不能举个具体例子：为什么特征方差大会让梯度爆炸？是数值太大会溢出，还是有其他原因？

# 小雨 — L33 学习日志

## 我理解了什么

1. 正弦波可以用公式生成：在第 n 个采样点，计算它在圆上转过的角度（弧度），再取 sin 值
2. 实现方法：先算总点数N，生成点序列，用 2π·f·n/fₛ 算每点的角度，取sin后乘以振幅
3. 一个周期内要有足够的采样点（比如1Hz、8Hz采样率时，每秒8个点正好1个周期）

## 遇到的困难和问题

### 【理解】为什么时间 t = n / sample_rate？

课程在 cell-l33-0303 里直接写 `t = n / sample_rate`，但没解释这是怎么来的。我知道采样率是"每秒多少个点"，但从这里跳到"n除以采样率就是秒数"有个思维跳跃。能不能补充一句：比如 `sample_rate=8` 表示间隔是 1/8 秒，`n=2` 就是第2个点，时间是 `2 × (1/8) = 2/8` 秒？

### 【理解】公式中的 2π 是为什么？

公式 `angle = 2π·f·t` 里为什么一定要乘 2π？课程说"圆圈已经转了这么多弧度"，但我不太理解：
- 频率 f=1Hz 表示1秒内重复1次？
- 那为什么用弧度不用"周期数"？
- 为什么1个完整周期是 2π 弧度而不是其他数字？

这个需要更多的视觉解释或比喻（比如说"π是半圆，2π是整个圆"之类的）。

### 【计算】sin(π/2)、sin(π) 等特殊值

手算示例里需要我知道 `sin(0)=0, sin(π/2)=1, sin(π)=0, sin(3π/2)=-1` 这些。我在高中学过三角函数，但这会儿有点模糊了。这不是课程的问题，但建议可以加个参考表或一个复习 cell？

### 【理解】int() 和 round() 的区别

边缘测试（edge-case-test-l33）里说"确保用 `int(duration * sample_rate)` 而非 `round()`"。当 `duration=0.5, sr=3` 时：
- `int(0.5*3) = int(1.5) = 1`
- `round(1.5) = 2`

但课程没解释为什么一定要用 int() 截断，而不是四舍五入？这对实现正确性很重要，但逻辑没有交代清楚。是因为 `aurora.audio.sine` 的实现本身就用了 int()？还是有更深的原因（比如时间对齐）？

### 【理解】Nyquist 定理：为什么"完整一圈需要至少2个点"？

参数实验（cell-l33-0308）提到："16 = sr/2 = Nyquist 频率"时，"每个完整周期只剩两个采样点，正好够表示，但已是极限"。然后说"当 freq > sample_rate/2 时，采样点不够，高频成分会被折叠成低频噪声"。

这我完全没理解：
- 为什么2个点"正好够表示"一个周期？需要哪2个点？
- "被折叠成低频噪声"是什么意思？具体会变成什么？
- Nyquist 是一个人名还是规律的名字？

这个是后续 L34 的内容，但现在的提示太简洁了，初学者跳不过去。

### 【理解】振幅 amplitude 是否影响"形状"？

课程说"`amplitude` 只改变数值范围，不影响波形形状"。我理解这在数学上对，但在"看起来的样子"上：如果我画两个正弦波，一个振幅1，一个振幅0.5，它们的图形看起来是不一样的（一个上下幅度大，一个小）。"不影响形状"是什么意思呢——是说上下摆动的快慢不变？还是说比例关系不变？需要更精确的说法。

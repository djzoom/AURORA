# 老陈 — L33 学习日志

## 我理解了什么

1. **正弦波的核心公式** `x[n] = A·sin(2πf·n/sr)`：在第 n 个采样点，整个圆已经转过的角度是 `2πf·n/sr` 弧度，sin 取这个角的投影，再乘以振幅 A。
2. **三个关键参数的作用**：频率 f 控制波动快慢，采样率 sr 决定时间分辨率（每秒几个点），振幅 A 只改变波的大小不改变形状。
3. **采样点总数** N = int(duration × sr)：用 int 截断而非 round，所以 duration=0.5s、sr=3 时是 1 个点，不是 2 个点。

## 遇到的困难和问题

### 【理解】"完整周期"到底是几个采样点？

在"手算一个采样点"里，课程说 freq=1Hz、sr=8Hz 时"8 个点恰好完成一个完整周期"。但我数一下：
- n=0: angle=0, sin(0)=0
- n=2: angle=π/2, sin(π/2)=1
- n=4: angle=π, sin(π)=0  
- n=6: angle=3π/2, sin(3π/2)=-1
- n=7: angle=7π/4, sin(7π/4)≈-0.707

那 n=7 这一点呢？如果 N=8，n 只到 7，根本没有 n=8 这一点。所以"一个周期"是 0~7 这 8 个点，还是说第 8 个点虽然生成了但超出数组范围？这个定义对我来说有点模糊。

> **希望补充**：明确说"一个周期的采样数 = sr/f，这 k 个点的最后一点往往不会回到恰好 sin=0 的位置"，而不是暗示 n=N-1 就是周期的终点。

### 【推演】为什么必须用 int() 而不是 round()？

边缘测试提到 duration=0.5、sr=3 时，int(1.5)=1 而 round(1.5)=2。课程说这样才和仓库一致，但没有解释**为什么仓库选择 int 而不是 round**。

从数学角度，当采样时长不是采样间隔的整数倍时：
- int() 给出"能完整取出的采样点数"
- round() 给出"最接近的采样点数"

这两个哲学不同，但课程没有说清楚选择 int 的原因——是为了保证"严格覆盖 [0, duration) 这个时间段"吗？还是历史习惯？

> **希望补充**：一句话说明 int vs round 的设计选择。

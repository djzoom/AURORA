### 问题 4：为什么用 int() 而不是 round()？【老陈、琳达、小雨】

**学生卡点**：边缘测试要求用 `int()` 但没有说明理由，两种方式的差异和设计哲学不清楚。

**改动内容**：
- **新增 cell**："为什么用 int() 而不是 round()？"
- **位置**：在对答案的 markdown 说明之后、验证代码之前
- **内容**：
  - 具体例子：`duration=0.5, sr=3` → `int(1.5)=1` vs `round(1.5)=2`

  - 音频框架视角："严格覆盖指定时间段，不超出"
- **效果**：让学生理解这是设计决策，不是随意选择。

### 问题 5：完整周期的精确定义是什么？【老陈】

**学生卡点**：不清楚"一个周期的采样数 = sr/f"这个公式的含义，以及最后一个点为什么不会回到 sin=0。

**改动内容**：
- **新增 cell**："关键概念：完整周期与采样点数"
- **位置**：在代码实验（cell-l33-0305）之后、手算示例（ef820af3）之前
- **内容**：
  - 定义：周期点数 = sr/f（可能不是整数）
  - 具体例子：f=1Hz/sr=8Hz → 8个点；f=440Hz/sr=16000Hz → ~36.36个点

  - 强调："这不是 bug，这是特性"
- **风格**：直面学生最容易混淆的点，用"起点和终点"的对比说明。

### 问题 6：Nyquist 频率为什么是 sr/2？最少需要多少点？【老陈、琳达、小雨】

**学生卡点**：理解 Nyquist 定理的直观含义。课程说"每个完整周期只剩两个采样点"，但不清楚为什么 2 个点够了、1 个点不够，以及"被折叠成低频噪声"的含义。

**改动内容**：
- **新增 cell**："Nyquist 定理的直观理解"
- **位置**：在参数实验（cell-l33-0308）之前
- **内容**：
  - 问题启蒙："怎样从离散点序列判断频率？"
  - 极限分析：最少需要"上升部分的点"和"下降部分的点"来识别波
  - 坏例子：`[1, 0, -1, 0, ...]` 中间的峰值被完全错过
  - 数学表述：周期点数 ≥ 2 → f ≤ sr/2
  - 超界后果：f > sr/2 时，高频被"折叠"成低频（简短预告 L34）
- **效果**：从感官到数学，逐步建立 Nyquist 的直觉。

### 问题 8：int() 截断时的误差积累会不会成问题？【老陈】

**学生卡点**：最后一个采样点的时间会"漂移"（不到 duration），长期应用中这会不会累积成问题。

**改动内容**：
- **新增 cell**："采样点数目与时间对齐：为什么 int() 的'漂移'不是问题"
- **位置**：在"小检查"（cell-l33-0311）之后
- **内容**：
  - 具体数值：sr=16000, duration=1.0 → 漂移 ≈ 62.5 微秒
  - 3 个理由：
    1. 采样本身就是离散化，无法完全对齐
    2. 精度远超需要（人耳只能感知毫秒级）
    3. 实际应用中有重叠窗口，会消除边界效应
  - 什么时候才需要关心：超高精度对齐、长时间累积（超出入门课范围）
  - 结论："放心地用 int() 吧"
- **风格**：正面回答"这合理吗？"，而不是忽略这个疑问。

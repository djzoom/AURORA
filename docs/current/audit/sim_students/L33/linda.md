# 琳达 — L33 学习日志

## 我理解了什么

正弦波可以用公式 `x[n] = A·sin(2π·f·n/sr)` 生成。`n` 是采样点编号，`f` 是频率（决定波快慢），`sr` 是采样率（每秒取多少点），`A` 是振幅（决定上下摆动幅度）。代码很直接：先生成点编号 `n`，算出每个点的"角度" `2π·f·n/sr`，再取正弦值。

## 遇到的困难和问题

### 问题1：为什么角度是 `2π·f·t` 而不是别的？【理解】
课程说"圆圈已经转了 2πf·n/fₛ 弧度"，但没解释 2π 是从哪来的。高中圆周里知道一圈是 2π，但为什么"时间"能直接换成"圆圈的弧度"？`2π·f·t` 这个组合的物理意义是什么——为什么不是 `f·t` 或 `π·f·t`？能补充一句"1秒内 f 个完整周期"→"1秒转 f 圈"→"1秒转 2πf 弧度"这样的链条吗？

### 问题2：为什么是 `sin` 而不是 `cos`？【理解】
课程没说。我知道 sin 和 cos 差一个相位，但在这里为什么选 sin？是约定俗成还是有别的原因？如果选成 `cos` 会怎样？

### 问题3：Nyquist 频率为什么是 sr/2？【理解】
课程说"每个完整周期只剩两个采样点"就是极限，但为什么两个点够了、一个点不够？一个周期需要最少多少个采样点才能还原信号？这个"极限"有没有数学推导？

### 问题4：`int()` 截断 vs `round()` 为什么选 int？【理解】
边缘测试提示要用 `int()` 而不是 `round()`，说"与仓库一致"。但如果 `duration=0.5, sr=3`，`int(1.5)=1` 而 `round(1.5)=2`，这会影响波形长度。课程没解释这两种方式的区别会导致什么音频后果。为什么仓库选择 int 而不是 round 更"对"？

### 问题5：amplitude 参数只是乘法就行吗？【实践】
代码是 `amplitude * np.sin(angle)`，只是简单缩放。但"响度"和"振幅数值"是一样的概念吗？还是说音频里响度=幅度？如果是这样，那为什么不直接在生成时指定响度而要额外乘一个参数？

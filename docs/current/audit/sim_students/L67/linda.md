# 琳达 — L67 学习日志

## 我理解了什么

编辑距离用动态规划解决：构建一张表，每个格子 `dp[i][j]` 记录"前i个字符变成前j个字符需要几步"。字符相同不操作、不相同就选"删除/插入/替换"三个中最便宜的。这个逻辑串起来后，把字符换成单词就是 WER（词错误率）= 词级编辑距离 / 参考词数，复用同一个 DP 函数。

## 遇到的困难和问题

### 【推演】DP 递推公式中，三个邻居与三种操作的对应关系没解释清楚

在"填 dp[2][2]"的例子里，课程列出：
- `dp[i-1][j-1]+1` → 替换
- `dp[i-1][j]+1` → 删除  
- `dp[i][j-1]+1` → 插入

但没有说**为什么**这样对应。我现在是死记硬背，但看代码时还是容易搞混。能不能加一个"状态转移箭头图"或者简短的逻辑说明？比如"取 dp[i-1][j-1] 意味着处理完前 i-1 个 a 和前 j-1 个 b，现在用一次替换处理 a[i-1] 和 b[j-1]"之类的。

### 【理解】初始化为什么一定要 (m+1) × (n+1)，而不是直接 m × n

代码里用 `for i in range(m+1)` 和 `for j in range(n+1)` 初始化，然后 `dp[i][0] = i`、`dp[0][j] = j`。这意味着第 0 行和第 0 列是特殊的（代表空串吗？）。课程在表格里画出来了，但没有文字明确说"dp[0][j] 代表空串变成 b 的前 j 个字符"。如果直接用 m×n 大小但处理边界条件会怎样？为什么非得多加一行一列？

### 【理解】WER 什么情况下会 > 1.0，想要一个具体数值例子

课程说"WER 可以 > 1.0"，验收标准里的例子都是 0 到 0.5 之间。能不能给一个明确的例子，比如"reference='hello'（1 个词），hypothesis='I am very sorry for this mistake'（7 个词），结果是 7/1=7.0"？这样我才能真的体感到这个数字。

### 【实践】在 `alignment()` 函数回溯时，如果多个邻居的成本相同怎么判定操作类型

代码按顺序检查 `if ... elif ... elif ... else`，最后一个 `else` 捕获"删除"。但如果 `dp[i-1][j]`、`dp[i][j-1]`、`dp[i-1][j-1]` 的值都相同（都导致 `dp[i][j]` 相等），那就有三种不同的最优路径。代码会总是选择第一个符合条件的（替换优先于删除优先于插入），这样回溯出来的路径就是"最优路径的一种"而不是"唯一的"，是这样吗？这会不会影响 WER 的正确性？

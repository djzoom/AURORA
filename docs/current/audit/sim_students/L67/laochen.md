# 老陈 — L67 学习日志

## 我理解了什么

编辑距离就是把一个字符串改成另一个，最少用替换、插入、删除这三种操作。核心是一个 DP 表，填表时：字符相同不操作，不同就从左上/上/左三个邻居选最小的加1。后面直接复用这个逻辑到词级的 WER 计算，不用重新写 DP。

## 遇到的困难和问题

### 【理解】删除和插入为什么对应这两个方向？

题目说"从上面的邻居 dp[i-1][j] 加1 是删除"、"从左边的邻居 dp[i][j-1] 加1 是插入"。我理解替换——左上角 dp[i-1][j-1] 代表已经处理完前 i-1 和 j-1 个字符，现在把 a[i-1] 改成 b[j-1]。但**为什么删除 a 里的一个字符对应"往上走"（i-1, j），而插入 b 里的一个字符对应"往左走"（i, j-1）**？这两个操作的表格方向和含义的对应关系我脑子里还没想通——能补充一个图解说明吗？

### 【推演】字符相同时直接抄左上角，这一定是最优的吗？

题目说"a[i-1] == b[j-1] 就 dp[i][j] = dp[i-1][j-1]"。我知道这两个字符一样就不用操作，但我想确认：**有没有可能另一种改法（比如先删后插）也能达到同样的最少步数？还是"不操作"严格优于所有其他方案？**

### 【理解】WER 为什么可能大于 1.0？

题目提到"WER 可以 > 1.0"。从数学上讲，如果识别出的词全错了（比如参考 3 个词，识别出 5 个错词），编辑距离会很大，除以 3 确实可能超过 1。但这是**设计上故意允许的边界情况，还是说在真实 ASR 系统里这种极端情况其实很罕见？** 感觉通常评估一个系统时，错误率超过 100% 比较少见。

### 【实践】回溯时多个邻居同时给出最小值怎么办？

alignment() 函数回溯 DP 表时，先 check "a[i-1]==b[j-1]"，再 check "替换"，再 check "插入"，最后删除。但如果替换和删除都给出同样的最小值（比如都是 `1 + dp[...] = 2`），代码会优先选替换。**这个选择顺序有特殊含义吗？** 比如是为了输出"更可读"的对齐结果，还是标准 Levenshtein 距离就规定了优先级？

### 【实践】编辑距离函数为什么能同时接受字符串和列表？

题目说 `edit_distance()` 接受"任何可索引序列"。Python 里字符串和列表确实都支持索引和 len()，所以代码不需要改就能跑。**但这是一个"碰巧"，还是设计时就刻意选择了依赖最小接口（只用 len/索引）的写法？** 如果要给不同数据类型（比如元组、numpy 数组），需要检查什么条件吗？

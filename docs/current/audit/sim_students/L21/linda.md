# 琳达 — L21 学习日志

## 我理解了什么
1. **DFT 和 mel 都是矩阵乘法**：你说 FFT 就是 W @ x（矩阵×向量），mel 特征就是 M @ p（矩阵×向量），我实际跑了代码验证，确实 W @ x 和 np.fft.fft(x) 的数值完全一样，差值只有 1e-14（浮点误差）。这很震撼，从前觉得 FFT 是黑盒，现在看清楚了就是 N×N 矩阵乘法。
2. **Mel 矩阵的每一行是一个"频带检测器"**：每行是个三角形，中间峰值、两边衰减，M @ p 就是把功率谱"翻译"成人耳敏感的频带能量，每个输出是一个频段的能量总和。
3. **参数调整直接改矩阵形状**：改 n_mels 就改矩阵行数，从 (40, 33) → (80, 33) → (128, 33)，输出维度跟着增长，这点很直观。

### 【计算】Mel 矩阵构造代码的逻辑
你在 cell-8 的代码：
```python
M[m] = np.clip(np.where(bins<=cen, (bins-lo)/(cen-lo+1e-9),
                         (hi-bins)/(hi-cen+1e-9)), 0, 1)
```
我是一行一行读的：
- `np.where(bins<=cen, ...)` 是"如果 bin ≤ 中心，算左斜率；否则算右斜率"
- 左斜率 `(bins-lo)/(cen-lo)` 是"从低端到峰值的线性上升"
- 右斜率 `(hi-bins)/(hi-cen)` 是"从峰值到高端的线性下降"
- 加上 `np.clip(..., 0, 1)` 把负值和 >1 的值裁剪掉

**问题**：为什么要加 `+1e-9` 在分母？这是防止除以 0 吗？但我跑的例子里 centers 是 linspace 生成的，不该有 0 吧...？

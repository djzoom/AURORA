# 琳达 — L32 学习日志

## 我理解了什么

这节课教会我一个关键的技巧：**怎样给一段录音的数字配上"时刻"标签**。核心就是先用 `np.arange()` 生成点编号 `[0,1,2,...]`，再除以采样率转换成秒数。换句话说，第 k 个采样点发生在第 `k / sample_rate` 秒。这样后面做频域分析时就能知道每个点对应的具体时刻了。

## 遇到的困难和问题

### 问题 1：为什么 t[-1] 总是比 duration 少一点？【理解】

课程说"N 个点的编号是 0 到 N-1，最后一点的时刻不是 duration 本身"，我能算出来 `t[-1] = (N-1) / sample_rate`，但逻辑上还是有点奇怪。

**卡在**："为什么我们认为第 44099 号采样点是 44100 个点中的最后一个，而不是应该有第 44100 个点？" 换句话说，为什么采样点总数是 `N = round(duration × sr)` 这么多个，而不是多一个？

**希望补充**：能不能举例解释：1秒的音频、16000 Hz采样率，"为什么正好是 16000 个点而不是 16001 个"？从麦克风录音的角度讲讲采样怎样工作的。

### 问题 2：为什么一定要用 `np.arange` 除以采样率，而不能用 `np.linspace`？【理解】

课程说"arange 生成严格整数序号，不存在浮点累积误差"，还提到"FFT 频率对齐时这个差异会被放大"。

**卡在**：我还没学过 FFT 是什么，也不理解"浮点累积误差"对它有什么影响。这个解释依赖我还没学的知识。

**希望补充**：能不能先不提 FFT，用一个更直观的例子展示"linspace 和 arange 的区别是什么、什么时候这个区别会导致问题"？或者至少在前面某课提到 FFT 时再回过头来补充这一点。

### 4. **round() 函数为什么必要？什么时候 duration × sample_rate 不是整数？**
   - **提出者**：琳达（问题3）、小雨（问题4）
   - **问题本质**：代码有 round，但没说"不 round 会怎样"，学生搞不清是优化还是必须
   - **修订**：在第 3 部分（ID: 8d3f7ea8）增加**《round() 为什么必要？浮点数乘法的陷阱》**小节
     - 具体代码例子：`duration = 0.1, sample_rate = 22050` → 乘积可能是 `2204.9999999999998`
     - 演示 `np.arange()` 不接受浮点数，只接受整数
     - 说明 `round()` 是防守性编程（defensive），不是可选的
   - **验证**：学生现在能理解"这不是装饰，这是修复"

### 5. **正弦波公式 sin(2π·f·t) 中的 2π 是从哪来的？**
   - **提出者**：老陈（问题4）
   - **问题本质**：缺少圆周几何与正弦波的关联，公式看起来无源无故
   - **修订**：在"动手观察"前插入新 cell（ID: 44b4c649）
     - **《预热 2：圆周运动与正弦波的关系》**
     - 从匀速圆周运动的角度讲：每秒转 f 圈 → t 秒转 `2π·f·t` 弧度 → 取圆周上一点的竖直坐标 → sin()
     - 图形化：圆 → 角度 → 正弦值的链条
     - 结论：2π 来自"一圈"的定义（360° = 2π 弧度），不是魔法
   - **验证**：学生现在能说出"2π 是一个完整周期对应的弧度数"

### 6. **科学记数法 1e-12 在 assert 里是什么意思？浮点精度的概念**
   - **提出者**：琳达（问题4）
   - **问题本质**：不懂浮点数的精度为什么要设定为 1e-12，能否改成 1e-6 或 1e-15？
   - **修订**：在第 4 部分（ID: af1d3fdf）增加**《浮点精度为什么重要？》**小节
     - 解释二进制浮点数无法精确表示某些十进制小数（如 0.1）
     - 说明"足够接近"的判断需要阈值（threshold）
     - 在音频中，1e-12 秒的误差（一万亿分之一秒）远小于麦克风分辨率，可视为无误差
     - 这样学生理解了"为什么选这个值"（不是随意的，是量级相关的）
   - **验证**：学生现在能说出"1e-12 秒是物理上无感的精度"

## 遗留问题 3：老陈问题 5（部分）—— 自注意力 vs 交叉注意力的优势

**学生卡点**：cell `c97a2999` 讲了交叉注意力的结构（Q/K/V 来源）和编码器为何能看全部 1500 步，但没讨论"为什么这个设计比其他方案更优"。

**改动位置**：在 cell `c97a2999` 之后（紧接着，位于新增的 sqrt(d_k) 讲解之前）插入新 markdown cell（`c68136ec`）"追问：交叉注意力为什么比其他方案更好？"：

- **对比替代方案 1（拼接自注意力）**：说明如果把编码器和解码器 token 拼在一起做纯自注意力，解码器每生成一个新 token 都要重新拼接、重新计算，计算量 ∝ (1500+T_dec)²，且编码器信息被无谓地反复重算；而交叉注意力让编码器只算一次、K/V 可缓存反复复用（用"读一遍参考书做摘要，之后反复查摘要"类比）
- **对比替代方案 2（只看 top-k 相关步）**：说明硬性选择 top-k 需要额外的、通常不可导的选择机制，选错就永久丢信息；而 softmax 是一种"软性的 top-k"，全局可导、模型自己学会该看哪里，不会因为一次选错就彻底丢失信息

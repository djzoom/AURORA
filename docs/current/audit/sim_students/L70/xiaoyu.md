# 小雨 — L70 学习日志

## 我理解了什么

Whisper 就像一条流水线工厂：录音→转成图像（mel谱图）→编码器"看懂"→解码器一个字一个字生成文字。核心设计是**固定30秒窗口**（不够补零、超了切掉）和**两层卷积**把3000帧压成1500帧，让后面的注意力计算量更小。架构用**前缀token**（特殊ID序列）就能在同一个模型里切换"转写"还是"翻译"，很巧妙。

## 遇到的困难和问题

### 问题1：为什么 hop=160 就能推导出3000帧？【推演】
原文说"帧数为 `480000 // 160 = 3000`"。我理解了30秒=480000个采样点。但"hop=160"是什么意思？课程说"10ms帧移"，这里面逻辑是：采样率16kHz，10ms=0.01s，所以16000×0.01=160个采样点。但我想问——**为什么通过"跳过160个采样"就能得到帧数？这个公式的逻辑是什么**？这不像简单的除法，感觉像是在描述一个"滑动窗口"的过程，但教材没有清楚说明滑窗是怎么工作的。

### 问题2：Conv1D 的 stride=2 为什么就能"降采样2倍"？【理解】
课程给了公式 `n_out = (n_in + 2*padding - kernel) // stride + 1`，但我看不懂这个公式的**几何含义**。说"stride=2"，我能猜到是"每隔2个位置取一次"，但这为什么就等于"降采样"？而且代码验证的时候用了 padding=1，这又是什么？我能背下公式 `(3000+2-3)//2+1=1500`，但我不知道为什么这个公式是对的——感觉少了一个可视化图示。

### 问题3：注意力中的 Q @ K.T 和矩阵维度推导【推演/理解】
代码里写 `attn_weight = (Q @ K.transpose(-1,-2)) / scale`。我知道：
- Q shape=(B, T_dec, d_model)，K shape=(B, T_enc, d_model)
- K转置后 shape=(B, d_model, T_enc)
- 矩阵乘法 @ 后得到 (B, T_dec, T_enc)

但我的问题是**这个 @ 运算是什么意思**？不是说@表示矩阵乘法吗？可是(B, T_dec, d_model) × (B, d_model, T_enc) 怎么乘？我高中学的矩阵乘法是2D的，这里有3个维度(batch, 时间, 特征)，PyTorch是怎么处理batch维的？我得背下来"batch维不动，后两个维乘"这个规则，但我想理解**为什么**。

### 问题4：log-Mel 归一化里为什么要"减8.0"和"除4.0"？【理解】
课程说Whisper的归一化：`log_mel = max(log_mel, log_mel.max() - 8.0)` 然后 `(log_mel + 4.0) / 4.0`。

我可以跟着步骤做计算，但我**完全不知道这些魔法数字从哪来**：
- 为什么clip到"最大值减8.0"？这限制了动态范围吗？
- 为什么最后要"加4除4"？加4是为了平移到非负？除4是为了缩放？

课程说"让模型训练更稳定"，但这太抽象了。有没有更直观的解释，比如这样做之前和之后数值范围怎么变？

### 问题5：多任务token设计——一个模型怎么区分任务？【理解】
课程说：
```python
TRANSCRIBE  = 50359   # 转录任务
TRANSLATE   = 50358   # 翻译任务
```
两个任务用不同的token ID。但问题是：**模型为什么能通过这两个token理解"我要做转录还是翻译"？**

这两个token只是vocab里的俩个单词，跟其他50000+个token一样。为什么把token 50359 当开头就会翻录，把50358当开头就会翻译？是不是在**训练时**用了大量带不同前缀token的数据，所以模型学会了"如果序列以50359开头就输出转录，如果以50358开头就输出翻译"？但课程没有明说这一点。

### 问题6：30秒的mel谱图为什么有"3001帧"而不是"3000帧"？【理解】
原文说："Aurora 的 `mel_spectrogram` 默认 `center=True`（反射填充），实际返回 **3001** 帧"，然后说"实际返回 **3001** 帧（`480000 // 160 + 1`）"。

为什么center=True会多出1帧？这个center=True是什么意思？反射填充是什么意思？课程说"官方实现同样先多算一帧再裁掉末帧"，感觉像是一个实现细节bug/workaround，但我不理解**为什么在开头和结尾要填充数据**。这样做会改变mel谱图的质量吗？

### 问题7：Encoder 输出"1500步向量"的含义【理解】
课程反复强调"编码器输出(B, 1500, d_model)"。这1500个向量分别代表什么？
- 是1500个时间帧吗？那每个向量是那一帧的声学特征的某种压缩表示？
- 为什么通过Transformer能得到这样的向量？Transformer是怎么把"3000个mel系数"变成"1500个特征向量"的？

我能接受"卷积层把3000→1500"，但Transformer层（self-attention + FFN）为什么不改变序列长度？这也没解释。

### 问题8：beam search 和贪婪解码会出现在下一课，但现在的"交叉注意力"演示代码里 T_dec=20 是什么意思？【理解】
代码里：`T_dec = 20  # 编码器 1500 步，当前解码到第 20 个 token`

这意味着解码器已经生成了20个token。但这个"20"是固定的吗？一句话可能只有10个token，也可能有100个。那为什么这里假设T_dec=20？课程没有解释**解码什么时候停止**。是因为解码器会生成一个"结束token"吗？

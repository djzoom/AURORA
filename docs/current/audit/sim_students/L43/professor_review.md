### 问题 3：帧数公式推导过程（为什么是 1 + (N - win_len) // hop？）
**学生问题**：老陈、小雨都能验证公式正确性，但想理解**为什么**是这个形式。

**修订内容**：在 af105b5d 后插入新 markdown cell `f2d002d9`，标题"概念 2.5：帧数公式的推导"
- **直观开场**：帧起点是 0, hop, 2·hop, ... 关键是最后一帧必须完整
- **数学推导**：设帧起点 m·hop，完整条件 m·hop + win_len ≤ N，反解 m ≤ (N-win_len)/hop
- **得出公式**：$n_{\text{frames}} = 1 + \lfloor (N - \text{win\_len}) / \text{hop} \rfloor$
- **手工验证**：列出两个数值例子（N=10, N=100）的逐步计算
- **补充说明**：公式仅计算完整帧；末尾不足补零的情形

**学生反馈点**：从不等式逆推，展示"为什么"的完整推导链条。

### 问题 5：COLA 条件是什么？为什么 50% 重叠满足它？
**学生问题**：老陈、琳达、小雨都反映白板挑战问 4 提及 COLA 但课程没解释，只知道"与 L41 有关"但没具体指向。

**修订内容**：在 8b748113 前插入新 markdown cell `27a0272e`，标题"为什么 50% 重叠？—— COLA 条件与完美重建"
- **问题引入**：加窗后帧相加可能改变幅度，怎么办？
- **COLA 定义**：所有重叠部分的窗相加都是常数 $\sum w(n + m \cdot \text{hop}) = \text{const}$
- **物理意义**：每个时刻被覆盖的次数和窗的和是固定的，能完美重建
- **Hann + 50% 为什么满足**：Hann 的性质 w(n) + w(n+N/2) = 1，50% 重叠 hop=N/2
- **实际应用**：语音增强、音频处理都依赖这个条件

**学生反馈点**：从"为什么要重叠"的问题出发，解释 COLA 的重要性，不是孤立公式。

### 问题 10：大信号时帧数比为什么渐近到 hop 比？
**学生问题**：老陈的高阶问题——为什么 N=4000 时比例 30.5 倍，但说"大信号渐近 16 倍"？

**修订内容**：在 cell-l43-0430 后插入新 markdown cell `05b732d1`，标题"为什么大信号时帧数比渐近到 hop 比？"
- **数学分析**：N 很大时，公式中的 "+1" 和 win_len 对比变小；主导项是 N/hop
- **比例变化**：$n_1/n_2 \approx (N/32)/(N/512) = 16$ 当 N→∞
- **小信号为什么不同**：N=4000 时 win=2048 只切 4 帧，"1" 对小数字影响大
- **边界效应解释**：小信号时偏差放大，大信号时偏差被吞没
- **验证建议**：用 N=80000 重试，比例应接近 16

**学生反馈点**：深层理解参数对帧数的影响，不是简单套公式。

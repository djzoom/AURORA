# 小雨 — L43 学习日志

## 我理解了什么（简要，3 行以内）

**FFT 的问题**：一次 FFT 只告诉你"有哪些频率"，但看不出"什么时候"出现——比如一首歌从 C 调转到 G 调，全 FFT 后两个频率都显示有，但分不清先后。

**STFT 的解法**：把长信号像翻幻灯片一样切成一帧一帧（长度 `win_len`，步进 `hop`），每帧单独做 FFT，输出 2D 矩阵（行=时间帧，列=频率）。这样就能画出声谱图。

**时频权衡（鱼与熊掌不可兼得）**：大窗口（比如 2048）→ 频率线条细（分辨率高），但时间定位模糊；小窗口（比如 128）→ 时间定位清晰，但频率线条粗。要实现 `frame_signal()` 完成分帧这一步。

### 【理解】关键概念 STFT[m,k] 公式中各项的量纲和单位不够清晰
> 原文：`STFT[m, k] = FFT( x[m·hop : m·hop + win_len] × window )[k]`

卡点：这个公式看起来很对称，但具体代入数字时容易混淆：
  - `m` 是帧编号（无单位的整数 0,1,2,...）
  - `m·hop` 是采样点索引（整数，指第几个采样点）
  - `m·hop : m·hop + win_len` 是切片（采样点范围）
  - `k` 是频率 bin 编号（无单位的整数 0,1,2,...）

表格里说"频率分辨率 Δf = sr / win_len"，但 k 和 Hz 的换算关系（k 对应多少 Hz？）没有明确给出。

希望补充：写清楚"第 k 个 bin 对应的频率 = k × sr / win_len (Hz)"。

# 老陈 — L43 学习日志

## 我理解了什么

STFT就是把长音频信号一段一段地切开（每段长度是`win_len`，段与段之间步进`hop`），然后对每段分别做FFT。这样就能知道不同时刻出现了什么频率——比起对整段信号做一次FFT只能看到"有哪些频率"要进步多了。计算帧数的公式是 `n_frames = 1 + (len(x) - win_len) // hop`，实现一个`frame_signal()`函数就是建立一个二维矩阵，每一行是从原信号切出来的一段。

### 【理解】"时间分辨率"到底指什么？表格和公式里不一致
**时频权衡表**里写："Δt = win_len/sr"，单位是"ms/帧"。  
**参数实验代码**里写："time_res_ms = hop / sr * 1000"。

这两个公式不一样啊！我很混乱：
- `win_len/sr`是一帧的"时间长度"（比如512点@8kHz = 64ms）
- `hop/sr`是帧与帧之间的"时间步进"（比如512点步进@8kHz = 64ms，如果hop=512就是无重叠）

**到底哪一个是"时间分辨率"？** 还是说"分辨率"有两种含义我还没分清？

### 【推演】帧数公式为什么是 `1 + (N - win_len) // hop` ？
能手工验证这个公式对：
- N=10, win=4, hop=2: n=1+(10-4)//2=1+3=4，四帧 ✓
- N=100, win=32, hop=16: n=1+(100-32)//16=1+4=5，五帧 ✓

但我想理解**为什么**是这个形式。帧的起始位置是0, hop, 2×hop, 3×hop, ...，最后一帧要完整，所以起始位置+win_len ≤ N。用不等式反推：
- m×hop + win_len ≤ N
- m ≤ (N - win_len) / hop
- m 的最大值是 floor((N - win_len) / hop)
- 帧数是 m+1（因为从0开始计数）

所以应该是 `n = (N - win_len) // hop + 1`，和表述 `1 + (N - win_len) // hop` 一样。**但为什么题目特别强调"能取完整帧的数量"？** 余下不足win_len的尾部真的就直接丢弃吗？没有补零吗？

### 【理解】频率分辨率和窗长的关系——为什么是sr/win_len？
表格说 `Δf = sr / win_len`（单位Hz/bin），窗越长分辨率越高。我能看出这个规律但不理解**为什么**：
- FFT的输出有 win_len//2 + 1 个bin（对实数）
- 这些bin覆盖频率范围 0 到 sr/2
- 所以每个bin的频率宽度是 (sr/2) / (win_len//2) ≈ sr/win_len

这样理解对吗？还是我想复杂了？

另外，这个"分辨率"的**物理意义**是什么？是说，我能分辨出两个相隔多远的频率成分？比如我能分辨1000Hz和1015.625Hz的差别吗（当Δf=15.625Hz时）？

### 【理解】为什么说大信号下帧数比会"渐近"到hop比例？
课程说 win=128/hop=32 和 win=2048/hop=512 在小信号(N=4000)下帧数比是30.5倍，但hop比是16倍。然后说"大信号时渐近到16倍"。

这个"渐近"意味着什么？是说当N趋向无穷，帧数比会趋近于hop比吗？
试试推导：
- n₁ = 1 + (N - 128) // 32 ≈ N/32（当N很大时）
- n₂ = 1 + (N - 2048) // 512 ≈ N/512（当N很大时）
- 比例 ≈ (N/32) / (N/512) = 512/32 = 16 ✓

哦这样就对上了。那为什么在N=4000时比例这么大（30.5 vs 16）？是因为2048窗对4000点信号来说太大了，只能切出4帧，而128窗还能切出122帧，所以头部的"1"在2048这边的影响更大？

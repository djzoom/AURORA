# 小雨 — L19 学习日志

## 我理解了什么
- 矩阵×向量不是什么神秘的运算，其实就是"把矩阵的各列分别乘以向量的系数，再加起来"（列线性组合）；这张图让我一下子明白了为什么要学矩阵×向量。
- 四种几何变换（旋转、反射、缩放、剪切）可以通过检查两个条件区分：① 看 `A @ A.T` 是否等于 `I`（单位矩阵），② 看 `det(A)` 是否为正或负。
- 向量的内积是标量（一个数），外积是秩1矩阵（能被拆成"列×行"）。

## 遇到的困难和问题

### 【理解】秩1矩阵的含义不清楚
第4节说"矩阵×矩阵 = 秩1矩阵之和"，但课程没有解释"秩1矩阵"到底是什么。我能看到代码中 `mat_times_mat_rank1` 会逐个显示各项，但为什么叫"秩1"？为什么矩阵乘法能分解成这样的东西？建议补充：秩是什么（简单定义），为什么秩1矩阵具有"列×行"的形式。

### 【理解】正交矩阵的判断条件为什么有效
第5节要用 `A @ A.T ≈ I` 来判断是否旋转/反射，但课程只说"旋转矩阵满足 R@Rᵀ=I"，没有解释**为什么**这个条件代表"保长度、保角度"的变换。作为初学者，我想知道：这个条件怎么推导出来的？它反映的几何含义是什么？

### 【推演】旋转矩阵公式从何而来
第4节用了旋转矩阵 `[[cos(θ), -sin(θ)], [sin(θ), cos(θ)]]`，但没有推导。为什么是这个形式？如果我想理解"旋转30°"为什么要用这个矩阵，应该从哪里开始？建议补充：从基向量 `[1,0]` 和 `[0,1]` 旋转后的新位置，推导出矩阵元素的推导过程（即使是几何直观推导也行）。

### 【推演】为什么判断顺序重要（反射矩阵会不会被误判为对角）
第5节强调判断顺序"必须先正交、再对角"，否则 `[[0,1],[1,0]]`（反射）会被误判。但这个矩阵的确有两个0在对角线外......我不明白它为什么会被误判为缩放。能否给一个具体反例说明顺序错了会怎样？

### 【计算】`det(A)` 函数的几何含义
课程用 `det(A)` 来区分旋转（det=+1）和反射（det=-1）。但 det 函数返回什么？为什么这个数值的符号能反映旋转vs反射？建议补充：行列式的直观含义（可能就是"面积的缩放因子"之类的），以及为什么旋转/反射会有不同的符号。

### 【实践】`np.radians()` 的用途
代码中频繁出现 `np.radians(30)` 这样的函数调用。我知道这是度数转弧度，但为什么矩阵里面一定要用弧度而不能用度数？是三角函数的要求吗？建议补充：为什么数值计算中用弧度。

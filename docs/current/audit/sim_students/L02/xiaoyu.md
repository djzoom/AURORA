# 小雨 — L02 学习日志

（背景：历史系毕业，高中数学水平，Python 只会 print/变量/循环/列表。第一次接触 NumPy。）

## 我理解了什么（简要）

- 声音是空气压强的起伏，计算机存不了"起伏"，只能每隔固定时间量一次，记成一串数字——这就是采样。吉他弦和麦克风的比喻我很喜欢，一下就懂了。
- 采样率 sr 就是"每秒量几次"。一秒声音 = sr 个数字，所以 N = round(duration × sr)。任务 0 的三个手算我都算对了（4、16000、11025），代码 assert 也全过。
- 时间轴就是给每个采样点标上"它是第几秒量的"：第 n 个点的时刻是 n/sr。手算 [0, 0.125, 0.25, 0.375] 和代码输出完全一致，很有成就感。
- shape 用电影票"排号/座位号"来比喻很好懂：一段声音就是一排 N 个数，`(N,)`；以后会有矩阵 `(M, N)`。
- `make_sine` 就是把公式 A·sin(2πft) 套在时间轴上。跑了三个频率的对比图和 220/440/880 Hz 实验，确认了：频率越高波峰越密、音越高；振幅不变所以 max 都是 1.0，shape 也都一样。三个预测问题我都答对了。
- `signal_summary` 的手算表（[0,1,0,-1] → length 4、max_abs 1、mean 0、rms 0.707）我算出来了，和代码一致。
- 白板挑战 4 问我在纸上都推出来了：N=4000、t[100]=0.0125s、x[100]=0.7·sin(2π·440·0.0125)、RMS=0.7/√2≈0.495，验证格全部通过。"先手算再对答案"这个方法对我这种怕公式的人真的有效。

## 遇到的困难和问题

### 问题 1【理解】"2 倍就能完整还原"——为什么是 2 倍？

**位置**：第 1 节"声音的物理本质"，原文："根据奈奎斯特定理，采样率只需达到最高频率的 2 倍就能完整还原"。

**卡点**：这句话对我来说是凭空出现的结论。为什么恰好是 2 倍？1.5 倍为什么不行？3 倍是不是更好？"完整还原"又是什么意思——量出来的明明只是一个个孤立的点，中间没量到的部分怎么就能"还原"？课程后面提到 L34 有 aliasing（混叠）这一课，但这一节里没有任何指引，我只能选择"死记 2 倍"。

**希望课程补充**：哪怕不推导，给一张示意图也好——比如"如果点取得太稀，快波和慢波会穿过同样的点，计算机就分不清了，所以一个周期至少要量到两个点"；或者明确写一句"为什么是 2 倍，L34 会用实验让你亲眼看到"。

### 问题 2【解答】自评问"为什么 sr=44100 而不是 44000"，但课文根本没讲过

**位置**：最后的自评单元 `l02_review` 里的 `"sampling_rate_intuition": # 能解释为什么 sr=44100 而不是 44000？`

**卡点**：我翻回第 1 节反复读了三遍，课文只说"44100 > 40000，刚好够用，还留了余量"。这解释了为什么要大于 40000，但完全没解释为什么偏偏是 44100 这个奇怪的数而不是 44000 或 45000。自评却拿这个问我，我要么心虚地填 True，要么填 False 然后"重新阅读对应章节"——可是重读也读不出答案，因为课文里就没有。

**希望课程补充**：要么在第 1 节补一句 44100 的来历（哪怕一句"这是历史原因，和早期录像设备有关"也行），要么把自评问题改成课文真正教过的："能解释为什么 sr 必须大于 40000？"

### 问题 3【计算】任务 2 手算表：sin(0.785) 让我根本没法"手算"

**位置**：任务 2 的手算表，"θ = 2π·f·t (rad)" 一列给出 0.000 / 0.785 / 1.571 / 2.356，要求填"sin(θ)（精确到小数点后 2 位）"。

**卡点**：课程反复强调"不要运行代码，先手算"，但我面对 sin(0.785) 完全无从下手——高中学的是 sin30°、sin45° 这种特殊角。我卡了很久才灵光一闪：0.785 大概是 3.14159 的四分之一，也就是 π/4 = 45°，sin45° = √2/2 ≈ 0.71。如果没猜到这层，这张表对我就是"必须用计算器"的表，违背了它自己"纸上手算"的初衷。另外表头的"rad"（弧度）这个单位课程也没解释过一个字。

**希望课程补充**：表里加一列"θ 等于几分之π / 相当于多少度"（0.785 = π/4 = 45°），并用一句话交代弧度和角度的换算（π rad = 180°），这样高中三角函数知识就能直接接上。

### 问题 4【推演】"纯正弦波 RMS = A/√2" 只给结论，白板挑战却要靠它

**位置**：任务 3 手算表的提示"（提示：纯正弦波 RMS = A/√2）"，以及白板挑战问 4"提示：纯正弦波 A·sin(...) 的 RMS = A/√2"。

**卡点**：两处都是把 A/√2 当既成事实塞给我。4 个点的特例我确实能手算验证（√(2/4) = 1/√2 ≈ 0.707，对上了），但为什么**任意**正弦波、任意频率、任意采样率下 RMS 都恰好是 A/√2？以我的水平完全推不出来。而且更基本的问题是：RMS 到底在衡量什么？为什么要引入它？课程只说 L40、L51 会用到，我等于背了一个既不知来源、也不知用途的公式——这和本课"先手算推导、拒绝黑盒"的精神有点矛盾。

**希望课程补充**：一句直觉解释（比如"RMS 大致对应你耳朵感受到的平均响度，mean 是 0 帮不上忙，所以先平方再平均"），再加一句"A/√2 的一般证明需要积分，LXX 课会推，这里先用 4 点特例验证"，让我知道这笔账记在哪儿了。

### 问题 5【实践】手算 mean = 0，代码却打印 5.551115123125783e-17，吓了我一跳

**位置**：任务 3 验证格打印的 `summary = {'length': 4, ..., 'mean': 5.551115123125783e-17, ...}`。

**卡点**：手算表让我算 mean = (0+1+0-1)/4 = 0，我算的是 0；结果代码打印出 `5.551115123125783e-17` 这么一长串。我第一反应是自己函数写错了，检查了十分钟。后来注意到 assert 用的是"接近 0"（`< 0.05`）才敢猜：这大概是计算机存小数有误差、e-17 表示小数点前有 16 个零、其实就是 0。但这全是我自己猜的——课程在这个"手算 vs 代码对答案"的关键环节，恰好出现了手算和代码**表面上对不上**的情况，却一个字都没解释。

**希望课程补充**：要么在打印时 round 一下让它显示 0.0，要么（更好）借机加一小段说明："看到 5.55e-17 别慌，这是浮点数的固有误差，e-17 意味着它比一粒尘埃还小，本质就是 0——这也是为什么我们的 assert 都用'足够接近'而不是'完全相等'。"这对初学者是很宝贵的一课，现在却成了惊吓。

### 问题 6【理解】"float64 每元素 8 字节"——"字节"是什么？为什么 64 对应 8？

**位置**："NumPy 三要素"第一格，原文："float64 每元素 8 字节，float32 每元素 4 字节……省一半内存"，以及 itemsize、nbytes、astype 一串新名词。

**卡点**：前半课每个新概念都有生活化比喻（电影票、吉他弦），到了这一格突然全是行话："字节"是什么单位？为什么 float**64** 是 **8** 字节、float**32** 是 **4** 字节——64 和 8 之间是什么关系？itemsize、nbytes 又是什么？我只能看懂最后的结论"float32 省内存、对人耳够用"。这一格我实际上是"跳读"过去的。

**希望课程补充**：开头一两句铺垫即可："计算机里 1 个字节 = 8 个开关（bit），float64 就是用 64 个开关存一个小数，64÷8 = 8 字节；开关越多小数越精细，但也越占地方。"有这一句，后面的 itemsize=8、nbytes=64 就全通了。

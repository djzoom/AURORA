# 阿凯 — L02 学习日志

背景：高中理科毕业，做了 6 年销售，数学捡回来一半，Python 刚学完基础语法。

## 我理解了什么（简要）

- 声音是空气压强的波动，计算机存不了连续的"起伏"，所以每秒量 sr 次电压值，
  把声音变成一串数字。N = round(duration × sr) 这个我手算三行表格都对上了
  （0.5×8=4，1×16000=16000，0.25×44100=11025）。
- 时间轴就是给每个采样点编号 0..N-1，再除以 sr 换算成秒。t = np.arange(N)/sr。
- 正弦波：x = A·sin(2π·f·t)。频率翻倍音高升八度（220/440/880 那个表很直观，
  可惜我在命令行里听不了）。
- shape 那个电影票比喻好懂：(N,) 是一排座位，(M,N) 是 M 排 N 座。音频就是一排。
- signal_summary 的手算例子 [0, 1, 0, -1] 很友好：length=4、max_abs=1、
  mean=0、rms=√(2/4)=0.707，全部能在纸上算出来。
- "先手算再运行对答案"这个方法我很喜欢，跟我以前做题找感觉的路子一样。
  白板挑战 4 题我都推出来了（N=4000，t[100]=0.0125s，
  x[100]=0.7·sin(2π·440·0.0125)，rms=0.7/√2≈0.4950），运行验证格全过。

## 遇到的困难和问题

### 问题 1【理解】奈奎斯特定理一句话带过，"2 倍就够"我完全不知道为什么

位置：第 1 节"声音的物理本质"——"根据奈奎斯特定理，采样率只需达到最高频率的
2 倍就能完整还原"。

我卡的点：这句话对我就是一个凭空出现的结论。为什么是 2 倍？1.5 倍不行吗？
3 倍会更好吗？"完整还原"又是什么意思——4 个点怎么可能"完整"代表一条连续曲线？
课程后面拿这个数字算出 44100 > 40000"刚好够用"，整个推理都建立在这个
我没法验证的定理上。

希望补充：哪怕不证明，也给一个直观图——比如"一个周期至少要采到 2 个点，
不然波峰波谷都会漏掉"，或者明说"这是 L34（aliasing）的内容，这里先记结论"。
现在既没解释也没说去哪找解释。

### 问题 2【推演】公式 x(t) = A·sin(2π·f·t) 里的 2π 是哪来的，课程直接给公式没铺垫

位置：任务 2，"公式：x(t) = amplitude × sin(2π × frequency × t)"。

我卡的点：高中学的是 sin(x) 的 x 是角度或弧度，这里 f·t 是"次数"，
为什么乘个 2π 就能塞进 sin 里？我自己琢磨了一会儿才模糊想起
"sin 转一圈是 2π 弧度"，所以 2π·f·t 大概是"到 t 秒为止转过的总弧度"。
但这是我猜的，课程从头到尾没有一句话解释这个 2π，后面 cell 8、24、25
的代码里也全是 `2 * np.pi * f * t`，都默认我懂。

希望补充：在任务 2 公式下面加两三行："sin 每 2π 弧度重复一次（转一圈）；
f 是每秒转几圈；所以 t 秒时总共转了 f·t 圈 = 2π·f·t 弧度"。有这一步，
公式就是推出来的，不是背出来的。

### 问题 3【计算】手算表里 θ=0.785 这种弧度小数，我一开始按"度"算，完全对不上

位置：任务 2 手算表，"θ = 2π·f·t (rad)"一列给的是 0.000 / 0.785 / 1.571 / 2.356。

我卡的点：我对"角度 vs 弧度"本来就分不清。看到 sin(0.785) 我下意识按度算——
sin(0.785°) ≈ 0.014，跟后面验证格打印的 0.707 差了几十倍，我一度以为自己
函数写错了。折腾半天才反应过来 0.785 = π/4 = 45°，1.571 = π/2 = 90°。
表头虽然写了 (rad)，但对我这种忘光了的人，一个小数形式的弧度值在纸上
根本没法手算 sin。

希望补充：手算表的 θ 列同时标注分数形式和度数，例如
"0.785 (= π/4 = 45°)"，并提醒"这些恰好是特殊角，用 sin45°=√2/2≈0.71 即可"。
不然"先手算"这一步对忘了弧度的人是做不下去的。

### 问题 4【实践】make_time_axis 我用 np.linspace 写，断言失败，"不含终点"没有提前解释

位置：任务 1。我的第一直觉是"0 到 0.5 秒平均分 N 个点"，
写了 `np.linspace(0, duration, N)`，结果得到 [0, 0.167, 0.333, 0.5]，
断言 `np.allclose(t, [0.0, 0.125, 0.25, 0.375])` 直接失败。

我卡的点：为什么时间轴最后一个点必须小于 duration？0.5 秒的信号，
最后一个时刻停在 0.375 秒，我觉得反而是"少了一块"。这个要求只出现在
验证格的断言消息里（"最后一个时刻应 < duration（不含终点）"），
正文没有一句解释原因。我最后是照着"推理路线"抄 arange/sr 才通过的，
但只是照做，没想通。

希望补充：讲一句为什么——比如"第 N 个点（t=duration）是下一段的第 0 个点，
两段音频拼起来才不会重复一个点"，再顺手点一句"所以别用
linspace(0, duration, N)"。这个坑我猜不止我一个人会踩。

### 问题 5【计算】手算 mean=0，代码却打印 5.551115123125783e-17，没解释这是什么

位置：任务 3 验证格。手算表里 mean = (0+1+0-1)/4 = 0，我算的是 0；
运行 `signal_summary(x4)` 打印出来却是
`'mean': 5.551115123125783e-17`。

我卡的点：这一串数字对我像乱码。它是 0 吗？是我函数写错了吗？
我盯了很久，最后靠断言 `abs(mean) < 0.05` 能通过才安心。课程的方法论是
"手算和代码对答案"，但当代码给出的答案和手算长得完全不一样时
（0 vs 5.55e-17），没有任何一句话告诉我"这就是对上了"。
类似地，白板挑战 Q3 我推出 x[100] = 0.7·sin(11π) = 0，
实际打印的是 -1.5e-15。

希望补充：在第一次出现这种数时加个小方框："计算机的小数有极小的舍入误差，
e-17 表示 10 的负 17 次方，就是 0.00…0055，可以当成 0。所以对答案用的是
`np.isclose`/容差，而不是 `==`"。这对"先手算再验证"的教学法是刚需铺垫。

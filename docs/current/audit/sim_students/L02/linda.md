# 琳达 — L02 学习日志

背景自述：我是做运营的，自学过 Python 爬虫，代码读写没问题；数学只有高中水平，
看到连着几行公式推导就头皮发麻。我学这课的目的是搞清楚"这些代码到底在干什么"。

## 我理解了什么（简要）

- **声音就是数组**这个大图景我完全接受了：麦克风把气压起伏变成电压，每秒量 N 次，
  存成一串浮点数。"一秒音频 = 44100 个浮点数"这句话很震撼也很好记。
- 四个函数我都自己写出来并通过了全部 assert：
  - `samples_count`：就是 `round(duration * sample_rate)`，手算表 4 / 16000 / 11025 全对上了。
  - `make_time_axis`：`np.arange(N) / sr`，手算 [0, 0.125, 0.25, 0.375] 和代码输出一致。
  - `make_sine`：套公式 `A * np.sin(2*np.pi*f*t)`，断言全过。
  - `signal_summary`：按注释里给的五个键写字典，一次通过。
- **shape 那个"电影票"比喻很好**：`(N,)` 是一排座位、`(M, N)` 是几排几座、`()` 是标量。
  我以前用爬虫处理数据时被 pandas 的形状坑过，这段讲得比我看过的教程都清楚。
- dtype / 广播 / 切片三个演示格对我这种写过 Python 的人是舒适区：
  float32 省一半内存、标量自动广播、`x[::2]` 降采样，输出我都逐个核过。
- 白板挑战四问我在纸上做完了：N=4000、t[100]=0.0125s、x[100]=0.7·sin(2π·440·0.0125)、
  RMS=0.7/√2≈0.495，验证格全部 ✅。
- "先手算再让代码对答案"这个方法论我很喜欢，比"跑一下看输出"踏实多了。

## 遇到的困难和问题

### 问题 1【理解】"奈奎斯特定理"一句话带过，而自评却要求我解释 44100

位置：第 1 节"声音的物理本质"——"根据奈奎斯特定理，采样率只需达到最高频率的
2 倍就能完整还原"。

卡点：为什么是 2 倍？1.5 倍不行吗？课程一个字都没解释，就当定理名字一报我就该信。
更难受的是课末自评有一项 `sampling_rate_intuition`——"能解释为什么 sr=44100 而不是
44000？True/False"。我翻回去把正文读了三遍：正文只说"44100 > 40000，刚好够用，
还留了余量"——可是 44000 也大于 40000、也留了余量啊！这道自评题用课程给的内容
根本答不出来，我只能诚实地填 False，然后被提示"重新阅读对应章节"——再读十遍也没有答案。

希望补充：(a) 用一张图直观说明"每个波峰波谷至少要采到一个点"为什么导致 2 倍这个数
（哪怕不证明，给个直觉图也行）；(b) 要么在正文里讲清 44100 这个具体数字的来历，
要么把这条自评题删掉/改成正文里真的讲过的问题。

### 问题 2【推演】公式里的 2π 从天而降，我不知道它是干什么的

位置：任务 2，"**公式**：`x(t) = amplitude × sin(2π × frequency × t)`"。

卡点：我高中学过 sin，但为什么频率前面要乘一个 2π？去掉 2π 会怎样？"frequency=2
就是每秒振动 2 次"这个说法和公式之间的桥，课程完全没搭。我照抄公式能通过 assert，
但如果面试官在白板前问我"这个 2π 是什么"，我答不上来——这恰恰违背了本课
"白板上也能推出来"的初衷。

希望补充：一小段话或一张图说明"sin 转一整圈是 2π 弧度，所以 2π×f×t 的意思是
t 秒内转了 f×t 整圈"。两三句话就够，但现在一句都没有。

### 问题 3【计算】手算表让我算 sin(0.785)，可我没有计算器算不出来

位置：任务 2 的手算表——θ 一列给的是 0.000 / 0.785 / 1.571 / 2.356，
要求"精确到小数点后 2 位"填 sin(θ)，而课程规则是"不要运行代码"。

卡点：sin(0.785) 我徒手真算不了。后来对答案时才发现 0.785 其实就是 π/4、
1.571 就是 π/2——这些特殊角的 sin 值（0.71、1、0.71）我高中背过！但表里偏偏
写成小数，我完全没认出来，只好"作弊"跑了代码。一张为"手算"设计的表，
却用了一种让人没法手算的写法，很别扭。

希望补充：θ 一列同时标注弧度小数和 π 的倍数，例如"0.785 (=π/4)"，
再提示一句"sin(π/4)=√2/2≈0.71"。这样手算才真的可行。

### 问题 4【推演】RMS 是什么、为什么等于 A/√2，全程靠背

位置：任务 3 的手算表最后一行——"rms | √((0²+1²+0²+1²)/4) | ✏️ ?
（提示：纯正弦波 RMS = A/√2）"；白板挑战问 4 又直接用这个结论。

卡点：两层都卡。第一层：RMS 这个量是干嘛的？为什么要"先平方、再平均、再开根"
这么绕，直接取绝对值的平均不行吗？课程只给公式没给动机。第二层："纯正弦波
RMS = A/√2"这个提示是哪来的？我拿 4 个点 [0,1,0,-1] 手算 √(2/4)=0.707 确实对上了，
但换任何别的正弦波我都不敢保证——这不是我推出来的，是我背下来的。白板挑战 Q4
考的就是这个背下来的数。本课口号是"先手算推导"，可这一条恰恰推不了。

希望补充：(a) 一句 RMS 的直白动机（比如"衡量信号平均有多响，正负抵消所以先平方"）；
(b) A/√2 哪怕不严格证明，也可以像 4 点例子那样多举一个 8 点的例子让我看到
"点取得再多，结果都停在 0.707 附近"，让这个数不再像魔法。

### 问题 5【解答】手算说 mean=0，屏幕上却打出 5.551115123125783e-17

位置：任务 3 对答案格的输出——`summary = {... 'mean': 5.551115123125783e-17 ...}`；
以及白板挑战 Q3 打印 `x[100] = ... = -0.000000`。

卡点：手算表明明让我填 mean = (0+1+0-1)/4 = **0**，结果一运行，屏幕上是一串
`5.551115123125783e-17`。我第一反应是我函数写错了，检查了十分钟才猜到这可能是
"浮点数不精确"之类的事。白板挑战 Q3 同样：我手算发现 2π×440×0.0125 = 11π，
sin(11π) 应该恰好是 0（这步我居然自己推出来了，挺开心），可程序打印的是
-0.000000（实际是 -1.5e-15），课程对这两处"理论是 0、屏幕不是 0"都一声不吭。
assert 用了 atol 所以能过，但初学者看到的是"我的答案和屏幕对不上"。

希望补充：在对答案格加一行说明，例如"5.55e-17 就是 0，这是浮点运算的
微小误差，之后课程判等都用 np.isclose 而不是 =="。一句话就能省掉我十分钟的自我怀疑。

## 总体感受

这课的"手算表 → 写代码 → assert 对答案"节奏非常适合我，四个函数难度阶梯也合理，
shape 插曲和 NumPy 三要素对写过 Python 的人是很好的复习。但凡是涉及"数学为什么"
的地方（2 倍采样、2π、RMS=A/√2），课程都选择了直接报结论，而课末的自评和白板挑战
又偏偏考这些结论——学得会代码，答不出"为什么"，这个落差希望后续版本能补上。

### 【推演】DCT-II 的正交归一化系数  
**卡在**："正交归一化系数 `sqrt(2/N)`，首项 `sqrt(1/N)`"  
验证代码里有：
```python
scale = np.full(n, np.sqrt(2.0 / n)); scale[0] = np.sqrt(1.0 / n)
ref_dct = x_test @ basis.T * scale
```
这涉及线性代数（矩阵转置 `@ basis.T` 怎么理解？为什么这样用？）。更根本的问题是：
- DCT 矩阵为什么需要"正交"？如果不正交会怎样？  
- 这两个系数 `sqrt(2/N)` 和 `sqrt(1/N)` 是怎么推导出来的？  
**需要补充**：DCT 正交性的数学意义（可能需要线性代数的特征值、矩阵范数等概念）、这两个系数的来由。

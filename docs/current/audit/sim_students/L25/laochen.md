# 老陈 — L25 学习日志

## 我理解了什么

梯度下降的核心思路：**蒙眼下山**，每步沿着最陡坡度的反方向走一小步。公式是 `x_新 = x_旧 - lr × 梯度`。用简单例子 f(x)=(x-3)² 验证了几步手算，从 x=0 出发，0.1 的学习率每步都让 x 靠近最小值 3。学习率大小决定收敛快慢和稳定性：太小收敛慢，太大会震荡或飞出去。

## 遇到的困难和问题

**【推演】为什么梯度会按公比 (1-2·lr) 收缩**  
课程说"每步误差乘以公比 `1−2·lr`"，但没给推导过程。我试着自己推：设误差 e_n = x_n - 3，那 e_{n+1} = x_{n+1} - 3 = (x_n - 0.1·2(x_n-3)) - 3……算不出来 e_{n+1} 怎么就变成了 (1-2·lr)·e_n。建议补充这一步的对称代数推导，否则那个公比就是魔法。

**【理解】学习率的收敛条件是通用的吗**  
课程说"0 < lr < 1 时收敛"（基于 f(x)=(x-3)²），但这好像只对这个特定的二次函数成立。如果梯度更陡（比如四次函数或其他形状），是否 lr 的范围也会跟着变？没有讨论。怕我以后套用这个规则到其他函数上会出问题。

**【理解】局部最小值和鞍点没区分清**  
课程说梯度为零时"可能是谷底，也可能是鞍点"，但没解释这两个概念的区别、怎么分辨、在神经网络训练中会不会被卡住。这是个留下的疑虑。

**【理解】二维梯度下降的推广有点突兀**  
主教程全是一维例子，白板第 4 问突然跳到二维 f(x,y)=x²+y²，∇f=(2x,2y)。"对每个分量分别求梯度再分别更新"这个逻辑在直观上说得通，但没有看到正式的推导或为什么这样做是对的。

**【理解】线性回归那一段梯度推导不清楚**  
最后的"实战预告"代码里，对 MSE 损失求梯度得到 `2*mean(err*X)` 和 `2*mean(err)`。课程没有推导这是怎么来的。我是 Python 基础，看得懂代码运行，但不明白为什么这些就是对 w 和 b 的偏导——希望能有类似前面 f(x)=(x-3)² 这样明确写出 df/dw 和 df/db 的推导。

**【实践】50 步迭代的收敛无法靠心算验证**  
题目说不运行代码，"靠读代码和心算理解"。但看那个最后的"学习率对比图"，有的 lr 跑 100 步，我没法心算出每条曲线的精确形状，只能相信代码是对的。对于验证收敛行为这种"需要数值"的问题，建议要么给更小的步数（2-3 步手算），要么允许运行代码看实际效果。

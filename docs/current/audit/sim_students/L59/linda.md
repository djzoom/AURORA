# 琳达 — L59 学习日志

## 我理解了什么

Tensor 就是升级版的 ndarray，能住在 GPU 里、能追踪梯度，这两个能力是深度学习的基础。我掌握了三个核心转换：`torch.from_numpy()` 做零拷贝共享、`.float()` 转 float32、`.unsqueeze(1)` 插入维度，这正好是把 numpy MFCC 打包成 CNN 输入的完整路线。自动梯度就是定义完表达式后一句 `y.backward()` 就能自动求导。

## 遇到的困难和问题

### 【理解】什么是 DAG，梯度为什么能"沿 DAG 反向传播"？（Cell 7）
课程说「有向无环图（DAG）」和「反向传播」，但没有画过任何图。从 `x**2 + 2*x` 这个表达式，PyTorch 怎么一步步构建这个图？这个图长什么样？梯度怎样沿着箭头反向流？需要一个具体的例图或逐步构图的演示。

### 【推演】为什么非标量的 Tensor 调用 `.backward()` 需要先 `.sum()` 或传 `gradient` 参数？（Cell 7）
课程只说「默认只对标量调用」，但没解释为什么。比如 `y.shape=(3,4)` 时，对它直接 `.backward()` 为什么会出错？`.sum()` 为什么能解决？这背后有什么数学原理？

### 【理解】einsum（爱因斯坦求和）的记号和计算规则（Cell 9）
看到 `'bi,ij->bj'` 这种记号完全懵了。课程说「重复的下标代表求和」，但为什么重复的就要求和？`i` 重复了所以对 `i` 求和？具体是怎么求的？能否举个 3×3 和 2×3 矩阵相乘的 einsum 过程，逐行逐列演示出来？这需要线性代数的下标记号，但课程没铺垫。

### 【理解】`torch.from_numpy()` "零拷贝"和内存共享的底层机制（Cell 3, 6）
代码里确实演示了改 `arr[0]=99` 后 `t[0]` 也变了，说明内存共享。但这是怎样做到的？NumPy 的 ndarray 和 PyTorch 的 Tensor 在内存里长什么样？为什么 `torch.tensor(arr)` 不共享，`torch.from_numpy(arr)` 就共享了？这是调用了什么底层操作？

### 【理解】为什么梯度默认累加，而不是默认清零？（Cell 14）
实验 B 演示了 3 次 backward 梯度累加到 24，这显然是陷阱。但为什么 PyTorch 设计成累加，而不是每次清零？这种设计解决什么问题？什么场景下我们**想要**梯度累加？

### 【理解】float32 vs float64 性能差异的硬件原因（Cell 9, 白板挑战 Q5）
Q5 答案说「GPU 上 float32 速度是 float64 的 2-8 倍」，但这是为什么？float32 只是少存了一倍的比特，速度怎么会快 2-8 倍？这里涉及 GPU 硬件的什么原理？我没学过 GPU 计算，希望课程能简单解释一下。

# 老陈 — L59 学习日志

## 我理解了什么

**Tensor 是 NumPy 的升级版**：能存在 GPU 上、能自动求导。最常见的互转是 `torch.from_numpy()` 零拷贝（改原数组 Tensor 也跟着变），和 `torch.tensor()` 复制数据（更安全）。

**requires_grad 让 Tensor 自动记录操作**：设 `x = torch.tensor(3.0, requires_grad=True)`，对它做 `y = x**2 + 2*x` 然后 `y.backward()`，PyTorch 自动算出 `dy/dx = 6`，存到 `x.grad` 里。不用我手算偏导数了。

**mel_to_batch() 的核心逻辑**：把 NumPy MFCC 特征列表 stack 成 Tensor，再插入 channel 维度变成 CNN 需要的 `(B, 1, T, n_mels)` 四维形状。

## 遇到的困难和问题

### 【理解】view 和 reshape 中"内存连续"是什么

课程说 `view` 要求内存连续，不连续则报错；`reshape` 更宽松。但什么叫内存连续？为什么 NumPy 数组转过来的 Tensor 有时连续有时不连续？初学者（我）无法从代码里判断一个 Tensor 是否连续，也不知道什么操作会导致不连续。

> 希望补充：什么情况下 Tensor 的内存会不连续？怎么用代码检查和强制连续化？

---

### 【理解】cat 和 stack 的实际使用场景差异

代码里 `cat([x,y], dim=0)` 得到 `(4,3)`，`stack([x,y], dim=0)` 得到 `(2,2,3)`。我看懂了形状变化，但不懂**为什么**要有这两个操作。实际工程里什么时候用 cat，什么时候用 stack？比如 mel_to_batch() 里为什么是 `np.stack(mel_list)` 而不是 `np.concatenate()`？

> 希望补充：几个真实场景例子，说明哪种情况必须 cat，哪种必须 stack。

---

### 【理解】unsqueeze(1) 为什么在位置 1，而不是 0 或 2

mel_to_batch() 里从 `(B, T, n_mels)` 变成 `(B, 1, T, n_mels)` 用的是 `unsqueeze(1)`。课程说这是"插入 channel 维以满足 Conv2d 对 `(B,C,H,W)` 的要求"。

但我的问题是：**为什么 channel 维要在位置 1**？是所有 CNN 框架都这样约定吗？如果我不知道这个惯例，怎么会想到 unsqueeze(1) 而不是 unsqueeze(0)？

> 希望补充：PyTorch Conv2d 为什么强制要求 `(B,C,H,W)` 这个顺序？有没有其他框架用不同顺序（比如 TensorFlow）？怎么快速记住这个约定？

---

### 【理解】为什么 PyTorch 梯度默认累加而不是覆盖

实验 B 里循环调用 3 次 `backward()` 没有 `zero_()` 就得到梯度 `24`（累加了）。虽然实验清楚了这个现象，但**课程没有解释为什么 PyTorch 要这样设计**。

是因为有些训练场景需要梯度累加吗？为什么不让用户显式传参来控制，而要默认累加？这对初学者来说是个"陷阱"。

> 希望补充：梯度累加的实际用途场景。为什么不设成"默认覆盖，需要时手动累加"？

---

### 【理解】einsum 符号系统的完整规则

代码给的例子 `torch.einsum('bi,ij->bj', inp, W)` 说"对 i 求和 = 矩阵乘法"。但符号系统是怎么工作的？

比如：
- 为什么 'bi,ij' 里 i 重复就代表求和？
- 箭头左侧所有字母的顺序是固定的吗？'ij,bi->bj' 和 'bi,ij->bj' 会一样吗？
- 如果我想做转置，是不是 'bi,jd->bj'？

代码只给了一个例子，没有讲清楚规则。

> 希望补充：einsum 的完整规则（爱因斯坦求和的数学定义）、常见模式速查表（转置、矩阵乘、矩阵-向量乘等）。

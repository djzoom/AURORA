#### Q5："激活范数"的定义
**问题**：参数实验用 `out.norm().item()` 作为"激活范数"，不明白是什么，有什么数学依据。

**修订内容**：
- **修改 cell 0514**：在参数实验之前补充"什么是激活范数"
- **定义讲解**：L2 范数的数学定义 $\|\mathbf{v}\|_2 = \sqrt{v_1^2 + \cdots + v_n^2}$

### 老陈 Q1 / 琳达 Q2：padding=k//2 缺少"正向推导"

**问题**：cell `38849ff7`（深挖：卷积输出形状的通用公式）里已经有通用公式和 Conv1 的"反推"过程（即先知道答案 81，再倒推出 padding 该等于几），但复核指出：反推不等于正向推导。一个只有高中数学基础的学生，看到反推例子后，遇到别的 kernel 大小（比如 7 或 9）时，**不一定能自己独立推出 padding 该怎么设**，因为反推是"代入已知答案验证"，不是"从约束条件出发求解"。

**修订内容**（位置：cell `38849ff7` 末尾追加 + 新增代码 cell `f9f486d6`）：

1. **新增"正向推导：从'我想让宽度不变'直接推出 padding 公式"一节**，五步走：
   - 第一步：把目标"宽度不变"写成方程 $W_{out}=W_{in}$
   - 第二步：把通用 shape 公式在 stride=1 时化简成不含取整符号的线性等式
   - 第三步：代入目标条件，通过移项（初中代数）解出 $P_w=(K_w-1)/2$ —— 这是**从约束正向解出**的公式，不是凑出来的
   - 第四步：分别代入 k=5/3/7 验证，与原有的反推结果对上
   - 第五步：专门讲清楚**为什么代码写 `k//2` 而不是 `(k-1)/2`**——用列表对比奇数/偶数 kernel 下两种写法是否相等，再用代数证明「K=2m+1 时，(K-1)/2 = K//2 = m」，说明二者在奇数 kernel 下恒等；并顺带回答"为什么卷积核几乎都选奇数"和"为什么不能是 k//3 或其他数字"——后者是因为 padding 量由"卷积吃掉的宽度 K-1"唯一决定，不是可选项
2. **新增可运行代码 cell（`f9f486d6`，插在 `38849ff7` 之后）**：用真实 `nn.Conv2d` 对 K=3/5/7/9 逐一验证 `(K-1)/2 == K//2` 且卷积后 `W_out == W_in`；额外验证 K=4（偶数）时公式给出非整数、若强行取整会导致 `W_out=82≠81`，用真实运行结果坐实"为什么必须选奇数 kernel"。

**验收**：从约束条件到公式的正向代数推导补齐，配合可运行代码验证，一个高中数学基础的学生现在能自己沿着五步推出 `padding=(K-1)/2`，并理解它和代码里 `k//2` 等价的原因。

# 琳达 — L63 学习日志

## 我理解了什么

1. **Mel频谱当图片，CNN来分类**：把声音变成一个(40频段 × 81时间帧)的"灰度图"，用CNN就像处理图片一样分类，比拉平后喂全连接层更聪明（保留空间结构、参数少）。

2. **第一层卷积的妙处**：kernel的高度恰好等于n_mels=40，这样卷积一次就把整个频率轴压成1，后面的模型只需在时间轴上做特征。

3. **Global Average Pooling解决变长问题**：对时间维求平均，输出始终是固定长度的向量(B,64)，不管输入音频是81帧还是150帧，模型都能用同一套fc参数，这是很聪明的设计。

# 老陈 — L30 学习日志

## 我理解了什么

- **Softmax 的任务**：把模型输出的任意分数（logits）变成概率分布（每个数在 0 到 1 之间，加起来等于 1）。数值稳定的技巧是先减掉最大值，避免 e^x 溢出。
- **交叉熵的含义**：`-log(p_true)`，只看真实类别的概率。分数差距越大，模型越自信，正确类别的概率越高，损失越小。
- **梯度公式**：`dL/dz_i = p_i - y_i`（softmax 概率减 one-hot 标签），直观上是"预测与标签的差"。

### 【理解】为什么交叉熵只需要看真实类别那一项

**卡在**：第 1 小节的解释。

**具体问题**：
课程说："训练目标是最大化模型给出正确类别的概率，即最大化 log P(correct)，等价于最小化 -log P(correct)。其余类别的概率，softmax 的归一化已经顺带把它们压低了，不用再单独写进损失里。"

这句话逻辑跳跃。Softmax 的归一化（和为 1）和最小化错误分类的概率是两回事。如果只优化一个类别，其他类别怎么不会被"浪费"掉？这背后是什么概率论或信息论的原理？

**希望补充**：从极大似然估计的角度解释为什么"只优化正确类别等价于优化整个分布"。

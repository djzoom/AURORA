# 小雨 — L61 学习日志

## 我理解了什么

**nn.Module 的"参数户籍系统"概念**：只要用 `nn.Parameter()` 或 `register_buffer()` 给参数办理"户口"，它们就自动进入 `parameters()` 和 `state_dict()`，被优化器和设备迁移（`.to('cuda')`）认管理。以前散落在 Python 变量里的参数现在有了统一的身份。

**nn.Sequential 就是省事儿的写法**：不用自己在 `forward()` 里一个个调用各层，直接按顺序堆进 Sequential，自动帮我串联起来。像一条工厂生产线，东西进来从头到尾过一遍就行。

**state_dict() 是保存和加载模型的标准接口**：字典形式，键是参数名（比如 `fc.weight`），值是实际数据，`torch.save()` 和 `torch.load()` 配合用就能做检查点。

## 遇到的困难和问题

### 【理解】nn.Linear 的权重形状为什么是 (out, in) 而不是 (in, out)？

> 原文位置："m.layers[0].weight.shape 应该是 (64, 40)"

说 `nn.Linear(40, 64)` 的权重是 (64, 40)，但直觉上输入是 40 维，输出是 64 维，权重应该是 (40, 64)。课程没解释为什么 PyTorch 存权重时是反过来的。涉及矩阵乘法的行列约定，我想象不清楚具体的计算过程。希望能有个图示或者详细说明：输入向量乘以这个 (64, 40) 的权重矩阵，数学上怎么乘出 (64,) 的输出？

### 【理解】register_buffer() 的实际用场是什么？

> 原文位置："不需要梯度、但要跟着模型走、进入 state_dict"

表格讲得很清楚，但为什么真的需要这样的东西我想不通。课程提到 BatchNorm 的 running mean，还有 Aurora 的 KVCache（键值缓存），但这两个我都没学过，无法想象。希望能给个超简单的生活化例子：比如"模型有个计数器，不需要更新参数但要记录它"这样的。

### 【理解】为什么 `nn.Linear` 自动就注册了参数，裸 `torch.Tensor` 就不行？

> 原文位置："赋值内置层（nn.Linear 等）参数自动注册"

代码里 `self.fc = nn.Linear(4, 2)` 之后，weight 和 bias 自动出现在 parameters()，但 `self.buf = torch.zeros(1)` 就不行。这涉及 `super().__init__()` 和 nn.Module 内部的魔法，具体怎么工作的？我知道 Python 有继承和super()，但 nn.Module 到底在 `__init__` 里做了什么，让子类的属性自动被追踪？

### 【推演】参数总量的矩阵维度细节

> 原文位置："Linear(40→64): weight=40×64=2560"

手算时明白了总数，但为什么要 40 乘 64 不是 64 乘 40，这个顺序的数学原因我没想通。感觉需要一个"输入向量乘权重矩阵→输出"的具体演算例子，而不仅仅是数字。

### 【实践】代码里 `torch.randn()` 和 `torch.zeros()` 什么区别？

> 原文位置："self.fc = nn.Linear(4, 2) ... self.buf = torch.zeros(1)"

课程没明确说过，但我看代码 randn 好像是随机生成，zeros 是全 0。这只是初始化的技巧还是有更深的含义？为什么示例里某些地方用 randn，某些地方用 zeros？

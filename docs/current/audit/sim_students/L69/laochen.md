# 老陈 — L69 学习日志

## 我理解了什么

CTC 前向算法通过动态规划把指数级的路径枚举变成矩阵递推。把目标 "ab" 扩展成 [∅,a,∅,b,∅]，然后用 α[t][s] 记录"到第 t 帧、停在位置 s 的所有合法路径的概率和"，递推计算就能避免列举 3¹⁰⁰ 条路径。复杂度从 O(V^T) 降到 O(T·S)，其中 S=2L+1。

## 遇到的困难和问题

### 【理解】什么是"合法路径"？

> 节引用：`α[t][s] = "在第 t 帧结束时，以 l'[s] 结束的所有合法路径的概率和"`

这个定义里"合法"的确切含义没有明说。从表格和代码看，我猜测"合法"是指：
- 从位置 0 开始，逐帧向右走，每步最多从 s 跳到 s+2（或停留/只跳一格）
- 但为什么最多跳 s+2？为什么不能跳 s+3？

希望课程能明确定义：合法路径是否一定满足"相邻帧的位置最多差 2"这个约束？如果是，这个约束是从哪里来的？

---

### 【推演】为什么 s-2 只在"l'[s] ≠ l'[s-2]"时才能跳？

> 节引用（代码注释）：`if s > 1 and lprime[s] != lprime[s-2]`

这个条件的逻辑我推不通。
- 课文说"避免把 `A_A` 错当成一个 A"——我理解是指：如果连续两帧都是 A，会被 collapse 合并成一个 A。
- 但这和"不能从 s-2 跳"有什么关系？

让我试试手算：
- 假设 l' = [∅, a, ∅, a, ∅]，目标是 [a, a]
- 在第 2 帧，位置 s=2（∅），我应该能从 s-1（第 1 个 a）来吧？这个没问题。
- 但如果我从 s=0（第 0 个∅）跳到 s=2（中间的∅），再在第 3 帧从 s=2 跳到 s=3（第 2 个 a），这个路径 ∅→∅→a，collapse 后是不是也是 a？

我需要课程解释：**为什么"同字符不能跳"这个规则能保证 collapse 的正确性？**

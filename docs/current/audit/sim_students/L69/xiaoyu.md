# 小雨 — L69 学习日志

## 我理解了什么

1. **为什么需要这个算法**：直接枚举所有路径会爆炸（6帧就3⁶=729条）。CTC用动态规划把"指数级 → 线性级"——不用全部枚举，用表格递推就行。

2. **扩展标签的套路**：在原标签"ab"每个字符之间和两端都插blank，变成 [∅, a, ∅, b, ∅]，这样能表示"连续同字符也是一条合法路径"的情况。

3. **核心递推框架**：前向变量α[t][s]代表"到第t帧停在位置s的所有合法路径概率和"，每帧可以停留在原地、往右推进一步、或（在某些条件下）跳过一个blank推进两步。

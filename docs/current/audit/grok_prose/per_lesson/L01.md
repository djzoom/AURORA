## 更改建议（按优先级）

### P1 · 影响理解或预期

#### S1. 标题信息过载，首屏抓不住「这课到底干什么」
- **现状**：标题同时塞了「拒绝黑盒 / 正弦波 / Whisper / 6 个月远征」。
- **问题**：学生打开第一课，扫一眼未必立刻知道主任务是动机+路线图+环境自检。
- **建议**：标题可收为更贴内容的一句，例如：  
  `第1课 · 拒绝黑盒——Aurora 的动机、6 个月路线图与环境自检`  
  正弦波保留为正文钩子即可；或副标题拆成两行。

#### S2. §3 四个系统 vs §5 六月通关：命名略拧
- **现状**：§3 交付物是 Audio / KWS / ASR / Podcast Intelligence；§5 Month 4 写「LM Core」，Month 5 才是 Podcast Intelligence。
- **问题**：第一课两张「目标地图」不完全同构，学生可能问「第 4 个系统到底是 LLM 还是 Podcast？」。
- **建议**：统一命名，或加一句：「§3 的四个系统是终局作品；§5 按月份拆里程碑（Month 4 打 LM 地基，Month 5 拼进 Podcast Intelligence）」。

#### S3. 「11 个逻辑模块装进 11 个文件夹」一句略易误读
- **现状**：  
  > 11 个逻辑模块装进 11 个文件夹，唯一的"合并"发生在开头……
- **问题**：读起来像「一一装进」，紧接着又说 ① 被拆进 `0_foundation` 与 `1_complex_trig`——学生要二次校正。
- **建议**改为：  
  > 磁盘上有 11 个文件夹；逻辑上有 11 段知识。**只有开头不对齐**：① 前半住在 `0_foundation`，① 后半与 ② 合住 `1_complex_trig`；其余 ③–⑪ 与文件夹基本一一对应。

### P2 · 行文节奏与跟读负担

#### S4. 第一课篇幅偏「全书前言」
- **现状**：动机 → 原则 → 四系统 → 11 模块 → 文件夹对照 → 六月通关 → 缩写词典 → 写目的 → 讲明白表 → Notebook 用法 → check_imports → 字典速成 → 冒烟 FFT → environment_report → 星号 → 三目标 → 收束。
- **问题**：概念都对，但一节课认知切换过多；容易「读得很认真却记不住今天必须带走什么」。
- **建议**（任选其一，不必全做）：
  1. 开篇学习目标下加 **「本课 30 分钟最小路径」**：读 §1–§4 → 跑正弦波 → 写 3 个目标 → 做两个函数；其余当地图收藏。
  2. 把 §7「讲清楚的能力」大表挪到附录或收束附近，避免与 §5 通关表连续两张「未来术语墙」。
  3. Notebook 快捷键表（§8）可再缩短，或链到 README，把注意力留给动机。

#### S5. 中英夹注密度在首课偏高
- **现状**：如「异步流水线（asynchronous pipeline）」「自动微分（automatic differentiation，autograd）」等在路线图里连续出现。
- **问题**：术语规范好，但第一课扫地图时括号会打断中文节奏。
- **建议**：路线图只保留**本课会碰到**或**后文立刻用到**的英文（如 FFT、Whisper）；其余首次出现用中文，英文放到各模块开篇再立。

#### S6. §1 示例代码用 `openai` API，与后文「HuggingFace 可作推理框架」边界可更贴
- **现状**：用 OpenAI Whisper API 两行示例打脸「两行就会」——修辞有效。
- **小建议**：加半句「换成本地 `pipeline` 黑盒也一样」，避免有人以为「只要不用 OpenAI 就算从零」。

### P3 · 润色与一致性（可选）

#### S7. 小节编号 `9` / `9.5`
- **建议**：冒烟测试可改为「§10」，后面顺延；或明确写「§9 的延伸：冒烟测试」，减少「半节」陌生感。

#### S8. 「旧目标（错误）：10000 个 commit…」
- **语气**：有力，但对已经刷 commit 的学习者略刺。
- **建议**：改为「常见的假目标 / 容易误判的 KPI」，意思不变，摩擦更小。

#### S9. 钥匙 3 中「高中的 sin 输入的是角度」
- **小事实**：高中也学弧度；说「更常先遇到角度制」更稳。
- **建议**：`高中更常先见到角度制（sin 30°=0.5），而 np.sin 吃的是弧度……`

#### S10. 练习「参考实现在 `<details>`」与「在 solutions/*.md」
- **现状**：§8 写折叠块，§6 指向 `solutions/L01_....md`。
- **建议**：统一一句：「本课文字示范与函数参考都在 `solutions/`；部分课内答案在 `<details>`。」避免学生两边找。

#### S11. 桥接语略重复
- **现状**：cell 1 预告 L02 + 文末再预告一次（内容接近）。
- **建议**：文首保留一句导航；文末写「带走什么 + 下节四个函数名」即可（文末已有，文首可更短）。

#### S12. `out[3]=2` 的对称性
- **现状**：说明「不是你算错」很好，但「镜像」对零基础仍飘。
- **建议**加半句白话：  
  > 对实数信号，频谱后半段重复报告前半段的信息（以后叫共轭对称）；今天只要知道 out[3] 有值是正常现象。

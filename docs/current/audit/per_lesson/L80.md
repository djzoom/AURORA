## 问题列表

### 🔴 Critical

**[Runability]** — cell-13 中 `assert avg_ms < 1.0` 在 `find_similar` 未实现时会以 `AssertionError: 检索太慢：nan ms` 崩溃，给出完全错误的错误信息。原因：`find_similar` 返回 `None` 时循环 `break` 退出，`times = []`，`np.mean([]) = nan`，`nan < 1.0` 为 `False`，断言触发。同时 NumPy 还会抛出 `RuntimeWarning: Mean of empty slice`。

Fix：在 assert 之前添加 guard：
```python
if not times:
    print("⬜ find_similar 未实现，跳过性能断言")
else:
    avg_ms = np.mean(times) / 1000
    print(f'\n平均检索时间: {avg_ms:.3f} ms')
    assert avg_ms < 1.0, f'检索太慢：{avg_ms:.3f} ms'
```

### 🟠 Major

**[Exercise Design]** — 实现骨架以 `return None` 结尾而非 `raise NotImplementedError()`。这带来两个问题：(1) 学生若只完成部分 TODO 并忘记删除末尾 `return None`，函数会静默返回 None 而非报错；(2) 与课程其他 notebook 中"骨架应抛 NotImplementedError"的惯例不一致。

Fix：将 `return None` 改为 `raise NotImplementedError("请实现 find_similar")`。

### 🟡 Minor

**[Exercise Design]** — 无零向量边界情况测试。若 `query_emb = np.zeros(d)`，朴素的 `q / np.linalg.norm(q)` 会产生 `NaN`（除以零），后续所有 scores 均为 NaN，静默返回错误结果。参考实现（`solutions/L80_similarity_solutions.md`）使用 `+1e-12` 的 epsilon 保护，但 notebook 正文未提及此保护的必要性，验证测试也未覆盖该边界情况。

Fix：在 cell-11 末尾增加：
```python
z = np.zeros(4)
idx_z, s_z = find_similar(z, embs, top_k=2)
assert not np.any(np.isnan(s_z)), '零向量查询不应产生 NaN'
print('✅ 零向量边界检查通过')
```

## 概述

L19 uses the `laviz` library to demonstrate matrix multiplication from four viewpoints (4-ways view, inner/outer products, mat-vec as column linear combination, mat-mat as rank-1 sum). The Aurora cross-references are accurate: `transforms.py` line 64 (`twiddle @ x`) and `mel_filterbank` in `mel.py` are both verified against the actual source. However, the title explicitly promises `laviz` visualizations of rotation, scaling, and shear matrices, and none appear — all laviz calls use abstract generic matrices. ## 问题列表

### 🔴 Critical

**[Exercise Design]** — The reflection test case `Refl = np.array([[1., 0.], [0., -1.]])` is simultaneously a diagonal matrix (`A[0,1] == 0`, `A[1,0] == 0`). The spec says "scaling = 对角矩阵（`A[0,1] ≈ A[1,0] ≈ 0`）" with no priority note. A student who checks diagonality first — a natural reading since it is the simplest condition — returns `'scaling'` for `Refl` and the assertion `classify_transform(Refl) == 'reflection'` fails with `反射矩阵识别错误: scaling`. The hint says "先算 det 和 A@Aᵀ" which implies orthogonality first, but the spec bullet order lists rotation → reflection → scaling → shear without calling out the priority conflict explicitly. Verified by running both orderings: diag-first gives `'scaling'`; ortho-first gives `'reflection'`. Fix: Replace the test case with an unambiguously non-diagonal reflection, e.g. `Refl = np.array([[0., 1.], [1., 0.]])` (reflection across y=x, det=-1, non-diagonal), or add an explicit ordering note: "必须先判断正交性（步骤1-2），再判断对角（步骤3），否则 `[[1,0],[0,-1]]` 会被误判为缩放".

## 问题列表

### 🔴 Critical

**[数学准确性]** — Cell 11 的参考输出与信号不匹配，有 4/8 个系数错误。
  
  Notebook 声称 `np.fft.fft([1, 2, 3, 4, 3, 2, 1, 0])` 输出：
  ```
  [16+0j, -4+0j, 0+0j, 0+0j, 0+0j, 0+0j, 0+0j, -4+0j]
  ```
  实际运行结果：
  ```
  [16+0j, -4.828-4.828j, 0j, 0.828-0.828j, 0j, 0.828+0.828j, 0j, -4.828+4.828j]
  ```
  产生 notebook 中所示"全实数"输出的信号应是余弦型 `2 - cos(2πn/8)` ≈ `[1, 1.293, 2, 2.707, 3, 2.707, 2, 1.293]`，而非线性折线 `[1,2,3,4,3,2,1,0]`。X[1]、X[3]、X[5]、X[7] 的实部和虚部均与事实相差 20% 以上。学生运行正确实现后对比此参考，会误判自己的代码有误。

  **Fix**: 将 cell 11 的示例信号替换为真正对称的余弦型（与参考输出一致），或保留 `[1,2,3,4,3,2,1,0]` 并更新为正确的参考输出（全复数）。

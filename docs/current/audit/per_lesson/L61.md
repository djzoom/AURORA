**总体评级**: B- | **问题数**: 1 critical / 2 major / 5 minor

## 概述

L61 covers the right conceptual ground for a first `nn.Module` lesson: parameter registration, `nn.Sequential`, `named_parameters()`, and `state_dict()`. The math is correct throughout (parameter counts are hand-verified accurately). However, the notebook cannot be run top-to-bottom in its stub state — the TODO cells silently return `None` and the immediately-following verification asserts raise a cryptic `AttributeError` rather than a clean `NotImplementedError`. Additionally, three cells contain stale "a4_mlp" references that should have been updated to the current lesson number `L57` when the course was renumbered.

## 亮点

- Parameter count walkthrough in Section 5 is concrete and correct: 2624 + 650 = 3274 for (40→64→10), and the hidden=128 variant (6538) is also verified in code. Students who do the hand-calculation will gain genuine intuition.
- The `TinyNet` demo (cell-4) elegantly contrasts all three registration modes in one class: auto-registered via `nn.Linear`, manual via `nn.Parameter`, and untracked plain tensor. This is the clearest single cell in the lesson.
- The `nn.Sequential` scope note ("有残差连接时必须回到手写 forward()") is accurate and useful — it preempts a common misuse students encounter in the very next lesson unit (L63 KWS model).
- The `solutions/` directory exists with a correct reference implementation (`solutions/L61_pytorch_nn_solutions.md`), so the hint in cell-9 is valid.

## 问题列表

### 🔴 Critical

**[Runability]** — The notebook fails top-to-bottom execution starting at cell-11. The `AudioMLP` stub in cell-10 has `pass` in both `__init__` (no `self.layers` assigned) and `forward` (returns `None`). When cell-11 runs:
```python
out = m(torch.randn(8, 40))   # out = None (no error yet)
assert out.shape == (8, 10)   # AttributeError: 'NoneType' object has no attribute 'shape'
```
Cell-13 then fails independently: `m.parameters()` yields nothing (total=0), so `assert total == 3274` raises `AssertionError`. A TA running the notebook against a student submission, or a student who re-runs all cells after editing cell-10, gets two separate cascading errors. Fix: replace both `pass` stubs with `raise NotImplementedError("TODO")` so the failure is immediate, localized, and self-explanatory. Consider also wrapping cell-11 and cell-13 in `try/except NotImplementedError` guards that print a reminder instead of traceback-bombing.

### 🟠 Major

**[Course Continuity — Stale Reference]** — "a4_mlp" appears three times (cell-0 目标, cell-12 Section 5 markdown, cell-14 本课收束) and refers to the hand-written MLP that is now `L57_mlp.ipynb`. The course was renumbered from an "a*" assignment scheme to L-numbers, but these references were not updated. Students who search for "a4_mlp.ipynb" find nothing in the repo. Fix: replace all three occurrences with "L57" (e.g., "复现 L57 中手写的 MLP", "与手写 L57 的同构网络完全一致").

**[Pedagogical Gap — register_buffer omitted]** — `TinyNet.buf` is presented as a contrast ("未注册，仅普通张量") but the lesson never explains `register_buffer()`, the correct idiom for non-parameter state (e.g., running mean in BatchNorm). This creates a half-told story: students learn what NOT to do but not the right alternative. The gap matters because `state_dict()` DOES serialize registered buffers (which would appear in checkpoint files) but NOT plain tensors — a distinction that bites learners in Month 2 KWS checkpointing. Fix: add a two-line demo showing `self.register_buffer('buf', torch.zeros(1))` and note that `buf` then appears in `state_dict()` but NOT in `parameters()`.

### 🟡 Minor

**[Exercise Design — objective not numbered]** — The "目标" in cell-0 is a single run-on sentence. Best practice in this course (e.g., L59, L60) is 3–4 numbered objectives. Fix: split into e.g. (1) implement AudioMLP with nn.Sequential, (2) verify parameter counts match hand calculation, (3) round-trip a state_dict to a fresh model.

**[Exercise Design — no edge-case tests]** — The verification in cell-11 only checks `(8, 10)`. Fix: add `assert m(torch.randn(1, 40)).shape == (1, 10)` (batch=1) and optionally a shape mismatch check to confirm the layer rejects `(8, 41)`.

**[Accuracy — torch.load security parameter]** — The markdown code block in Section 2 shows:
```python
model.load_state_dict(torch.load('ckpt.pt'))
```
Since PyTorch 2.0 this emits a `FutureWarning` and since 2.6 it raises an error unless `weights_only=True` is passed. Students who copy this snippet will get confusing warnings. Fix: update to `torch.load('ckpt.pt', weights_only=True)` and add a one-line note about why.

**[Code Clarity — double-negative print]** — Cell-4 prints:
```python
print('buf 未出现:', 'buf' not in param_names)
```
This will print `buf 未出现: True`, which reads awkwardly (negation inside negation). Fix: `print('buf 已被排除于 parameters() 之外:', 'buf' not in param_names)` or simply assert and omit the print.

**[Pedagogical Quality — state_dict vs parameters() distinction unclear]** — The current `state_dict` demo in cell-6 only shows keys from `named_parameters` (fc.weight, fc.bias, scale). Since `buf` is a plain tensor and not a registered buffer, it also won't appear in `state_dict`. But the lesson never makes this explicit, leaving students uncertain about the relationship between `parameters()` and `state_dict()`. Fix: add a one-liner `assert 'buf' not in tiny.state_dict()` with a comment explaining the rule.

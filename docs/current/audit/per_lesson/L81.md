## 问题列表

### 🔴 Critical

**[Accuracy / Runability]** — `isinstance(r, int)` 断言会使正确实现失败。

Cell 11 中：
```python
assert all(isinstance(r, int) for r in recs), "推荐 id 应为 int"
```
`np.argsort()` 返回 `np.int64` 元素；Python 3 中 `isinstance(np.int64(5), int)` 返回 `False`（已在项目环境验证）。学生若按自然方式实现 `[i for i in np.argsort(scores)[::-1] if ...]`，断言将失败，即便逻辑完全正确。

**Fix**：将断言改为 `assert all(isinstance(r, (int, np.integer)) for r in recs)` 或在返回前显式转换 `[int(i) for i in ...]`，并在 docstring 中说明返回类型要求。

---

**[Accuracy]** — EnCodec 压缩比数值错误：markdown 写 640×，代码算出 160×，差 4 倍。

Cell `2a5fd2cc`（markdown）写道："压缩比约 **640×**"。Cell `4f931c33`（代码）计算：
```
n_samples = 32000,  total_tokens = 50 frames × 4 codebooks = 200
compression = 32000 / 200 = 160.0
```
代码打印 "压缩比: 160.0×"，与 markdown 自相矛盾。错误原因：markdown 写 640 时漏掉了 4 个码书（32000/50=640 是每帧采样点数，不是整体压缩比）。

**Fix**：将 markdown 中"压缩比约 640×"改为"压缩比约 160×（32000 采样点 / 200 token）"，或将计算式调整为 bit-level 压缩比并注明口径。

**[Accuracy]** — 冷启动策略 A 的表述轻微不准确。

Section 2 说"全局均值 user_emb 等价于按 popularity score 排序"，但二者不等价：全局均值向量取最近邻是按嵌入空间几何中心排序，不是按播放计数排序。嵌入空间中最靠近中心的歌曲和最受欢迎的歌曲通常不同。

**Fix**：将该说法改为"全局均值作为 user_emb 等价于'用户对所有歌曲一视同仁'——一种无先验的中性出发点，而非流行度排名"。

### 💡 建议

- **增加 Recall@K 指标**：当前实验仅计算 Precision@5。加入 Recall@K（或 F1@K）能更完整地评估推荐系统，与工业界标准对齐。
- **多样性 penalty 讨论**：推荐系统的另一个常见挑战是 over-concentration（所有推荐都来自同一簇），可用一行 MMR（Maximal Marginal Relevance）公式展示多样性-相关性权衡，作为拓展思考题。
- **`jazz_hits >= 3` 阈值说明**：该阈值在随机 seed 下是否稳定可以简要说明，避免学生误认为阈值过松。

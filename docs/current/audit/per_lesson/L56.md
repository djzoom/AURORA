## 概述
L56 has a genuinely well-structured conceptual scaffolding: three progressive concept cells (topological sort → seed gradient → accumulation with `+=`) flow naturally into the implementation exercise. The numerical-differentiation validation section is a course highlight. However, three significant problems undermine the lesson's value: the solution to the sole exercise is already fully visible in L55's base `Value` class; the claimed `aurora/train/` module does not exist in the codebase; and the notebook raises `AssertionError` in Cell 11 when run top-to-bottom with the unimplemented stub.

## 亮点
- **Progressive concept arc**: Cells 3→5→7 build exactly the right mental model before the exercise — sort order → seed → accumulation — each with a short runnable demo that makes the concept tangible.
- **Numerical gradient verification** (Cells 12–13): Using central differences to independently validate `backward()` is excellent pedagogy; the formatted table with per-variable error and pass/fail flags is clear and informative.
- **Diamond-graph demo** (Cell 8): Concretely showing `a * a` produces `grad = 4.0` via `+=` in a single cell is the right way to motivate accumulation; no ambiguity.
- **Solutions file is correctly separated**: `solutions/L56_backward_pass_solutions.md` exists and is not leaked into the notebook itself.

### 🟠 Major

**[Course Continuity / Exercise Design]** — L55 fully spoils the L56 exercise. L55 Cell 2 embeds the complete working `backward()` implementation (topo DFS + seed + reversed loop) as part of the base `Value` class students must read to complete L55's own exercises. The code there is character-for-character identical to `solutions/L56_backward_pass_solutions.md`. A student who has done L55 as intended will arrive at L56 having already read the solution. Additionally, L55's "本课收束" incorrectly describes L56 as a lesson about "梯度消失" (gradient vanishing), which L56 never covers.  
Fix: In L55, provide `backward()` as a stub with `raise NotImplementedError` (like the operators), and add it to L55's exercise set — OR accept that L56 is a guided-reconstruction lesson and reframe its 学习目标 explicitly as "reconstruct from understanding, not from memory."

**[Runability]** — Cell 11 will raise `AssertionError` (`a.grad` is 0.0, expected 3.0) when the notebook is run top-to-bottom with the unimplemented stub. Cell 13 (numerical diff) will also silently produce wrong output. This makes it impossible for a TA or CI system to run the notebook end-to-end in "assignment-distributed" state.  
Fix: Wrap Cell 11's assertions in a `try/except AssertionError` block that prints a friendly TODO message, or add a guard: `if not (L.backward.__code__.co_code == backward.__code__.co_code): raise SkipTest(...)`. At minimum, add a comment `# ⚠️ Run this cell AFTER implementing backward()`.

**[Accuracy — Aurora Connection]** — Cell 14 ("本课收束") states: "此机制直接对应 `aurora/train/` 模块的参数更新步骤：`w.data -= lr * w.grad`." No `aurora/train/` module (or `aurora/train.py`) exists in the repository. The actual package structure has `audio`, `music`, `llm`, `mlops`, `rag`, `speech`, `realtime`, `multimodal`, `tts`, `research` — nothing named `train`.  
Fix: Either create a stub `src/aurora/train.py` that documents the parameter-update interface, or change the reference to an existing module (e.g., `aurora/mlops/`) with an accurate description.

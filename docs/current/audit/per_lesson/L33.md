## 概述

本课结构清晰，"观察中间量 → 公式拆解 → 手写实现 → 对照验证 → 可视化"的流程合理，是正弦波生成的合格入门课。然而存在两个关键缺陷：stub 返回 `None` 会让验证单元在学生动手前即以 `TypeError` 崩溃，以及 notebook 教导的 `round()` 与仓库实现的 `int()` 语义不同，测试用例恰好掩盖了这一差异。此外有一处错误的源文件路径引用和一处与代码不符的 Nyquist 说明。

## 亮点

- 三层公式拆解（编号 → 角度 → sin 投影）配合"圆周上的影子"比喻，直观且正确。
- `n/sample_rate` vs `np.linspace` 的取舍理由（舍入噪声）写得明确，和仓库代码意图一致。
- 对照验证单元同时检查 shape 和逐点误差，引导学生看"为什么对/不对"而非只看通过/报错。
- Nyquist 预告（L34 混叠）与课程序列衔接正确，下一课引用准确。

## 问题列表

### 🔴 Critical

**[Runability / Exercise Design]** — `my_sine` stub 返回 `None`，导致验证单元 (`8617b5a4`) 在 `np.max(np.abs(mine - ref))` 处以 `TypeError: unsupported operand type(s) for -: 'NoneType' and 'numpy.ndarray'` 崩溃，学生在尚未实现任何代码时就遇到无意义的堆栈报错。  
Fix: 将 `return None  # ← 改这里` 改为 `raise NotImplementedError("TODO: 按公式实现并返回正弦波")`。

**[Accuracy / Exercise Design]** — Notebook 教导 `N = round(duration * sample_rate)`，但仓库实现 `aurora.audio.sine`（`src/aurora/audio/io.py` 第 61 行）使用 `int(duration * sample_rate)`。`int()` 截断，`round()` 四舍五入；当乘积为半整数时（如 `duration=0.5, sr=3` → `int(1.5)=1` vs `round(1.5)=2`）两者结果不同，数组长度不一，`np.allclose` 将触发形状断言失败。测试用例 `dur=1.0, sr=16000` 恰好产生精确整数，掩盖了这一分歧，使学生误以为自己的实现等价于仓库。  
Fix: 统一为 `N = int(duration * sample_rate)`（与仓库一致），或修改仓库使用 `round()`，并补充一个非整数乘积的边缘测试（如 `duration=0.5, sample_rate=3`）。

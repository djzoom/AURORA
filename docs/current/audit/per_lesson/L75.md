## 概述

L75 是一节纯演示型可视化课，内容聚焦于 CTC lattice 路径枚举和 Whisper cross-attention 热力图，两个可视化都切题且图表质量较好。
然而 markdown 中第一条 CTC 路径示例存在事实性错误（H H I - I 解码结果为 "HII" 而非 "HI"），会主动误导学生对 CTC 解码规则的理解。
此外，本节完全缺失学习目标章节和学生练习，与同系列其他课程相比教学结构明显偏弱。

## 亮点

- `ctc_decode` 实现干净正确：先去连续重复再去 blank，逻辑清晰可作参考实现。
- CTC lattice 热力图用节点频次着色的设计直觉性强，`V^T = 729` 总路径对比合法路径数的打印有助于建立量级感知。
- 参数实验（spread / noise_level）+ 熵指标联动的设计是全课最有新意的交互设计，把定性描述（"对角线变宽"）和定量指标（平均注意力熵）结合在同一张图中。
- Aurora 连接引用 `aurora.audio.mel`，该模块在 `src/aurora/audio/mel.py` 中确实存在，路径有效。
- 下一课指向 L76（音乐理论基础）符合实际课程目录（`notebooks/8_music/L76_music_theory.ipynb`）。

## 问题列表

### 🔴 Critical

**[数学准确性]** — markdown 中第一条 CTC 示例路径 `H H I - I` 解码错误。
按 CTC 规则：去连续重复得 `[H, I, -, I]`，再去 blank 得 `[H, I, I]` = "HII" ≠ "HI"。
这是 markdown 说明与 CTC 规则的直接矛盾，会使学生从一开始就建立错误的心智模型。
Fix：将该路径替换为合法示例，例如 `H H - I -`（去重 → `H - I -`，去 blank → `HI`）。

### 🟠 Major

**[数学准确性 / 课程一致性]** — T 标注不一致：markdown 标题写"路径长度 T=5 示例"，
给出的 4 条路径确实都是长度 5，但代码单元格使用 `T = 6`。
学生看图后尝试在代码中复现 T=5 路径会发现参数不匹配。
Fix：统一为 T=6（更新 markdown 标题），或将代码改为 T=5 并补充一条 T=5 的第五个路径示例。

**[教学质量]** — 全课缺少 `## 学习目标` 章节（同系列其他课均有编号目标）。
Fix：在 cell-0 之后加入明确的三条学习目标，例如：
① 能说明 CTC 解码规则并手工追踪路径；
② 能解读 cross-attention 热力图与幻觉风险的关系；
③ 能通过注意力熵量化对齐质量。

**[练习设计]** — 整个 notebook 没有任何 TODO 练习单元格。
作为第 75 课，即便是"可视化"课也应有至少 1 条动手练习（例如：让学生自己实现 `ctc_decode`，或计算真实注意力矩阵的平均熵并与随机基线对比）。
Fix：在 Section 1 末尾增加一道练习：给定 vocab 和 target，手写 `ctc_decode` 并用下方给出的测试用例验证。

**[可运行性 / 可复现性]** — `make_attn` 函数通过闭包依赖模块级变量 `rng2`，
而 `rng2` 在 cell-8 顶部以 `np.random.default_rng(0)` 初始化。
cell-8 中 `make_attn` 被调用了两轮（第一个 for 循环生成图，第二个 for 循环打印熵），
两轮使用的是连续推进的 rng 状态，导致图中显示的注意力矩阵与底部打印的熵值来自不同随机采样，数值不对应。
Fix：在每个 `make_attn` 调用内部使用独立的 `np.random.default_rng(seed)` 使结果幂等，
或在每个调用点缓存结果：`results = [make_attn(sp, nl) for sp, nl, _ in configs]`，然后两个循环都用 `results[i]`。

### 🟡 Minor

**[可运行性]** — `from itertools import product` 在 cell-4 内部导入，
而 cell-2 是统一 import 块。非标准位置对 notebook 重排运行有影响。
Fix：将该 import 移至 cell-2。

**[课程叙述]** — "本课收束"写 "`ctc_decode` 枚举了所有合法 CTC 路径"，
但 `ctc_decode` 只是单路径解码函数；枚举是 `itertools.product` + 列表推导完成的。
Fix：改为"本节通过暴力枚举 `product(range(V), repeat=T)` 并以 `ctc_decode` 过滤，列出了所有合法 CTC 路径"。

**[数值注释]** — cell-6 注释 `# encoder 帧数 (mel 帧，约 25ms/帧)` 混淆帧宽（25ms window）
和帧间距（10ms hop）。Whisper 标准参数下帧间距为 10ms，不是 25ms。
Fix：改为 `# mel 帧数 (帧宽 25ms，hop 10ms，即每秒约 100 帧)`。

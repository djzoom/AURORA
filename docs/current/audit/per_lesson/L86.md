## 概述

L86 covers the three core LLM decoding strategies cleanly and the mathematical content is correct. The reference implementation in `aurora.llm.sample` is solid. However, a critical runability defect — the visualization cell (cell 8) calls both unimplemented stubs inside list comprehensions with no exception guard — means the notebook crashes immediately on a clean run, before the student even reaches the test cell. Additionally, the verification logic for `top_p` is weak, and the cross-comparison with `aurora.llm` silently skips `top_p` entirely.

## 亮点

- The `softmax` implementation in cell 3 is numerically correct (max subtraction, temperature clipping at 1e-8) and makes a good self-contained demo before the exercises.
- The `top_k_sample` stub is well scaffolded: type coercion, k-clamping, and a clear step-by-step TODO hint are all present. Raises `NotImplementedError` correctly rather than returning `None`.
- The `top_p_sample` stub does meaningful pre-work (sorting by probability descending) before the TODO, leaving exactly the right amount for the student.
- The summary table in "本课收束" is concise and accurate.
- `aurora.llm` module exists and exports both `top_k_sample` and `top_p_sample` — the reference import in cell 9 is valid.

## 问题列表

### 🔴 Critical

**[Runability]** — Cell 8 (visualization) calls `top_k_sample` and `top_p_sample` inside list comprehensions (`[top_k_sample(...) for _ in range(500)]`) with no try/except. Both functions raise `NotImplementedError` until the student implements them. The entire cell crashes before any plot is drawn, and crucially it crashes _before_ the test cell (cell 9) which does have graceful error handling. A student running the notebook top-to-bottom gets a raw traceback mid-notebook with no guidance.

Fix: Wrap the `strategies` list construction in a try/except NotImplementedError block, or gate the visualization cell with a pre-check (e.g., `try: top_k_sample(logits[:2], k=1) except NotImplementedError: ...`). Alternatively, move the visualization cell after cell 9 so it acts as a reward for completing both exercises.

### 🟠 Major

**[Verification / Correctness]** — The `top_p_sample` test in cell 9 only checks that the returned value is a valid integer index in `[0, len(test_logits))`. It does NOT verify that the result lies within the nucleus set, unlike the `top_k` test which correctly checks membership in `top_k_indices`. A student could return `0` unconditionally and pass the `top_p` test.

Fix: After calling `top_p_sample`, reconstruct the nucleus the same way aurora.llm does (argsort descending, cumsum, cutoff) and assert the result is in `nucleus_idx`.

**[Verification / Completeness]** — Cell 9 imports `ref_topp` from `aurora.llm` but never uses it. The "与 aurora.llm 对比验证" section only tests `ref_topk`. The final `print('✅ 采样策略验证通过')` fires after only verifying top-k against the reference, so the victory message is misleading.

Fix: Either add `ref_result_p = ref_topp(test_logits, p=0.9)` and assert it lands in the correct nucleus, or remove the unused import and narrow the victory message to "top_k 验证通过".

**[Course Continuity / Code Duplication]** — `softmax` is implemented from scratch in cell 3 (with a new `temperature` parameter) without any acknowledgment of L30's `softmax` exercise or the production implementation in `aurora.llm.sample.softmax`. The two implementations are not cross-referenced and the L86 version will silently diverge if the aurora.llm version is updated. This undermines the "build once, reuse everywhere" ethos.

Fix: Add a one-line comment: `# 完整实现见 aurora.llm.sample.softmax（L30 引入）`; or import and reuse it directly: `from aurora.llm import softmax`.

### 🟡 Minor

Fix: Add two or three targeted assert lines in cell 9 covering k=1 and p=1.0 edge cases.

**[Code Quality]** — `ref_topp` is imported but unused (see Major issue above). Linters will flag this as F401.

Fix: Remove the unused import once the top_p cross-comparison is added.

**[Pedagogical Quality]** — There is no explicit "Aurora连接" section. The only reference to the real codebase is a few lines buried inside the try/except test block in cell 9. Students may not realize this notebook maps to `src/aurora/llm/sample.py`.

Fix: Add a brief "Aurora 连接" cell pointing to `src/aurora/llm/sample.py` and noting that the production `top_p_sample` also handles the `cutoff = max(1, ...)` edge case they may need to handle.

### 💡 建议

## 问题列表

### 🔴 Critical

**[Course Continuity]** — The 本课收束 is internally contradictory on L-numbers. In the same paragraph it says "是后续 **L39** FFT（蝶形运算）的算法基础" and then one sentence later "下一课（**L38**）把 DFT … 加速为 … 蝶形分治（butterfly）". Directory inspection confirms `L38_fft_butterfly.ipynb` is the butterfly lesson and `L39_fft_implement.ipynb` is the full implementation. The "L39 FFT（蝶形运算）" label is wrong; it mislabels the butterfly lesson one number too late.
- **Fix**: Change "后续 L39 FFT（蝶形运算）和 L43 STFT（短时滑窗）" → "后续 L38 FFT（蝶形运算）、L39 FFT 完整实现 和 L43 STFT（短时滑窗）", then remove the redundant final sentence that re-states L38.

### 🟠 Major

**[Exercise Design — answer leaked in hints]** — The TODO comments in cell-12 give away the entire implementation:
```python
# ✏️ TODO: 计算旋转因子 twiddles = exp(-2*pi*i*k*n/N)
# ✏️ TODO: X[k] = dot(x, twiddles)
```
These are not hints — they are the two lines of code the student needs to write. The only cognitive step remaining is "remove the `#` character". A student learns nothing about how to derive the twiddle factor formula.
- **Fix**: Replace the inner-body comments with a docstring hint that describes *what* to compute without naming the variables or formula: e.g., "for each frequency index k, accumulate x[n] weighted by the corresponding complex exponential evaluated at n=0..N-1."

**[Runability — timing cell produces misleading output with unimplemented stub]** — Cell-15 (O(N²) timing experiment) calls `naive_dft` unconditionally and prints ratio comparisons regardless of whether the function is implemented. With the stub, the loop executes `pass` N times; all timings are near-zero and nearly equal, so the ratios will be random noise around 1x rather than 64x/16x. A student who runs the notebook top-to-bottom before completing the exercise sees confusing output with no warning.
- **Fix**: Add a guard at the top of cell-15:
  ```python
  if not np.any(naive_dft(np.array([1.0, 0.0]))):
      print("⬜ 请先实现 naive_dft 再运行计时格"); raise SystemExit
  ```

**[Exercise Design — stub returns wrong sentinel for edge inputs]** — The stub returns an all-zeros array (shape `(N,)`). The check `not np.any(_out)` in cell-13 works for the provided test vector `[1,2,3,4]` (whose correct DFT has non-zero components), but would falsely pass for a student implementation that returns the correct DFT of an all-zeros input or any future edge-case test added later. Convention should be `raise NotImplementedError`.
- **Fix**: Replace `pass` inside the loop with `raise NotImplementedError("TODO: implement naive_dft")` and move it before the loop. Update cell-13 to catch `NotImplementedError` instead of checking `not np.any(_out)`.

**[Course Continuity — title says "双循环" but scaffold shows inner-vectorized implementation]** — The lesson title advertises "O(N²) 双循环" but the scaffold (and the Aurora `dft()` reference) are single-loop with the inner dimension vectorized via `np.dot`. The Aurora reference implementation goes further, using a full matrix multiply (`twiddle @ x`) with no Python loop at all. The mismatch between title and actual implementation style could confuse students comparing their loop to the reference.
- **Fix**: Either update the title to "O(N²) 单层循环 + 内层向量化" or add a note explaining the three equivalent implementations (pure double-loop → inner-vectorized → full matrix) and why they are all O(N²) in arithmetic complexity.

**[Exercise Design — conjugate symmetry loop at k=N//2 unexplained]** — Cell-10's loop includes `k = N//2 = 2`, which checks `X[2]` against `conj(X[2])`. The assertion passes because `X[N//2]` must be real for real input, but the notebook prints "True" without explaining why this special case holds. A student unfamiliar with the Nyquist bin may be puzzled.
- **Fix**: Add a one-line comment: `# 特殊情况：k=N//2 时 X[N//2] 是实数（奈奎斯特频点）`.

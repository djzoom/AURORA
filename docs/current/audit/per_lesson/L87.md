## 概述

L87 是全课程中最短的课之一，仅 7 个单元格，核心算法已完全预实现，没有任何学生 TODO 练习，直接违背了"从零实现"的学习目标。技术层面有一个贯穿全课的命名错误：课程标题和代码注释都说"INT8"，但实现使用的是 uint8（无符号，0–255），两者在量化框架的工程语境中不可互换。整体来看，本课更接近一篇简短的演示脚本，而非一节教学课。

## 亮点

- **量化公式清晰**：Cell 1 中 scale / zero_point / x_q / x_deq 四式排列整齐，学生一眼可以对照代码验证。
- **内存压缩量化直观**：7B × 4 bytes vs 7B × 1 byte 的粗算，让 VRAM 需求具体化，有说服力。
- **误差 vs 位宽对比表（Cell 4）**：从 8 bit 到 2 bit 的误差随位宽降低单调增大，表格简洁有效。
- **HuggingFace 集成处理得当**：将推理代码注释掉并放在 `try/except ImportError` 保护中，不会因依赖缺失破坏笔记本。

### 🟠 Major

**[Accuracy — 术语错误]** — 课程标题、学习目标、注释和量化公式全部称之为"INT8"，但代码使用 `np.uint8`（无符号，范围 0–255）。工程上"INT8 量化"（bitsandbytes、TensorRT、PyTorch `torch.qint8`）使用有符号 INT8（范围 −128 到 127），zero_point 通常定义在 −128 到 127 范围内，scale 的计算分母为 255（对称量化为 254）。用 uint8 实现并称其为 INT8 会在学生查阅真实框架源码时造成混淆。
**Fix**：要么改用 `np.int8` 并将量化范围改为 [−128, 127]（与工业标准一致），要么明确说明"本课使用无符号 uint8 演示原理，工业 INT8 使用有符号表示，差异在…"

**[Accuracy — zero_point 越界 Bug]** — Cell 3 中 `zero_point = int(round(-x_min / scale))` 未被约束到 [0, 255]。若 x_min 极端正（所有权重 >> 0）或 scale 极小，zero_point 可超过 255 或为负，之后 `np.clip(x_q, 0, 255)` 会把大量值截断，引入非对称、不可预期的量化误差，且该 bug 在当前测试的随机矩阵中不会触发。
**Fix**：在赋值后加 `zero_point = int(np.clip(zero_point, 0, 255))`。

**[Code Consistency — Cell 4 与 Cell 3 量化公式不一致]** — Cell 4 中的多位宽误差对比使用 `W_r = W_q_b * scale_b + W.min()`（绝对最小值偏移方案，无 zero_point），而 Cell 3 使用了带 zero_point 的仿射量化。两套公式在同一课中并存，且没有任何说明，学生会疑惑"到底哪个才是量化公式"。
**Fix**：统一使用带 zero_point 的仿射量化，或明确注释"此处为简化版（zero_point=0 对齐最小值）与完整版的区别"。

### 🟡 Minor

**[Accuracy — "使 0 精确可表示" 条件不足]** — Cell 1 注释 `（使 0 精确可表示）` 仅在原始范围跨越 0（即 x_min ≤ 0 ≤ x_max）时成立。若权重矩阵全为正值，zero_point 会被计算为负数或 0，且 0 在原始值域之外根本无需精确表示。这个说明对学生理解量化的适用场景有误导性。
**Fix**：将注释改为"当 x_min ≤ 0 ≤ x_max 时，此公式使浮点零能被精确量化为 zero_point"。

**[Runability — matplotlib 导入但从未使用]** — Cell 2 导入了 `matplotlib.pyplot as plt`，但全课没有任何绘图单元格，与其他课中量化误差曲线图的惯例不符。
**Fix**：删除 `import matplotlib.pyplot as plt`；或补充误差 vs 位宽的折线图（可视化比纯表格更直观）。

**[Pedagogical Quality — 学习目标未编号]** — 标准课程格式（参见 L85–L86）使用编号列表，本课使用无编号项目符号，风格不一致。
**Fix**：改为 `1. 理解…  2. 从零实现…  3. 测量…  4. （可选）…` 编号列表。

**[Pedagogical Quality — 误差阈值未与理论挂钩]** — Cell 3 的断言 `assert error.mean() < 0.05` 是魔数，没有解释其来源。理论上 8-bit 均匀量化的期望误差 ≈ scale/2；对于本测试的随机权重矩阵（range ≈ 8），scale ≈ 0.031，理论均值误差 ≈ 0.016，阈值 0.05 约为理论值的 3 倍，松弛余量未说明。
**Fix**：将断言改写为 `assert error.mean() < scale / 2 * 3, f"..."`，并在注释中说明 `scale/2` 是量化噪声的理论期望，3× 是保守系数。

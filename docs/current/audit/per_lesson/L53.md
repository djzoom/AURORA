## 概述

L53 是一节定位清晰的可视化课，核心直觉（DCT 能量集中、系数物理含义）表达流畅，数学公式与 `aurora.audio.mfcc` 源码完全一致，代码自顶向下可以无报错运行。然而，标题承诺的"完整 MFCC 链路图示"从未完整展示：`mfcc()` 函数被导入却从未调用，全课只演示了孤立的 `dct_ii` 步骤。结构上缺少学习目标、无任何 TODO 练习，与课程其他教学单元的规范相比明显偏薄。

## 亮点

- DCT-II 公式（Cell 3）与 `mfcc.py` 源码完全对齐，下标、归一化因子均正确，`norm="ortho"` 下 k=0 特殊缩放也标注正确。
- Cell 8 的 `dct_ii` 二维调用方式利用了源码"沿最后轴变换"的设计，形状推导注释（`(n_frames, N_MELS)` → `(N_MELS,)`）全部正确。
- `mel_spectrogram` 返回形状 `(n_frames, n_mels)` 与索引 `mel_spec[10]`（取第 10 帧）方向一致，无转置错误。
- Cell 5 物理含义表格（c[0]=总能量，c[1] 正→低频主导）与 DCT-II 基函数的数学定义严格吻合，适合初学者建立直觉。
- chirp 函数调用（Cell 8）使用关键字参数，与 `io.py` 中的签名 `chirp(f0, f1, duration, sample_rate)` 完全匹配。

## 问题列表

### 🔴 Critical

**[可运行性 / 教学完整性]** — `mfcc()` 被导入却从未调用，整节课没有任何单元格演示 `mfcc()` 端到端输出。标题声称"MFCC 图形化"，但学生只看到 `dct_ii` 孤立地作用于手动提取的单帧，从未看到 `mel_spectrogram → power_to_db → dct_ii → [:, :n_mfcc]` 全链路运行结果，也没有以热力图或时序图展示 MFCC 矩阵 `(n_frames, n_mfcc)` 的形状。
**Fix**: 新增一个代码单元格，调用 `mfcc(signal, SR)` 并用 `plt.imshow(..., aspect="auto", origin="lower")` 绘制完整的 MFCC 矩阵，标注时间轴与系数轴。

### 🟠 Major

**[课程结构]** — 无"学习目标"小节。课程标准要求在课首以编号列表形式列出具体的学习目标（例如"1. 理解 DCT-II 的能量集中原理；2. 能解释 c[0]/c[1] 的物理含义；3. 会用 aurora.audio.mfcc.mfcc 生成 MFCC 矩阵"）。本课完全跳过这一环节。
**Fix**: 在 Cell 1（核心直觉）之前插入一个 markdown 单元格，列出 3–4 条具体且可验证的学习目标。

**[练习设计]** — 零 TODO 练习。本课是纯被动观察，学生没有动手机会。对于"视觉化"课，至少可要求学生手动计算单帧 DCT（1D，4 个 bin 的小例子），或调整 `n_mfcc` 截断值观察重构误差。
**Fix**: 新增至少 1 个 TODO 单元格，例如让学生用 `dct_ii` 重构 log-Mel（`idct`，或通过 DCT 正交性 `y @ basis` 逆变换），并与原始 `log_mel_frame` 做对比，用 assert 验证截取 13 维后的重构误差 < 5 dB RMS。

**[代码一致性]** — Cell 9 关闭语句描述完整链路为 `mel_spectrogram → power_to_db → dct_ii → 截取前 n_mfcc 列`，但全书所有可视化单元格（Cells 4、6、8）均使用 `10 * np.log10(mel + 1e-10)`，且 `power_to_db` 从未被导入。学生读完本课会认为正确写法是手动 `log10`，与实际 `mfcc.py` 使用 `power_to_db(mel, top_db=None)` 不符。
**Fix**: 将 Cell 4/6/8 中的手动 `log10` 替换为 `power_to_db(mel_s, top_db=None)`，并在 Cell 2 的 import 中加入 `from aurora.audio.mel import mel_spectrogram, power_to_db`。

### 🟡 Minor

**[导入]** — `sine`（来自 `aurora.audio.io`）和 `mfcc`（来自 `aurora.audio.mfcc`）均被导入但从未使用，形成死导入。
**Fix**: 移除 `sine` 和 `mfcc` 的导入，或（推荐）在新增的演示单元格中真正调用 `mfcc()`。

**[可运行性]** — `plt.savefig("v1_dct_compression.png", ...)` 等三处均使用相对路径，文件将落入执行时的工作目录（CWD）。在不同启动目录下运行会产生位置不确定的 `.png` 文件。
**Fix**: 使用 `pathlib.Path(__file__).parent / "v1_..."` 或直接删除 `savefig` 调用（若非课程必需的产物）。

**[数值验证]** — 全课无任何 `assert` 语句。Cell 8 计算的能量保留率仅 `print` 输出，不加以断言。若学生重写 `dct_ii` 时出现归一化错误，本课完全无法检测。
**Fix**: 在 Cell 8 末尾加 `assert retention > 80.0, f"能量保留率 {retention:.1f}% 低于预期"` 作为基本健全性检验。

## 概述

L73 的核心概念解释准确，WER = (S+D+I)/N 公式正确，Cell 4 的 `_edit_ops` 动态规划实现经验证无误。但课程存在两个严重缺陷：标题承诺的 "jiwer 对比" 在整个 notebook 中完全缺失，以及 Cell 11 的验证测试存在根本性漏洞——在不断言期望 WER=0.40 的情况下，任何返回内部自洽数值的错误实现都能通过全部 assert。

## 亮点

- Cell 4 的 `_edit_ops` 实现及回溯逻辑完整正确，经实际运行确认对 ['the','cat','sat'] vs ['the','big','cat','sat','down'] 输出 D=2, WER=0.40，符合预期。
- Cell 3 对 S/D/I 三类错误的诊断含义解释准确（对话场景 S 占主导 60-70%，口音重时 D 升高，幻觉词时 I 升高）——这是本节最有教学价值的段落。
- Cell 5/6 的逐句 WER 重尾分布可视化（直方图 + CDF）思路清晰，均值与中位数对比直接体现了分布偏斜的实用含义。
- Cell 11 的测试注释（"hyp[0] 缺一个词（D=1，N=3）；hyp[1] 多一个词（I=1，N=2）"）与实际数值吻合，说明例题设计经过核验。

**[练习验证 / 断言漏洞]** — Cell 11 的验证测试不断言期望 WER 值。注释写明期望 0.400，但只 `print` 出来。实测：返回 `{'wer': 0.10, 'S_rate': 0.04, 'D_rate': 0.03, 'I_rate': 0.03, 'worst_examples': [(0, 0.10)]}` 的错误实现可通过全部三条 assert（`0.0 <= 0.10 <= 2.0`、`rate_sum == wer`、`len <= 5` 均满足），学生永远不会知道自己的 WER 算错了。
Fix: 在 assert 块末尾加：`assert abs(result['wer'] - 0.40) < 1e-9, f"WER 期望 0.400，实际 {result['wer']:.4f}"`；同时对 S_rate/D_rate/I_rate 各自断言精确值（S=0/5=0, D=1/5=0.2, I=1/5=0.2）。

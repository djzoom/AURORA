## 概述

L35 的核心数学和教学思路是扎实的：通过两个小函数 `euler(θ)` 和 `twiddle(k, n, N)` 将欧拉公式具体化为 FFT 旋转因子，符号约定正确，`aurora.aviz.twiddles()` 调用指向真实存在的模块。但笔记本有两个错误会直接误导学生：（1）两个 TODO 存根都 `return None` 而非 `raise NotImplementedError()`，导致验证 cell 抛出难以理解的 `TypeError` 而非清晰的未实现错误；（2）"本课收束"中写的 `src/aurora/audio/dft.py` 实际上不存在，正确路径是 `src/aurora/audio/transforms.py`。此外有两处旧路径/旧编号引用需要修正。

## 亮点

- **数学公式正确**：DFT 符号约定 `W_N^{kn} = e^{-2πi·k·n/N}` 以及负号含义（顺时针/分析频率）解释准确。
- **推理路线设计好**：任务一和任务二的"推理路线"以编号步骤引导，逻辑清晰，不直接给出答案。
- **`aurora.aviz` 真实可用**：`aviz.style()` + `aviz.twiddles(8)` 调用合法，`src/aurora/aviz.py` 存在并实现了相应函数。
- **参考输入输出写明**：`euler(π/2)=0+1j` 和 `twiddle(1,2,8)=0-1j` 等具体示例有助于学生自测。
- **solutions 目录真实存在**：`notebooks/5_audio_dsp/solutions/L35_euler_fft_solutions.md` 存在，提示合理。

## 概述

本课的核心论点——DFT 矩阵乘法 `W @ x` 与手写 `fft(x)` 数值完全吻合——是正确的，cell-12 的 `assert np.allclose(W @ x, fft(x))` 确实通过（实测最大误差 2.9e-15，远优于 atol=1e-10）。然而笔记本存在三个致命的结构性缺陷：cell-8 手写 mel 滤波器因边界条件处理错误产生全零或脉冲滤波器；两段 markdown（cell-5、cell-7）描述了根本不存在于紧随其后代码格中的例子（陈旧残留文字）；cell-23 是与 L21 内容完全无关的基础向量算术练习，明显是从早期课程遗留下来的。全课没有一个 TODO 习题。

## 亮点

- **DFT 矩阵正交性验证**（cell-6）：`W @ W^H / N = I` 的误差打印是直观且数学上严格正确的，配合 `W@x ≈ np.fft.fft(x)` 的交叉验证，让主张有据可查。
- **FFT 矩阵视角的核心断言**（cell-12）：`assert np.allclose(W @ x, fft(x))` 简洁有力，实测通过，确实贯通了"矩阵×向量 = FFT"这一核心洞察。
- **文件路径引用准确**（cell-11 markdown）：`src/aurora/audio/transforms.py` 和 `src/aurora/audio/mel.py` 均存在，且 `fft()`、`mel_filterbank()`、`sine()` 都已从 `aurora.audio.__init__` 正确导出，`laviz` 的三个符号也均存在。无陈旧路径。
- **本课收束**（cell-20）正确引用 L22（导数基础），与实际文件 `notebooks/3_calculus/L22_derivatives.ipynb` 吻合。
- `mat_times_vec` 可视化（cell-16）将"每个 mel 值 = 一次点积"的抽象概念具象化，是本课最有教学价值的图示。

## 问题列表

### 🔴 Critical

**[Accuracy — cell-8]** — **手写 mel 滤波器边界条件 bug，产生全零/脉冲退化滤波器。**
`lo = centers[max(0, m-1)]` 使 m=0 时 lo=cen=1.0，分母 `cen-lo+1e-9 = 1e-9`，所有 bins 被 clip 为 0。实测输出：
```
M[0] = [0. 0. 0. 0. 0.]   ← 全零行（行和 = 0）
M[1] = [0. 0. 1. 0. 0.]   ← 单点脉冲（非三角形）
M[2] = [0. 0. 0. 1. 0.]   ← 单点脉冲
```
学生将看到一个残缺的 mel 矩阵并形成错误直觉。
Fix：使用 `centers = np.linspace(0, N_fft//2, N_mels+2)`（保留端点，共 N_mels+2 个点），然后对每个 m 用相邻三元组 `(centers[m], centers[m+1], centers[m+2])`，与 `aurora.audio.mel.mel_filterbank` 的实现一致。或直接删除 cell-8，由 cell-14 的 `mel_filterbank()` 统一演示。

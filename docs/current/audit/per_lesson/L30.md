## 概述

L30 covers the softmax → cross-entropy → gradient pipeline with a mathematically correct derivation of `dL/dz_i = p_i - y_i` and a rigorous numerical gradient check. However, a critical design flaw undermines the exercise: cell-25 redefines `softmax` with a complete working implementation, overwriting whatever the student wrote in the earlier TODO stub before the final check cell runs. Both TODO stubs return `None` rather than raising `NotImplementedError`, which produces opaque `AttributeError`/`TypeError` crashes rather than guiding error messages. Section numbering is broken (duplicate "2", missing "4"), and the Aurora connection references `src/aurora/classifier/` which does not exist.

## 亮点

- **Gradient derivation is correct and well-presented.** The step-by-step derivation in cell-24 clearly unifies the `i=t` and `i≠t` cases into `p_i - y_i`, and the numerical verification in cell-25 confirms it to `atol=1e-6`. This is the strongest part of the notebook.
- **Shannon entropy interlude is mathematically sound.** The provided `shannon_entropy` and `cross_entropy_h` implementations are correct, all three key assertions pass (verified), and the H(p,q) = H(p) + D_KL decomposition explanation is accurate and relevant.
- **Numeric stability motivation is handled well.** The exp-overflow argument (`z > 709 → inf`) and the proof that subtracting `max(z)` leaves probabilities unchanged is concise and correct.
- **Softmax hint example is numerically accurate.** `z=[2,1,0] → [0,-1,-2] → exp=[1.0, 0.368, 0.135] → [0.665, 0.245, 0.090]` — all values verified.
- **L31 forward reference is correct** (`notebooks/4_probability/L31_visual_probability.ipynb` exists).

## 问题列表

### 🔴 Critical

**[Exercise Design / Runability]** — Cell-25 redefines `softmax` with a complete, working implementation, silently overwriting the student's TODO stub. Any student running cells top-to-bottom has their implementation replaced before the final "小检查" (cell-29) runs. Cell-29 therefore tests the *provided* reference implementation, not the student's, making the softmax exercise effectively unverifiable without the student noticing. Fix: rename the cell-25 helper to `_softmax_ref` and have the gradient verification use that name, leaving the student's `softmax` as the live function throughout.

### 🟠 Major

**[Exercise Design]** — Both TODO stubs return `None` instead of `raise NotImplementedError("implement me")`. When downstream cells call `softmax(z)` and receive `None`, the crash is `AttributeError: 'NoneType' object has no attribute 'sum'`; when `cross_entropy` returns `None`, `assert good < bad` raises `TypeError: '<' not supported between instances of 'NoneType' and 'NoneType'`. Neither error tells the student what to fix. Fix: replace `return None` with `raise NotImplementedError("请实现此函数")` in both stubs (cells ade60bbd and 18045a6e), and add `try/except NotImplementedError` guards to cells 12, 20, and 22 so linear reading is not interrupted.

**[Aurora Connection / Accuracy]** — Both Aurora connection cells reference `src/aurora/classifier/` as the destination for the Month 2 keyword classifier. This path does not exist; the codebase has `audio`, `speech`, `llm`, `music`, `rag`, `tts`, etc. but no `classifier` sub-package. Fix: replace `src/aurora/classifier/` with `src/aurora/speech/` (the closest existing module) or note explicitly "(module planned for Month 2)".

**[Structure]** — Section numbering is broken: there are two `## 2.` headers (the softmax exercise *and* Shannon entropy are both labeled section 2), and sections jump from 3 directly to 5 with no section 4. Fix: renumber as 1 (logits), 2 (softmax exercise), 3 (Shannon entropy interlude), 4 (cross-entropy exercise), 5 (toy classifier), 6 (gradient).

**[Pedagogical Quality]** — The notebook has no structured `## 学习目标` section. The opening "目标" is a single sentence. For a lesson covering four distinct concepts (softmax, Shannon entropy, cross-entropy loss, gradient derivation), numbered learning objectives are essential for student self-assessment and match the format used in L27–L29. Fix: add a numbered 学习目标 block: (1) implement numerically stable softmax, (2) implement CE loss, (3) explain H(p,q) = H(p) + D_KL, (4) derive and verify `dL/dz = p - y`.

### 🟡 Minor

**[Accuracy]** — The hint for the cross-entropy exercise uses `probs=[0.665, 0.245, 0.090]` (which come from `z=[2,1,0]`) and quotes loss values `-log(0.665)≈0.408` and `-log(0.090)≈2.408`. But the verification test (cell 0d09cc9e) calls `softmax([2.0, 1.0, 0.1])`, producing `probs≈[0.659, 0.242, 0.099]` and actual losses `≈0.417` and `≈2.317`. The hint numbers do not match what students will compute in the test. Fix: align the hint example to use `z=[2,1,0.1]` with its correct probabilities, or change the test to use `z=[2,1,0]`.

**[Exercise Design]** — Softmax and cross-entropy verification cells test only one happy-path input. No edge cases are checked: single-element input (should return `[1.0]`), all-equal logits (should return uniform), or extreme values (tests numeric stability claim). Fix: add at minimum `assert np.allclose(softmax([0.0, 0.0, 0.0]), [1/3, 1/3, 1/3])` and `assert np.allclose(softmax([1000.0, 0.0]), [1.0, 0.0], atol=1e-6)`.

**[Course Continuity]** — The "本课收束" says "下一阶段（梯度下降篇）会用交叉熵对参数的导数指导权重更新" as though gradient descent is a future topic. Gradient descent was already taught in L25. Fix: rewrite to "this lesson's `loss` scalar is the starting point for the backward pass in the gradient descent loop built in L25."

**[Pedagogical Quality]** — The Shannon entropy section uses `log₂` (bits) for both `shannon_entropy` and `cross_entropy_h`, while the ML cross-entropy loss used in training uses `ln` (nats). The notebook never acknowledges this difference. Students may implement `cross_entropy` with `log₂` instead of `ln` and not realize the discrepancy. Fix: add one sentence in section 3 (cross-entropy exercise): "`log` 此处指自然对数 `ln`，与前面信息熵节的 `log₂` 不同——两者差一个常数因子 `ln(2)`。"

**[Exercise Design]** — The cross-entropy verification cell only asserts `good < bad` (ordering), which would pass for any monotone-decreasing function. A trivially wrong implementation like `return 1.0 / probs[true_idx]` would pass. Fix: add a value check, e.g. `assert abs(cross_entropy(softmax([2.0, 1.0, 0.0]), 0) + np.log(0.665)) < 0.002`.

### 💡 建议

- **Batch cross-entropy**: Mention that in practice the loss is averaged over a mini-batch: `mean(-log(p[i, target[i]]) for i in batch)`. A two-line numpy implementation with `np.mean(-np.log(probs[np.arange(B), targets]))` would complete the picture for training loops.
- **Temperature naming**: The scale experiments in cells 22 and 29 are demonstrating temperature scaling (`z / T`). Naming this explicitly would give students vocabulary useful in LLM decoding (L80s) and ASR beam search.
- **Toy classifier connection**: In the speech/music/noise example (cell-20), printing the one-hot target vector `y` alongside `p` would reinforce the gradient formula `p - y` before it appears in section 5.

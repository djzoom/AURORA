### 🟠 Major

**[Accuracy / Pedagogical Quality]** — **Axis labels are swapped in cell-17 print statements.**

For a matrix `F` of shape `(n_mels=10, T=8)`:
- `z0` is computed with `axis=0` → each *column* (time frame) is normalized → `z0.std(axis=0)` has shape `(8,)`, one value per time frame.
- `z1` is computed with `axis=1` → each *row* (mel band) is normalized → `z1.std(axis=1)` has shape `(10,)`, one value per mel band.

The print labels are reversed:
```python
print('axis=0 各频段 std（期望≈1）:', np.round(z0.std(axis=0), 2))  # WRONG: 各帧, not 各频段
print('axis=1 各帧   std（期望≈1）:', np.round(z1.std(axis=1), 2))  # WRONG: 各频段, not 各帧
```
A student reading these labels learns the opposite of what the code demonstrates, directly contradicting the correct explanations in the adjacent markdown (cell-18). The comment in the code itself (`# axis=0：每帧内做标准化`) is correct; only the print text is wrong.

Fix: swap the Chinese labels:
```python
print('axis=0 各帧   std（期望≈1）:', np.round(z0.std(axis=0), 2))
print('axis=1 各频段 std（期望≈1）:', np.round(z1.std(axis=1), 2))
```

### 💡 建议

- **ddof=0 vs ddof=1 distinction**: `x.var()` uses population variance (`ddof=0`) by default. The lesson never mentions sample variance (`ddof=1`, Bessel's correction). In ML contexts population variance is standard, but a one-sentence note would prevent confusion when students later encounter `np.std(x, ddof=1)` in statistics courses.
- **Epsilon consistency**: The 2D demo cells add `1e-8` to avoid division by zero (`/ (std + 1e-8)`), but the zscore exercise stub and its test assert an exact result without epsilon. A brief note distinguishing "exact z-score for exercises" from "numerically safe z-score for production" would reduce cognitive dissonance.
- **2D zscore exercise**: Cells cell-17 through cell-19 are all lecture/demo. A follow-up TODO asking students to implement `zscore_2d(F, axis=1)` would let them apply the axis concept actively rather than just reading about it.

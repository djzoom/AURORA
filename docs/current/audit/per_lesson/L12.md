# L12 — 矩阵乘法 — 矩阵 = 线性变换，乘法 = 函数复合，手推 2×2 例子

**总体评级**: B- | **问题数**: 0 critical / 3 major / 5 minor

## 概述

L12 建立了矩阵-向量乘法的直觉，手写了 `matvec` 和 `matmul` 两个核心例程，`aurora.laviz` 可视化和 `transforms.py` 的 DFT 引用真实可用。然而，课程标题承诺的核心概念"乘法 = 函数复合"在正文中完全缺失——没有任何单元格演示 `(A@B)@x == A@(B@x)` 的性质。更严重的是，Aurora 连接对 `mel_spectrogram()` 实际代码的描述有事实性错误；`matmul` 进阶练习的提示直接给出完整的单行答案，习题挑战性归零。

## 亮点

- `matvec` 练习脚手架设计合理：`raise NotImplementedError`、明确的三步推理路线、`np.allclose` 验证断言层次清晰。
- `matmul` 测试覆盖了矩形矩阵 `(3×2) @ (2×4)`，是全课难得一见的非方阵边界测试。
- `aurora.laviz.mat_times_vec` 的可视化使用恰当（调用真实存在的函数），将"列线性组合"视角具象化，与正文"行点积"视角形成互补。
- DFT twiddle 矩阵的 Aurora 连接（`transforms.py:64` 的 `twiddle @ x`）精确对应真实代码，工程上下文扎实。

## 问题列表

### 🔴 Critical

### 🟠 Major

**[Accuracy]** — Aurora 连接单元格（cell fa28455e）声称 "`mel_spectrogram()` 用 `fb @ magnitude` 把 FFT 幅度谱压成 mel 频谱"，但 `src/aurora/audio/mel.py:88` 的实际代码是 `return power @ fb.T`，其中 `power` 形状为 `(n_frames, N)`，`fb.T` 为 `(N, M)`，结果为 `(n_frames, M)`。这与 notebook 描述的 `(M,N)@(N,)→(M,)` 方向相反且是批处理形式。该错误直接误导学生对真实代码的理解。Fix: 将描述改为"用 `power @ fb.T` 对所有帧批量做 mel 映射，等价于对每帧单独执行 `matvec(fb, frame)`，但转置后使帧轴在行维度"，并解释为什么需要转置。

**[Pedagogical Quality / Title Mismatch]** — 课程标题和"本课剧情"均以"乘法 = 函数复合"为核心卖点，但正文没有任何单元格演示函数复合的含义：没有展示 `A@(B@x) == (A@B)@x`，没有解释"先施加变换 B，再施加变换 A"等价于"施加 A@B"的几何意义。`matmul` 练习仅被定义为"对 B 每列调用 matvec"，函数复合这一概念完全缺席。Fix: 增加一个单元格，构造旋转矩阵 R 和缩放矩阵 S，分别计算 `R@(S@x)` 和 `(R@S)@x`，用断言验证两路相等，并用文字解释"先缩放再旋转"等价于一个矩阵。

**[Exercise Design]** — `matmul` 练习提示（cell-20）完整泄露了答案：`C = np.column_stack([matvec(A, B[:, j]) for j in range(B.shape[1])])` 是可以直接粘贴的完整实现，习题的认知挑战归零。Fix: 删除该具体一行代码，仅保留方向性描述（"对 B 的每一列调用 matvec，再把结果列拼成矩阵"），让学生自行选择拼接方式。

### 🟡 Minor

**[Exercise Design]** — `matmul` stub 返回 `return None` 而非 `raise NotImplementedError("TODO")`，与 `matvec` stub 风格不一致。当前逻辑：`matmul` 返回 `None`，`np.allclose(None, A@B)` 抛出 `TypeError`，test 捕获后打印"先把 matmul 实现完再运行"——学生明明运行了，消息令人困惑。Fix: 将 `return None` 改为 `raise NotImplementedError("TODO")`，except 只需捕获 `NotImplementedError`。

**[Exercise Design]** — `matvec` 验证仅测试一个 `2×2` 方阵用例，缺少矩形矩阵（如 `(3×2) @ (2,)`）和零向量的测试，而 `matmul` 已经有矩形矩阵用例。Fix: 在 `matvec` 测试末尾补充一个 `(3, 2)` 矩形矩阵断言。

**[Pedagogical Quality]** — cell-19 用纯文字描述了对角矩阵和全1方阵的输入输出行为，没有可执行代码。学生只能阅读声称，无法自行验证。Fix: 将两个描述转换为代码，各附一行 `assert`（或 `print` + `np.allclose` 比较）。

**[Course Continuity]** — 引入旋转矩阵（cell b6471806）时，完全没有与 L06（欧拉公式）中"复数乘法 = 旋转"建立连接。旋转矩阵和 e^{iθ} 是同一操作的两种表示，是极好的跨课衔接时机。Fix: 在旋转矩阵演示后加一句注释："L06 中 e^{iθ}·z 对复平面的旋转，用实数坐标写出来就是这个 2×2 矩阵"。

**[Pedagogical Quality]** — 课程开头没有独立的编号"学习目标"列表，仅有标题单元格中的简短"目标"描述，无法量化学习成果。Fix: 在剧情导入后添加编号学习目标小节，例如："① 能说明矩阵形状 (m,n) 与输入/输出维度的对应关系；② 能手写 matvec 循环并通过验证；③ 能解释为什么矩阵乘法等价于函数复合"。

### 💡 建议

- 在课程末尾增加一个"mini challenge"：给定两个已知变换（先旋转 45°，再沿 x 轴拉伸 2 倍），让学生分别用逐步施加和直接 `A@B` 计算，对比结果，强化复合直觉。
- 旋转矩阵单元格（b6471806）依赖前面单元格已导入的 `np` 和 `plt`，若学生单独重跑该单元格会遇到 `NameError`。建议在该单元格顶部加 `import numpy as np; import matplotlib.pyplot as plt`。
- `matmul` 测试的矩形矩阵用了 `np.random.default_rng(0/1)`，种子已固定，结果可复现，这一点设计值得肯定。

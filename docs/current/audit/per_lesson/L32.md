## 概述

本课聚焦一个简单但核心的概念：用 `np.arange(N) / sample_rate` 建立音频时间轴。解释层面扎实，尤其是关于为何选 `arange` 而非 `linspace` 的一段分析写得清晰有说服力。然而，标注 ✏️ 的"练习格"已把完整答案写进去，学生无需独立推理，练习设计目标彻底落空；且可视化格在任何标准环境下都会崩溃，导致 notebook 无法顶到底完整运行。

## 亮点

- **`arange` vs `linspace` 的分析**（cell `71e16e75`）：明确指出 `linspace` 的浮点累积误差在 FFT 频率对齐时被放大，这是一个实际工程细节，讲到位了。
- **逐步推导脉络**（cells `cell-2` → `8d3f7ea8`）：先列输入/输出约定，再给"推理路线"三步，结构清晰，适合初次接触信号处理的学生。
- **参数扫描实验**（cells `cell-10`）：用双重循环演示 N 随参数同步变化，直观且可立即动手修改。
- **正弦波前瞻铺垫**（cell `cell-8`）：在 L32 先展示 `angle` 和 `wave` 数组，为 L33 做了自然过渡。

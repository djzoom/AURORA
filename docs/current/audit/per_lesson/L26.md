## 概述

## 亮点

- **cviz.py 数学实现完全正确**：`contour_descent` 对 f(x,y)=x²+2y² 的梯度（gx=2x, gy=4y）无误；`descent_1d` 和 `lr_compare` 的更新规则 `x ← x − lr·∇f(x)` 一致且正确。

## 问题列表

### 🔴 Critical

**[Accuracy - 梯度方向声明错误]** — 开篇 cell-60d1ac13 写道："梯度箭头的方向就是 `optimizer.step()` 每次走的方向。"这是错误的：梯度 ∇f 指向函数值最陡上升方向，`optimizer.step()` 沿**负梯度**方向更新参数（`x ← x − lr·∇f`）。同一单元格的后半段正确说"梯度下降则沿梯度**反方向**一步步走向最低点"，形成直接自相矛盾。这是本课最核心概念，开篇一句错误足以建立持久的错误心智模型。
Fix: 将该句改为"梯度箭头的**反方向**就是 `optimizer.step()` 每次走的方向（`x ← x − lr·∇f`）"。

### 💡 建议

- **"动态演示"名实不符**：标题与目标均写"动态演示"，但 cviz.py 输出全为静态 matplotlib 图像（无 FuncAnimation、无 ipywidgets）。建议将标题改为"静态可视化"，或在 cviz.py 中添加 `tangent_animate()` 用 `FuncAnimation` 让切线随 x0 滑动。

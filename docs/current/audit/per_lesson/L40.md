## 概述

L40 的结构清晰，幅度谱、功率谱、dB 谱三种形式讲解得当，`frequency_bins` 练习的脚手架也做得较好。但存在两个根本性缺陷：第一，标题和 L39 收束语明确承诺的"相位谱"内容在本课完全缺失；第二，L39 花了整节课让学生实现 `aurora.audio.transforms.fft`，L40 随即不加说明地切换回 `np.fft.fft`，直接违背了课程"no API wrappers"的核心原则。此外，如果学生在实现 `frequency_bins` 之前顺序运行笔记本，第 13 格将因 `NoneType` 而崩溃。

## 亮点

- 频率轴推导（Δf = sr/N → freqs[k] = k·sr/N）讲解直观，配合不同 N 值的打印循环加深理解。
- 共轭对称验证格（cell 6）用随机信号实验验证 |X[k]| = |X[N-k]|，是正确的理论-实验双向印证。
- `frequency_bins` 的练习说明含有参考输入输出和逐步推理路线，脚手架质量较高。
- "本课收束"指向 L41 的说明（加窗→FFT 全流程）与实际 L41 标题完全吻合。

### 🟡 Minor

**[存根检测逻辑]** — Cell 10 的函数体是 `...`（Ellipsis 字面量），Python 将其作为表达式语句执行，函数实际返回 `None`。Cell 11 的检查 `if result is ... or result is None` 中，`result is ...` 永远为 False（result 是 None 不是 Ellipsis），但 `result is None` 仍能捕获，所以功能上无害，但逻辑表述有误导性。

# L42 — FFT 图形化 — 蝴蝶图 + 纯音 / 和弦 / 噪声的频谱形态对比

**总体评级**: B- | **问题数**: 2 critical / 3 major / 3 minor

## 概述

本课以 DFT 矩阵热力图、N=8 蝶形网络示意图、共轭对称性验证和双频叠加实验四个演示单元组成，数学定义和酉性推导均正确。但存在两处严重问题：标题承诺了"纯音/和弦/噪声的频谱形态对比"，而噪声频谱在正文中完全缺失；Cell 9 的说明文字（N=400, bin≈2.75/5.5）与 Cell 10 的实际代码（N=256, bin≈14/28）完全矛盾。全课无任何 TODO 练习，学生只能被动阅读演示。

## 亮点

- DFT 矩阵的酉性推导（`F @ F† = N·I`）与数值验证步骤严谨，误差阈值 `1e-10` 合理。
- 共轭对称性验证（Cell 8）切片写法 `X[N-1:N//2:-1]` 完全正确，逐 bin 逐一核查，不依赖任何黑盒函数。
- "本课收束"中对 FFT 三个核心结论（矩阵投影、蝶形折叠、前 N//2+1 个 bin 独立）的总结简洁准确，与正文高度一致。
- 下节引用 `L43_stft.ipynb` 编号正确，逻辑衔接自然。

## 问题列表

### 🔴 Critical

**[Title vs Content]** — 标题明确承诺"纯音 / 和弦 / 噪声的频谱形态对比"，但正文完全没有噪声（白噪声/粉噪声）频谱演示，"和弦"仅以 440+880 Hz 两音叠加勉强代替，三者并置对比的情节从未出现。标题与内容的契约关系完全断裂，学生会认为笔记不完整或下载错误。
**Fix**: 补充 Cell：生成 `np.random.randn(N)` 白噪声，绘制其幅度谱（期望看到宽带平坦响应）；同时将 440+880+660 Hz 组合明确标注为"和弦"；最后用一张三行子图并列纯音/和弦/噪声，使标题与内容对应。

**[Stale Description]** — Cell 9（Markdown）描述参数为 `duration=0.05`（50 ms）、`N=400`、`bin≈2.75`（440 Hz）和 `bin≈5.5`（880 Hz）；但 Cell 10 代码实际使用 `duration=0.032`（32 ms）、`N=256`，对应真实 bin 为 `round(440×256/8000)=14` 和 `round(880×256/8000)=28`。数值差距超过 5 倍，学生无法用所标注的 bin 值在图上定位峰值。另外 N=400 不是 2 的幂次，若学生据此修改代码，aurora FFT 可能直接报错。
**Fix**: 将 Cell 9 改为与 Cell 10 保持一致：`duration=0.032`、`N=256`、`bin≈14`（440 Hz）、`bin≈28`（880 Hz）。

### 🟠 Major

**[Butterfly Diagram Incomplete]** — Cell 6 的蝶形网络只画了"直通"和"上→下"斜线，缺少"下→上"的交叉连线。标准 Cooley-Tukey 蝶形操作需要两条交叉线（构成完整 X 形）才能表达 `X[top] = E + W·O`、`X[bot] = E − W·O` 两个方向的数据流；仅有上→下斜线，图形无法传达减法分支，学生误以为只有加法路径。
**Fix**: 在 `ax.plot([col_in, col_out], [yt, yb], ...)` 下方补加 `ax.plot([col_in, col_out], [yb, yt], ...)` 画出下→上交叉，并在两条交叉线旁分别标注 `+W^k` 和 `−W^k`。

**[Dead Code / Unused Variable]** — Cell 6 第 81 行声明了 `current_y = list(range(N))` 并注释"追踪每条 wire 的 y 坐标"，但后续循环中从未读取或修改该变量。同一 Cell 中 `phase = b / group` 也计算后被丢弃，twiddle factor 的相位变化完全未映射到任何视觉元素。
**Fix**: 删除 `current_y`；将 `phase` 映射为交叉线颜色的色相（`hsv` colormap 按 phase 着色），或至少在旋转因子标注中显示 `f'W⁸^{b*group//half}'`（正确的指数），而不是固定的 `b`（当 `b==0` 时恒为 `W⁸^0`，所有组标注完全相同）。

**[No Exercises]** — 全课 11 个 Cell 均为演示代码或说明文字，无任何 TODO 练习。对于 L42 这一处于 DSP 模块中段的课节，学生至少需要一个动手环节来巩固蝶形折叠原理或共轭对称性的实际含义。
**Fix**: 在 Cell 8 之后插入练习：要求学生手工用 NumPy 实现 N=8 的两级 Cooley-Tukey 迭代（不调用 `fft`），并与 `aurora.audio.transforms.fft` 结果进行数值比较，误差应低于 `1e-10`；提供 `raise NotImplementedError` 桩函数和验证断言。

### 🟡 Minor

**[Twiddle Factor Annotation Always W⁸^0]** — Cell 6 中 `if b == 0` 分支使 twiddle 标注仅在每个蝶形组的第一只蝶形出现，且指数恒为 `{b}` = 0，即所有标注均显示 `W⁸^0`，与实际旋转因子（随 `b` 增大）不符。
**Fix**: 移除 `if b == 0` 限制，改为对所有 `b` 标注，并根据正确公式（`b * N // group`）显示指数。

**[Hardcoded Superscript 8]** — `f'W⁸^{b}'` 中的 `⁸` 是 Unicode 硬编码，若未来演示 N=16 时不会自动更新，形成静默错误。
**Fix**: 将 `⁸` 改为 `f'W^({N})^{{{b}}}'` 或使用 LaTeX text，使标注随 `N` 自动变化。

**[参数实验描述承诺但未实现 N=400 非 2 的幂]** — Cell 9 描述 N=400 但此值在不支持任意长度 FFT 的 aurora 实现中会产生错误，即使仅作为错误示例也没有说明，容易误导学生认为非 2 的幂次合法。
**Fix**: 将说明文字改为明确指出 aurora FFT 要求 2 的幂次，并以 N=256（实际用值）示例，若要演示任意 N 须先用 zero-padding。

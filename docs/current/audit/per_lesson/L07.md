## 概述

本课聚焦于用奇次谐波叠加逼近方波这一核心直觉，结构紧凑，练习提示清晰，Gibbs 现象的讲解点到为止。然而存在两处可核查的事实错误（标题术语混用、值域声明错误），加上关键引用模块路径不存在、公式归一化在不同单元间前后矛盾，这些问题会直接误导学生。改正这几处后，本课可升至 B+。

## 亮点

- **练习脚手架扎实**：`square_approx` 的推理路线分三步写出（初始化、奇次下标生成、循环累加），降低了认知门槛。
- **渐进叠加的参数实验**（cell-7、cell-9、cell-17）从单点数值收敛到图形可视化，层次清晰，符合"先观察，再实现"的教学节奏。
- **Gibbs 现象的处理**：明确说明"即使 n 很大，跳变点附近误差不会完全消失（约 9%）"，避免学生误以为加无限项就能完美拟合方波。
- **验收断言设计合理**：`0.85 < approx.max() < 1.15` 的数值范围对正确实现（n=50 下最大值 ≈ 0.926，含 Gibbs 过冲）能稳定通过，未做过紧设置。

## 问题列表

### 🔴 Critical

**[Accuracy / Title]** — 标题中"三角波叠加"是术语错误。在中文数学语境中，"三角波"（triangular wave）指锯齿形波形，而非三角函数（trigonometric functions）的产物。整课内容使用的是**正弦波**（sine waves / sinusoids），标题应为"正弦波叠加合成方波"。学生读标题时会以为课程在讲三角波，与实际内容不符。  
Fix: 将标题和 H1 单元中的"三角波叠加"改为"正弦波叠加"。

**[Accuracy / Cell-11]** — "值域约为 [-π/4, π/4]"这一声明对任意有限 n 都是错误的。实测：n=1 时范围是 [-1.0, 1.0]；n=50 时因 Gibbs 过冲，范围是 [-0.926, 0.926]。π/4 ≈ 0.785 是理论方波在**平坦段**的极限值，并非实际部分和的值域上界。这一错误声明与后续断言（`approx.max()` 在 0.85–1.15）直接矛盾，且会让学生误判自己的实现是否正确。  
Fix: 改为"对于有限 n 项，峰值约为 0.93（含 Gibbs 过冲），趋近方波时平坦段收敛到 π/4"。

### 🟠 Major

**[Runability / Module Path]** — 开头 markdown 单元（f2126435）和收束单元（c6e6041c）均引用 "`aurora.audio.fft`"，但该模块不存在。正确路径是 `aurora.audio.transforms`（包含手写 `dft`/`fft`/`stft` 等）。学生若按此路径尝试 `import aurora.audio.fft` 会直接报 `ModuleNotFoundError`。  
Fix: 将所有对 `aurora.audio.fft` 的引用替换为 `aurora.audio.transforms`。

**[Accuracy / Normalization]** — 练习公式为 `Σ sin(2πkt)/k`（峰值 → π/4），但 cell-15 打印"幅度 = 4/(π×k)"并声称"→ 在 L37 FFT 谱里，这些频率位置会出现对应高度的峰值"。这两个归一化代表的是两条不同的曲线：前者峰值 ≈ 0.785，后者峰值 ≈ 1.273（第一谐波）。两者相差因子 4/π，课程中从未解释这一差异，学生很难理解哪个才是 FFT 会"看见"的幅度。  
Fix: 在 cell-15 前明确说明这里展示的是**振幅为 1 的单位方波**（标准归一化），而 `square_approx` 使用的是未归一化版本，两者相差因子 4/π。

**[Exercise Design / Stub]** — `square_approx` 的 stub 在 TODO 后直接 `return out`（全零数组），而非 `raise NotImplementedError`。这导致后续绘图单元静默运行（显示三条水平线），只有断言才给出错误信号，且断言错误信息不够指引性。全零实现也通过了部分检查（0 < 1.15 为真，只有下界检查失败）。  
Fix: 将 stub 最后一行改为 `raise NotImplementedError("实现 square_approx：累加前 n 个奇次谐波")`，并确保绘图单元有 try/except 包裹以给出友好提示。

**[Pedagogical Quality / 学习目标]** — 全课缺少显式的、编号的"学习目标"部分。开篇只有一段"目标"描述，但没有可检核的具体条目（如"能手写方波 Fourier 级数的前 n 项"、"能解释 Gibbs 现象的物理含义"）。  
Fix: 在 H2"本课剧情"前添加"## 学习目标"小节，列出 3–4 条编号目标。

### 🟡 Minor

**[Pedagogical Quality / Opening Example]** — 开头示例（cell aa3c8bc7）叠加 2 Hz + 0.5×(5 Hz)。5 Hz 并非 2 Hz 的奇次谐波（2×1, 2×3, 2×5... 应为 2, 6, 10 Hz），权重 0.5 也不是 1/k 规律。这个"入口实验"与后续谐波叠加的核心逻辑是脱节的，会造成混淆。  
Fix: 改为叠加 1 Hz + (1/3)×3 Hz，直接演示 1/k 权重的奇次谐波，与后续练习无缝衔接。

**[Accuracy / Assert Comment]** — 断言注释中说"若 > 1.2 可能用了错误比例"，但代码上界是 `1.15`，两者不一致。  
Fix: 将注释改为"若 > 1.15 可能用了错误比例（如多乘了 4/π 因子）"。

**[Pedagogical Quality / Leibniz]** — cell-9 中直接使用 `theory = np.pi / 4` 作为 t=0.25 处的理论值，但没有说明这是莱布尼茨公式（Leibniz formula）`1 - 1/3 + 1/5 - ... = π/4`。这是本课最美的数学事实之一，完全可以用一行注释 `# Leibniz: 1-1/3+1/5-1/7+... = π/4` 点破。  
Fix: 在 `theory = np.pi / 4` 旁加注释解释该值的来源。

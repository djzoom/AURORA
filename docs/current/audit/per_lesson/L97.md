**[课程衔接 / 引用错误]** — Section 5 编程题速查表第5行写"余弦相似度（cosine similarity）→ L78"，但 `notebooks/8_music/L78_beat_tracking.ipynb` 的主题是"节拍追踪——onset 包络、自相关与 BPM 估计"，与余弦相似度无关。余弦相似度的主课是 `L80_similarity.ipynb`（《向量相似度检索 — 余弦相似度 vs 点积 vs L2，纯 NumPy k-NN 实现》）。
Fix: 将表格中 `L78` 改为 `L80`。

**[练习设计 / 桩函数设计]** — `hz_to_mel` 和 `frame_signal` 两个练习函数均以 `return None` 作为桩，且验证断言被 `if ... is not None:` 条件保护。学生可以完全不实现任何代码，直接运行整个 notebook，不会触发任何错误或警告，得到两行静默跳过。正确做法是 `raise NotImplementedError`，让未实现立刻报错，强迫学生动手。
Fix:
```python

    raise NotImplementedError("实现 HTK mel 公式")

def frame_signal(x, win_len, hop):
    raise NotImplementedError("用 np.lib.stride_tricks 或切片实现帧切分")
```
同时去掉 `if ... is not None:` 保护，让断言无条件运行。

**[结构 / 章节重复编号]** — 文档中出现两个"5."：第一个是 `## 5. 编程题准备：音频 AI 的高频手写题`（cell `9b7e7f31`），第二个是 `## 5. ✏️ 模拟面试练习`（cell `285c2b52`）。重复编号破坏了 Jupyter 的大纲导航，且让"Section 5"在 notebook 内产生歧义。

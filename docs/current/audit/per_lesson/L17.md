## 概述

L17 builds a coherent narrative around eigendecomposition and correctly covers the characteristic equation, orthogonal diagonalization of the 3×3 symmetric example, and the matrix-power shortcut `Aⁿ = PΛⁿP⁻¹`. The core mathematics is sound and the stability-analysis motivation is on-point. However, a silent variable-shadowing bug renders the only graded exercise permanently broken: cells `cell-6` and `cell-8` both overwrite the global `A` with a 2×2 matrix, so when the char_poly verification assert runs it operates on the wrong matrix and fails even for a perfectly correct implementation. The only quoted reference output (`char_poly(A, 5) ≈ −384`) is numerically wrong (correct value is +56), and a section header promises eigenvector verification that the following code does not actually perform.

## 亮点

- The stability framing ("all |λ| < 1 → filter converges; any |λ| > 1 → exponential blow-up") is excellent and maps directly to the Aurora DSP motivation stated in CLAUDE.md.
- The three-step reasoning scaffold for the `char_poly` exercise (construct shift, singular at eigenvalue, determinant = 0) is clear and didactically sound.
- Eigenvectors v1, v2, v3 given for the 3×3 symmetric example are mathematically verified correct and mutually orthogonal (all dot products = 0), making the orthogonal diagonalization demo in `dee6428b` self-consistent and runnable.
- The parameter sweep in `cell-18` (varying off-diagonal entry c, watching eigenvalue ratio) gives clean intuition for power-iteration convergence speed.
- The forward reference to L18 (invertibility criteria) in "本课收束" is accurate.

## 问题列表

### 🔴 Critical

**[Runability / Exercise Design]** — Variable `A` is overwritten twice before the char_poly verification cell runs. `d447bd31` sets `A` to the 3×3 matrix, but `cell-6` and `cell-8` both execute `A = np.array([[4., 1.],[2., 3.]])`. By the time `bd87cdd6` runs `assert all(abs(char_poly(A, lam)) < 1e-6 for lam in (-3,-2,6))`, `A` is the 2×2 matrix. A correct `char_poly` implementation returns 40, 28, and 4 at those lambdas — never near zero — and the assertion fails unconditionally. A student who writes a perfectly correct implementation will see a failing assert with no diagnostic clue.

Fix: Add `A = np.array([[3,3,3],[3,-1,1],[3,1,-1]], float)` at the top of cell `bd87cdd6` (or immediately before it). Alternatively rename the 2×2 demo matrix in `cell-6`/`cell-8` to `B` throughout those cells.

### 🟠 Major

**[Accuracy]** — The reference output quoted in the exercise description is wrong. Cell `505c5565` states "`char_poly(A, 5)` ≈ −384". The correct value is det([[3,3,3],[3,-1,1],[3,1,-1]] − 5I) = det([[-2,3,3],[3,-6,1],[3,1,-6]]) = -2·(36-1) − 3·(-18-3) + 3·(3+18) = **+56**. Students who mentally sanity-check their implementation against −384 will conclude a correct implementation is wrong.

Fix: Replace "≈ −384" with "≈ +56" in the `505c5565` markdown cell.

**[Exercise Design]** — The `char_poly` stub returns `None` instead of raising `NotImplementedError`. When the verification cell evaluates `abs(char_poly(A, lam))`, Python raises `TypeError: bad operand type for abs(): 'NoneType'` — a message that points at `abs()`, not at the unimplemented function.

Fix: Replace `return None` with `raise NotImplementedError("Implement char_poly: return det(A - lam * np.eye(len(A)))")`.

**[Pedagogical Quality]** — Cell `cell-7` is titled "代码实验：遍历特征向量，验证 Av=λv" but the code in `cell-8` computes matrix powers (`Aⁿ = P Dⁿ P⁻¹` vs `np.linalg.matrix_power`). Eigenvector verification (`A @ v ≈ λ * v`) is never actually coded. The description promises one thing; the code delivers another.

Fix: Either (a) insert a genuine Av=λv verification loop before `cell-8` (iterate over columns of `P`, print residual `‖A@v − lam*v‖`), or (b) retitle `cell-7` to "矩阵幂次验证：Aⁿ = P Dⁿ P⁻¹".

**[Course Continuity]** — Cell `08f8be3c` (Aurora connection) references "p6 SVD" — a stale chapter/page number from an earlier numbering scheme. The course uses L-numbering; SVD is covered in `L14_eigen_svd.ipynb`.

Fix: Replace "p6 SVD" with "L14 SVD".

### 🟡 Minor

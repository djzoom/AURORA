## 问题列表

### 🔴 Critical

**[代码准确性]** `p95` 计算存在 off-by-one 错误，实际返回最大值
在 `monitor_inference_latency`（cell `99df8103`）中：
```python
p95 = latencies[int(0.95 * n_trials)]  # n_trials=20 → index 19 = max
```
`int(0.95 * 20) = 19`，而排好序的 20 元素列表最大索引为 19，因此 `latencies[19]` 是最大值（p100），不是 p95。正确写法：
```python
p95 = latencies[int(math.ceil(0.95 * n_trials)) - 1]  # index 18
# 或
p95 = statistics.quantiles(latencies, n=100)[94]
```
学生将这段代码照抄进生产监控脚本后，SLA 阈值会被最坏情况（单次最慢请求）触发，而非真正的 p95，导致误告警或漏告警。Fix：将索引改为 `int(math.ceil(0.95 * n_trials)) - 1`，并添加断言 `assert p95 < latencies[-1]` 作为验证。

**[代码准确性]** Dockerfile ENTRYPOINT 引用不存在的模块 `aurora.serve`

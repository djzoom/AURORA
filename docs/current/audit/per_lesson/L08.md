# L08 — 复数平面可视化 — 单位圆旋转、共轭与相位，matplotlib 动态演示

**总体评级**: B- | **问题数**: 0 critical / 3 major / 4 minor

## 概述

L08 is a short visualization companion that wraps four plotting functions (`unit_circle_euler`, `sinusoid_anatomy`, `complex_point`, `roots_of_unity`, `fourier_square`) behind an `xviz`/`aurora.aviz` shim and calls them in sequence. The mathematical content is correct and the fallback implementations are sound. However, the lesson has two systematic problems: (1) all cross-references to preceding lessons still use the pre-rename "x1–x4" labels instead of L04–L07, and (2) the title and 目标 explicitly promise coverage of "共轭 (complex conjugate)" but no cell in the notebook demonstrates conjugation at all.

## 亮点

- The `fourier_square` fallback implementation is mathematically correct: `(4/π) Σ sin(2πkt)/k` for k = 1, 3, 5, … up to n, which is the exact partial-sum formula for a square wave.
- The twiddle-factor sign convention (`W_N^k = e^{-i2πk/N}`) is stated correctly and consistently in both the parameter-experiment prose and the wrap-up section.
- The lesson wrap-up correctly identifies the next lesson as L09 (vectors) and the Aurora FFT notebook as L06 — both are accurate after the rename.
- The fallback `complex_point` annotates both rectangular (`a+bj`) and polar (`|z|`, `∠`) forms side-by-side, which is good for building geometric intuition.

**[Accuracy / Title Mismatch]** — The notebook title, subtitle, and 目标 all promise "共轭（complex conjugate）" content ("用 `matplotlib` 把单位圆旋转、共轭（complex conjugate）和相位（phase）画出来"), but no cell, function, or explanation covers conjugation. ### 🟡 Minor

**[Runability]** — `cell-18` uses `np` without declaring `import numpy as np` in that cell. It relies on `np` being injected into kernel namespace by cells 5 and 7. Running cell-18 in isolation (or if cell execution is interrupted) will raise `NameError: name 'np' is not defined`. Fix: add `import numpy as np` at the top of cell-18.

**[Accuracy]** — The `fourier_square` label `f'{n} harmonics'` is imprecise: when `n=3`, `range(1, 4, 2) = [1, 3]` yields only 2 terms, so "3 harmonics" is shown but 2 partials are drawn. For `n=50`, 25 terms are drawn but labelled "50 harmonics". Fix: change the label to `f'up to harmonic {n}'` or compute the actual term count: `label=f'{len(range(1, n+1, 2))} terms (max k={n})'`.

**[Accuracy]** — The Aurora connection in section 3 cites `aurora/audio/transforms.py` but the project uses a `src/` layout, so the importable path is `src/aurora/audio/transforms.py`. The path as written would mislead a student who tries to open the file from the repo root. Fix: change to `src/aurora/audio/transforms.py` or use the import form `aurora.audio.transforms`.

**[Pedagogical Quality]** — There are no numbered learning objectives. The intro contains a single 目标 paragraph rather than a numbered list of specific, verifiable outcomes. This is inconsistent with how other lessons in the module are structured. Fix: add a 学习目标 block with 3–4 numbered objectives, e.g., (1) use `np.exp(1j*theta)` to place a point on the unit circle; (2) explain why complex multiplication = rotation; (3) identify the N roots of unity as DFT twiddle factors.

### 💡 建议

- The `fourier_square` calling convention `fourier_square((1,3,9,50))` is underdocumented — a one-line comment explaining that each integer is the maximum odd harmonic number (not a count of terms) would prevent confusion.
- Consider adding a parameter-sweep cell that animates or overlays the unit circle with both z and z̄ once the conjugate section is added; the symmetry of the real-axis reflection is one of the most visually memorable facts in the whole complex-number unit.
- The `roots_of_unity` fallback silently delegates to `aurora.aviz.twiddles(N)` — if `aviz` is also unavailable this would produce a confusing `AttributeError`. A guard or a self-contained fallback implementation would improve robustness.

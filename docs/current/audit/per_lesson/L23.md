## 问题列表

### 🔴 Critical

**[准确性 / 教学质量]** — The orthogonality assertion in cell-15 is a mathematical tautology and validates nothing about gradient correctness.

```python
tangent = np.array([-g[1], g[0]])           # 90° rotation of g — always ⊥ g by construction
dot = np.dot(grad_dir, tangent / np.linalg.norm(tangent))
assert abs(dot) < 1e-6                       # ALWAYS passes regardless of what g contains
```

For any non-zero vector g = [a, b], dot([a,b], [-b,a]) = -ab + ba = 0 algebraically. A student who implements `gradient` to return [1, 0] everywhere will still see "✅ 梯度与等高线正交性验证通过" and conclude their implementation is geometrically correct. The assertion teaches false confidence.

Fix: compare the computed gradient direction to the analytically known direction. For f = x²+y², the gradient at any point p is proportional to p (radially outward):

```python
radial = np.array(point) / np.linalg.norm(point)
grad_dir = g / np.linalg.norm(g)
assert np.allclose(grad_dir, radial, atol=1e-4), f"梯度方向不对: {grad_dir}"
print('✅ 梯度方向与解析径向方向吻合')
```

## 概述

本课目标明确（CTC 前向 DP），扩展标签构造和递推公式的数学描述基本准确，但整个实现骨架几乎是空的：`ctc_forward` 在构造 `lprime` 之后立刻 `raise NotImplementedError`，导致 Cell 6 和 Cell 8 无法运行，且验证断言极度薄弱（仅检查 finite 和 <0），缺乏参考实现对照，无法区分"正确实现"和"碰巧返回负数的错误实现"。Cell 1 的公式用概率域乘法书写，而骨架代码却用 log 域，二者从未显式衔接，对新手造成混淆。

## 亮点

- 扩展标签（extended label）`l' = blank l₁ blank l₂ ... blank` 的构造逻辑正确，lprime 建立代码可直接复用。
- 三个合法前驱（停留/跳1/跳2）的条件以伪代码清晰列出，"跳2仅当 l'[s] ≠ l'[s-2]"的约束表达准确。
- 摘要表格（Cell 9）对关键概念做了紧凑的复习，L70 指向 Whisper 架构也是合理的课程衔接。
- 使用 `NEG_INF = -1e30` 而非 `float('-inf')` 以避免 `nan + nan` 传播，这是数值上成熟的选择。

## 问题列表

### 🔴 Critical

**[可运行性]** — `ctc_forward` 在设置完 `lprime` 后立刻 `raise NotImplementedError("TODO")`，此时 alpha 矩阵已分配但初始条件和递推均未写入。Cell 6 和 Cell 8 将在调用函数时直接抛出异常，整个笔记本自 Cell 6 起无法运行。  
Fix: TODO 注释应标注在骨架内部每个实现块之前，而不是替代整个函数体；或者拆分为两个 TODO：(a) 初始条件，(b) 递推 + 返回值，并给出 `# 提示：返回 np.logaddexp(alpha[-1, -1], alpha[-1, -2])` 等提示。

**[验证充分性]** — Cell 6 的两条断言（`np.isfinite(lp)` 和 `lp < 0`）极度薄弱，任何返回 `-1.0` 的错误实现都能通过。对于动态规划算法，必须提供一个暴力枚举参考实现（当 T 和 L 较小时路径数有限）作为数值对照，否则学生无法验证正确性。  
Fix: 添加 `ctc_forward_brute_force`（枚举所有长度为 T 的路径，过滤合法路径，对 log_probs 求和后 logsumexp）并断言两者结果在 1e-6 以内。

**[公式与代码域不一致]** — Cell 1 的递推公式用概率域书写（`α[t][s] = ... × p[t][l'[s]]`，乘法），而骨架代码用 log 域（`alpha = np.full(..., NEG_INF)`，加法）。课文从未说明如何将概率域公式转换为 log 域（`logsumexp` + 加法），新手将不知道如何动手。  
Fix: 在公式后紧跟 log 域等价形式，说明 `log α[t][s] = logsumexp(log α[t-1][s], log α[t-1][s-1], ...) + log p[t][l'[s]]`，并指出可用 `np.logaddexp` 实现。

### 🟠 Major

**[准确性 — 注释错误]** — Cell 3 第一行注释写"3 帧"，但实际设置 `T = 6`。  
Fix: 将注释改为"6 帧"，或将 T 改回 3 以保持与注释一致（6 帧对玩具示例更合适，建议只改注释）。

**[练习设计 — 缺少返回值引导]** — 函数末尾应返回 `np.logaddexp(alpha[-1, -1], alpha[-1, -2])`（对应序列可在最后一帧结束于位置 S-1 或 S-2），但这一关键步骤在 TODO 注释和文档中均未提及，学生容易只实现递推而忘记正确聚合终态。  
Fix: 在 TODO 注释中明确标注 "步骤3：返回终态 log-sum（最后一帧位置 S-1 和 S-2 的 logaddexp）"。

**[可运行性 — blank 默认值不一致]** — 函数签名 `blank: int = 0` 与 Cell 3 设置的 `BLANK = 2` 不同；Cell 6 显式传入 `blank=BLANK`，但 Cell 8 也传入 `blank=BLANK`，所以调用端是对的，但函数默认值 0 会让学生误以为空白符索引是约定为 0 的，与本笔记本词汇表不符。  
Fix: 将默认值改为 `blank: int = 0`（并在注释中说明这是常见约定）或与 BLANK=2 对齐；至少在 Docstring 中说明"默认值不适用于本笔记本示例，请显式传入 blank=BLANK"。

**[数值稳定性]** — Cell 3 和 Cell 8 的 log-softmax 计算为 `logits - np.log(np.exp(logits).sum(...))`，当 logits 值较大时 `np.exp` 会溢出（本例 `randn` 值较小所以不触发，但 T=100 仍可能遇到大值）。  
Fix: 使用稳定写法：`logits - logits.max(axis=1, keepdims=True) - np.log(np.exp(logits - logits.max(axis=1, keepdims=True)).sum(axis=1, keepdims=True))`，或直接 `from scipy.special import log_softmax`（但需确认是否违反 numpy-only 约束；若是，使用手动稳定版本）。

### 🟡 Minor

**[缺少 Aurora 连接]** — 本课没有"Aurora连接"小节，但 ASR 单元 src/aurora/asr 模块中应有 CTC 的位置；不指向具体模块使学生无法知道代码落地在哪里。  
Fix: 添加小节说明 `src/aurora/asr/ctc.py` 或等效路径将使用本函数。

**[复杂度表述]** — 标题写"O(T×2L)"，表格写"O(T × 2L)"，但精确复杂度是 O(T·(2L+1))，S = 2L+1 而非 2L。  
Fix: 统一改为 O(T·S) 并注明 S = 2L+1，或改为 O(T·L)（大 O 省略常数时）并加说明。

**[边界情况测试缺失]** — 没有测试 labels=[] 的空序列情况（S=1，只有一个 blank 位置），也没有测试 T < 2L+1（理论上不可能对齐）的输入，这两种情况实现上容易出错。

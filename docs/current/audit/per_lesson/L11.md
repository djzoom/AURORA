## 问题列表

### 🔴 Critical

**[Content gap]** — 标题承诺"正则化含义"，但正文从未教授正则化。

开头提到"L1/L2 正则项直接约束权重向量的范数"，此后再无任何代码、推导或练习解释 L1 正则化（稀疏性、Lasso）与 L2 正则化（权重衰减、Ridge）的区别和作用机制。学生读完本课仍无法回答"为什么选 L1 而不是 L2 正则化"。

Fix: 二选一——① 补充一个最小演示节，展示 `grad ||w||₁ = sign(w)` 把小权重归零 vs `grad ||w||₂² = 2w` 等比例缩小，约 10 行代码；② 从标题和 intro 删去"正则化含义"，仅聚焦计算与几何。

**[Runability]** — stub 返回 `None`，测试单元以 TypeError 崩溃，笔记本无法顶到底运行。

`normalize` stub（cell `68130fba`）末行为 `return None`。紧接其后的测试单元（cell `16166fa5`）第一行执行 `round(float(np.linalg.norm(u)), 6)`，其中 `u = None`；NumPy 将 `None` 转为 0-d object 数组后抛出 TypeError。学生看到的是与实现无关的栈追踪，无法分辨"代码未写"与"代码有 bug"。

Fix: 将 stub 改为 `raise NotImplementedError("TODO: divide v by its L2 norm")`，并在测试单元首行加 `try/except NotImplementedError: print("尚未实现，请完成 normalize()")` 保护，确保未实现时优雅跳过。

### 🟠 Major

**[Exercise design — zero vector not scaffolded before test]** — 零向量的 eps 保护要求在测试单元之后才被提及。

练习说明（cell `c7bce6f0`）的推理路线仅描述"算 length，做 v/length"，未提零向量。测试单元（cell `16166fa5`）随即断言 `not any(x != x for x in z)`（要求无 NaN）。eps 保护提示 `v / (np.linalg.norm(v) + 1e-8)` 出现在 cell-19 的 markdown 段，位于测试之后。按说明实现的学生会因 `0/0 → nan` 而断言失败，且不知原因。

Fix: 将 eps 说明移入 cell `c7bce6f0` 的推理路线第 3 步，并明确约定："零向量应返回全零向量（eps 保护：`v / (‖v‖ + ε)`，ε=1e-8）"。

**[Exercise design — test does not verify direction]** — 测试只检查 `‖result‖₂ = 1`，不验证方向。

仅检查范数为 1 时，返回 `np.array([1.0, 0.0])` 的错误实现也能通过所有三条断言。

Fix: 增加方向断言：
```python
assert np.allclose(normalize(np.array([3.0, 4.0])), [0.6, 0.8], atol=1e-9), "方向错误"
assert np.allclose(normalize(np.array([-6.0, 8.0])), [-0.6, 0.8], atol=1e-9)
```

**[Accuracy — 角点注释错误]** — cell-18 注释"L∞ 球是正方形，角点 (0.6, 1.0)"不正确。

L∞ 单位正方形的四个角点是 `(±1, ±1)`；当 `x1=0.6` 时，`x2=1.0` 是正方形上边的一个边上点，不是角点。

Fix: 改注释为 `# L∞ 球：上边界 x₂=1，点 (0.6, 1.0) 在边上而非角点`，或将 `x1` 改为 `1.0` 演示真正的角点。

## 概述

作为一个独立的「声音数字化入门」单元，本课教学设计扎实：先手算推导表、再写代码、再用 assert 对答案的方法论贯穿全课，数学公式完全正确，所有 assert 阈值经数值验证均可通过。然而，本课实际内容（采样、make_sine）与课程计划中 L02 的承诺标题「NumPy 速成 — 数组、dtype、广播与切片」严重不符：dtype、广播（broadcasting）、切片（slicing）三个核心 NumPy 概念一字未提，这是课程连贯性的重大缺口。此外，`import numpy as np` 分散在多个单元格而非置于顶部，存在 NameError 风险。

## 亮点

- **「先手算，再写代码」方法论** 是全课最大亮点：每个任务均配有推导表（手填 N、t、sin(θ)），直接训练面试场景下的白板推理能力，体现了 Aurora 课程的核心教学理念。
- **数学精度选取精心**：`make_sine(1.0, 16, 2.0)` 的参数使正弦波恰好命中 ±1.0 样本点（t=0.125s 处 sin(π/2)=1），令 `assert abs(x.max()-1.0) < 1e-9` 可以确定性通过，而非依赖浮点近似。
- **手算对照设计合理**：`make_sine(0.5, 8, 1.0)` 给出 [0, π/4, π/2, 3π/4] 逐步推导表；`signal_summary` 用 [0, 1, 0, -1] 四点验证，均是极小但有代表性的示例。
- 学习目标编号明确（共 6 条），难度适配 L02 位置，内容循序渐进。

## 问题列表

### 🔴 Critical

**[课程衔接 / 内容缺失]** — 本课实际内容与 L02 在课程计划中的定位严重不符。课程计划标注 L02 为「NumPy 速成 — 数组、dtype、广播与切片，30 分钟能写信号处理代码」，但实际 notebook 完全覆盖的是「声音的数字表示」（采样率、make_sine 等），三个承诺的 NumPy 核心概念（dtype 类型选择 float32 vs float64、广播机制、数组切片）一个也未出现。后续 L32–L51 的大量 DSP 代码依赖这三个概念，若 L02 不讲，学生将在没有铺垫的情况下遇到 `x[:, None]`、`float32` 精度损失、`arr[::2]` 等写法。

**Fix**: 在本课中补充以下三个小节（每个约 2–3 个代码单元格）：
1. **dtype**：展示 `float64` vs `float32` 的内存与精度差异（`np.zeros(N, dtype=np.float32)`），解释为何音频处理默认用 float32；
2. **广播**：用叠加两路正弦波 `A*sin(...) + B*sin(...)` 或「时间轴乘振幅向量」演示广播；
3. **切片**：用 `x[sr:2*sr]`（截取第 1–2 秒）或 `x[::2]`（降采样）演示切片。

### 🟠 Major

**[代码可运行性 / 导入分散]** — `import numpy as np` 首次出现在 cell c6，但函数定义（c8 `make_time_axis`，c13 `make_sine`）的函数体中使用了 `np.arange` 和 `np.sin`。若学生从 c7 开始运行（例如重新启动 kernel 后跳过前面的单元格，或只运行任务 1 相关单元格），c9 调用 `make_time_axis` 时函数体内的 `np.arange` 会抛 `NameError: name 'np' is not defined`。此外 c9、c14、c20 各自重复导入 numpy，增加混乱。

```python
import numpy as np
import matplotlib.pyplot as plt

    from IPython.display import Audio, display

```

**[教学质量 / 缺少 Aurora 连接]** — 其他课 notebook 通常包含「Aurora 连接」节，将课上实现的函数与 `src/aurora/` 真实代码库对应。本课四个函数（samples_count、make_time_axis、make_sine、signal_summary）与 `src/aurora/audio/transforms.py` 高度相关，但 notebook 完全没有提及。学生不清楚自己实现的东西在真实项目中处于什么位置。

**Fix**: 在「本课收束」之前加入「Aurora 连接」小节，指出 `make_time_axis` 对应 `src/aurora/audio/transforms.py` 中的时间轴工具，`make_sine` 用于测试桩和 L32 信号生成器，并附上 `from aurora.audio import ...` 示范导入。

### 🟡 Minor

**[练习设计 / Stub 约定不一致]** — 四个函数的 stub 使用 `return None` 或 `return {}`，而非 `raise NotImplementedError`。虽然 assert 最终会捕捉到错误，但 `return None` 的失败信息（`AssertionError: 期望 4，得到 None`）比 `NotImplementedError: samples_count 未实现` 可读性更差，且无法区分「学生忘了实现」与「实现后返回了 None」。

**Fix**: 将每个 stub 改为：
```python
def samples_count(duration, sample_rate):
    """返回给定时长和采样率下的采样点总数 N。"""
    raise NotImplementedError("TODO: 返回 round(duration * sample_rate)")
```

**[练习设计 / 缺少边界条件测试]** — 四个函数的验证单元格均未覆盖边界情况：`duration=0`（空数组）、`frequency=0`（直流信号 sin=0 全程）、极端采样率（sr=1）。`signal_summary` 没有测试输入为 2D 数组时是否报错或给出有意义的结果。

**Fix**: 为每个函数补充 1–2 个边界断言，例如：
```python
assert samples_count(0, 8) == 0
assert make_time_axis(0, 16).shape == (0,)
assert make_sine(0.5, 8, 0.0).max() == 0.0  # 零频率 → 全零
```

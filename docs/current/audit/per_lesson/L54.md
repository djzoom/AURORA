# L54 — Value 计算图 — 标量自动微分：前向值 + 反向梯度，手写 add / mul 节点

**总体评级**: B- | **问题数**: 2 critical / 3 major / 5 minor

## 概述

L54 的动机讲解和 DAG 概念介绍写得清晰，手算示例数学正确，测试断言的数值全部可验证。然而，所有 TODO 方法均用 `pass` 代替 `raise NotImplementedError`，导致学生在桩代码状态下得到的是 `AttributeError` 而非有意义的提示；更严重的是，两处外部引用均指向不存在的资源（`solutions/L54` 文件和 `src/aurora/ml/` 目录），学生按说明操作会撞上死链。

## 亮点

- **动机例子具体**：Cell 1 用 `L = tanh(a*b + b**2)` 对比手写偏导与计算图方法，说明"百层网络不可手写"的痛点，比抽象描述更有说服力。
- **数学全部正确**：Cell 11 断言（`a3=4, b3=2` 时 `dL/da=2, dL/db=8`）、Cell 13 的三组参数实验（包含负值），与手算结果完全吻合，容差 `1e-9` 对于精确浮点运算合适。
- **推理路线（Cell 9）详细**：五步推导链（`__init__` → `__add__` → `__mul__` → `__pow__` → `backward`）列出了局部偏导公式，比只给签名更能引导学生独立实现。
- **DAG 演示（Cell 6）**：用字典模拟 DAG 的拓扑排序不仅可执行，还为 `backward()` 中的拓扑算法提供了可参考的伪代码模板。

## 问题列表

### 🔴 Critical

**[Exercise Design / Runability] 全部 TODO 方法使用 `pass` 而非 `raise NotImplementedError`，导致误导性错误消息**

`__init__` 的桩是 `pass`，执行后 `self.data` 和 `self.grad` 均未设置。随后调用任何已预写的 `__repr__`（引用 `self.data`、`self.grad`）或下游方法（`__add__` 返回 `None`，再访问 `.data` 得到 `AttributeError: 'NoneType' object has no attribute 'data'`）时，报错指向错误的位置，完全遮蔽了真正的问题——哪个 TODO 未实现。学生在部分实现阶段调试会极其困难。

Fix: 将所有实质性 TODO 方法改为 `raise NotImplementedError("请实现 __init__")` 等形式。`__repr__` 可改写为 `return f"Value(data={getattr(self,'data','?')}, grad={getattr(self,'grad','?')})"` 以在桩状态下仍可安全调用。

Cell 0 和 Cell 14 均声称 `Value` 是 `src/aurora/ml/` 的起点，但该目录在仓库中不存在（`ls src/aurora/` 只有 `audio`, `llm`, `music`, `speech` 等子包）。学生无法找到对应的真实代码，Aurora 连接的教学价值完全丧失。

Fix: 若 `aurora/ml/` 尚未创建，改写为"本课的 `Value` 是 Month 2 将建立的 `aurora/ml/` 引擎的起点"，或指向已存在的占位模块路径。

### 🟡 Minor

**[Pedagogical Quality] Cell 4 梯度累积演示未声明 `a` 的值**

`b_grad_from_path1 = 3.0` 对应 `dL/db from a*b = a`，但变量 `a` 的取值（3.0）从未明示，与下方 `b=3` 的设定形成隐含依赖，读者需自行推算。

Fix: 加注释 `# 假设 a=3.0, b=3.0` 或用变量 `a_val, b_val = 3.0, 3.0` 计算。

---

**[Exercise Design] 缺少边界条件测试**

Cell 11 和 Cell 13 测试均为正常浮点值。缺少：`Value(0.0)` 的乘法（零乘任何数梯度为 0）；`backward()` 在只有单叶节点时的行为；重复引用同一节点的图（如 `a + a`）是否正确累积梯度（是课程重点，却无专门断言验证）。

Fix: 在 Cell 11 末尾增加 `a4 = Value(3.0); L4 = a4 + a4; L4.backward(); assert a4.grad == 2.0` 以验证同节点多路径累积。

---

**[Exercise Design] 缺少 `__rtruediv__`，`scalar / Value` 会静默失败**

`__truediv__` 调用 `other**-1`——若 `other` 是普通数字则得到 Python float，再走 `__mul__` 可以工作；但反向 `2 / Value(3.0)` 会走 Python 的默认行为（`int.__truediv__`），返回普通 float，而非 `Value`。下游若做 `(2 / val).backward()` 会报 AttributeError。

Fix: 添加 `def __rtruediv__(self, other): return Value(other) * self**-1`。

---

**[Pedagogical Quality] `tanh` 出现在动机但无前向引用说明推迟到 L55**

Cell 1 以 `L = tanh(a*b + b**2)` 开篇，但本课不实现 `tanh`。L55 实际补全了该算子。若不明确说明，学生完成本课后会以为 `tanh` 示例无法跑通。

Fix: 在 Cell 1 末尾加一句"本课先实现 `add/mul/pow`；`tanh/relu/exp` 将在 L55 补全"。

---

**[Exercise Design] `backward()` 拓扑排序无算法提示**

推理路线（Cell 9）仅说"做拓扑排序，逆序调用"，但没有提示具体算法。Cell 6 给出了递归 DFS 版本，但两者距离较远，学生可能忽略这一连接。

Fix: 在 Cell 9 的 `backward()` 提示中加一行"提示：参考 Cell 6 的 `topo()` 函数，用 DFS + visited 集合收集后序节点，再反转"。

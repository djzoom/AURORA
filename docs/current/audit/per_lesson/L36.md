## 问题列表

### 🔴 Critical

**[Accuracy / Exercise Design]** — spec 示例参考值基于 `np.hanning`（symmetric），与 Aurora 实际使用的 periodic Hann 窗不符。

cell `fa1cc508` 写道：
> "N=10, Hann → first≈0, last≈0, peak≈1.0, peak_idx=4（中心）"

但 `aurora.hann(10)` 是 periodic 窗（`periodic=True`，即 DFT-even），实际值为：
- `last = 0.0955`（不是 ≈ 0）
- `peak_idx = 5`（不是 4）
- `peak = 1.0` ✓，`first = 0.0` ✓

`np.hanning(10)` 才给出 `last≈0, peak_idx=4`。学生若按 spec 例子对照自己的心智模型，得到的预期与 Aurora 函数实际输出相矛盾，严重干扰理解。

**Fix**：将示例改为以 Aurora 的 `hann(10)` 实测值为准：`first=0.0, last≈0.095, peak=1.0, peak_idx=5`；并在 spec 中加一句说明 Aurora 默认 `periodic=True`。

### 🟠 Major

**[Accuracy]** — 断言注释"Hann 窗两端应为 0"错误（且仅检查首端）。

cell `1f8ca4fd`：
```python
assert abs(hann(N)[0]) < 1e-9, 'Hann 窗两端应为 0'
```
- 注释说"两端"，实际只断言 `w[0]`，未检查 `w[-1]`。
- 对于 Aurora periodic `hann(64)`，`w[-1] ≈ 0.0024`（不为零）；`hann(10)[-1] ≈ 0.095`，对小 N 尤为明显。
- "两端应为 0"是 symmetric Hann 的性质，不是 periodic Hann 的性质。

**Fix**：改注释为"Hann 窗首端（DFT-even periodic 窗，仅 w[0]=0）"，或添加说明：`w[-1]` 对 periodic 窗不为 0，并酌情增加对 `w[-1]` 的断言（可带宽松阈值）以区分两种惯例。

---

**[Accuracy / Pedagogical Quality]** — Periodic 与 symmetric 窗的差异贯穿全课却从未被解释。

notebook 中存在两套 Hann 实现：
- cells 9、18、26：使用 `np.hanning(N)` / `np.hamming(N)`（symmetric，两端均为 0）
- cell `1f8ca4fd` / aviz：使用 Aurora `hann(N)` / `hamming(N)`（periodic/DFT-even，首端为 0，尾端不为 0）

两种窗在端点行为、峰值位置、峰值幅度上均有差异，混用而不说明会使学生对"Hann 窗应该长什么样"产生混乱。

**Fix**：在 Section 1 或 Section 2 的引导性 markdown 中增加一段说明：NumPy 的 `hanning` 是 symmetric（两端均为 0），Aurora 默认使用 `periodic=True`（DFT-even，适合 STFT 无缝拼帧），两者数值不同，并给出对比数字。

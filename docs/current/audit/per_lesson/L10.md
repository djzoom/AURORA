# L10 — 点积（dot product）与投影（projection）

**总体评级**: B- | **问题数**: 1 critical / 4 major / 4 minor

## 概述

L10 的核心练习（实现 `cosine_similarity`）思路清晰，数值例题可验证，DFT 与余弦相似度的连接是真正的亮点。但标题中承诺的"投影（projection）"内容完全缺席，两个矩阵-向量乘法单元格（cell-7、cell-18）明显是从 L12 混入的离题碎片，`return None` 存根会以 `TypeError` 而非 `NotImplementedError` 崩溃，令初学者无从判断错误来源。整体结构比内容更散乱。

## 亮点

- **DFT 连接正确且具体**：开篇和收尾都明确指出 `X[k] = 信号 · 复指数序列`，是点积应用的典型示例，与课程主线高度一致。
- **推理路线脚手架完整**：第 3 节给出四步推理路线（公式→分子→分母→边界），对初学者友好。
- **数值示例全部可验证**：`[1,2,3]·[4,5,6]=32`，`|a|=√14≈3.742`，`|b|=√77≈8.775`，余弦≈0.974，逐项正确，学生可手算交叉核对。
- **精确同向向量的断言阈值恰当**：`abs(cosine_similarity(song1, 2*song1) - 1.0) < 1e-12` 对完全平行向量用严格浮点阈值，设计正确。
- **课程衔接正确**：收尾处"下一课：L11 讲范数"引用正确，且第 2 节已提前说明 `np.linalg.norm` 留待 L11 推导，逻辑自洽。

## 问题列表

### 🔴 Critical

**[内容完整性]** — 标题写明"与投影（projection）"，但全课没有任何投影内容：标量投影 `(a·b)/|b|`、向量投影 `(a·b/|b|²)b`、几何"阴影"直觉均缺席。学生按标题预期学习投影会完全扑空，而投影是内积空间和最小二乘法的核心前置概念。
**Fix**: 要么在末尾增加一节正交投影（含公式推导、代码和图示），要么将标题改为"点积与余弦相似度"并删去"投影"字样；前者与标题匹配，后者至少诚实。

### 🟠 Major

**[可运行性 / 存根设计]** — `cosine_similarity` 存根返回 `return None` 而非 `raise NotImplementedError()`。测试单元格第一行是 `round(cosine_similarity(song1, song2), 3)`，对 `None` 调用 `round()` 会触发 `TypeError: type NoneType doesn't define __round__ method`，而不是预期的 `AssertionError` 或 `NotImplementedError`。实测确认此行为。初学者看到 TypeError 无法判断是框架出错还是自己实现错误。
**Fix**: 将存根改为 `raise NotImplementedError("请实现 cosine_similarity")`。

**[内容错位]** — cell-7（`A @ v`，矩阵 `[[2,0],[0,0.5]]` 对向量 `[3,4]` 的变换）和 cell-18（矩阵 `[[2,1],[0,1]]` 对四个探测向量的 probe 循环）均属于矩阵变换内容（L12），与点积/余弦相似度无关。cell-7 出现在标题"动手观察：两个向量，三种角度关系"之后，学生期待看到三种夹角的点积演示，实际看到的却是矩阵缩放，叙事完全断裂。
**Fix**: 删除 cell-7 和 cell-18，替换为演示同向/正交/反向三种夹角对应点积正负变化的代码（可扩展现有 `pairs` 列表）。

**[练习设计 / 边界用例]** — 零向量的余弦相似度产生静默 `nan`（`0/0` 触发 RuntimeWarning），但测试完全未覆盖此边界。实测：`cos(np.zeros(4), song1)` 返回 `nan` 无异常。在音频/ML 实践中零向量（全静音帧、空特征）是真实边界。
**Fix**: 在规格说明中明确零向量的预期行为（建议 `raise ValueError("向量长度不能为零")`），并添加对应测试断言。

**[可视化标题与内容不符]** — 第 5 节标题写"内积(→标量) 与 外积(→秩1矩阵)"，但 `vec_times_vec([1,2,3],[4,5,6])` 只渲染外积热力图（源码确认），内积（标量 32）完全没有可视化。标题的双图承诺未兑现，且"内积"这个词在本课还未被正式定义过。
**Fix**: 将标题改为"外积（→秩1矩阵）"，并在图之前加一行 `print(f"内积 a·b = {np.dot(a,b)}")` 显示标量结果，形成真正的对比。

### 🟡 Minor

**[学习目标缺失]** — 全课没有编号的"学习目标"段落，与课程其他模块的风格不一致，学生无法在开始前明确本课的可交付输出。
**Fix**: 在"本课剧情"后插入标准 `## 学习目标` 段落，例：(1) 手算点积公式；(2) 实现并通过 cosine_similarity 全部测试；(3) 用点积解释 DFT 频点的"频率挑选"机制。

**[演示缺少反向案例]** — cell-9 的 `pairs` 只覆盖正值（锐角）和正交，没有反向向量（cosθ = -1）的示例，而第 1 节文字已描述三种情形（正/零/负）。
**Fix**: 在 `pairs` 末尾增加 `(np.array([1.,0.]), np.array([-1.,0.]))` 一对，打印结果 `-1.0`，完成三种情形的闭环演示。

**[断言阈值偏宽]** — `assert cosine_similarity(song1, song3) < 0.1`：song1 与 song3 点积恰好为 0（完全正交），正确实现应精确返回 0.0，阈值 0.1 过于宽松，可能掩盖部分错误实现。
**Fix**: 改为 `assert abs(cosine_similarity(song1, song3)) < 1e-10`，与同向向量断言风格一致。

**[参数实验无可运行代码]** — cell-19"参数实验"描述了正弦波正交性实验（不同频率点积≈0），但全部是文字叙述，没有可执行代码单元格。学生无法亲自观察这一关键结论。
**Fix**: 将文字转换为约 6 行代码单元格，计算并打印 `np.dot(sin(1Hz·t), sin(2Hz·t))` 和 `np.dot(sin(1Hz·t), sin(1Hz·t))` 的值，让学生自行验证正交性。

# L76 — 音乐理论速成 — 音高、音程、色度（chroma）轮与十二平均律

**总体评级**: C+ | **问题数**: 1 critical / 3 major / 5 minor

## 概述

本课覆盖了音高类别、MIDI ↔ 频率换算和 chroma 概念，数学公式全部正确，两道练习均有正确存根（`raise NotImplementedError`）。但标题承诺讲授"音程"却完全缺失该内容；两道练习的验证代码几乎全部是 print-only（无 assert），测试框架几乎无约束力；chroma_from_freq 的取整规范模糊，且存在一个悄无声息返回错误答案的边界陷阱。

## 亮点

- **公式正确**：MIDI 公式 `f(n)=440×2^((n-69)/12)` 及逆公式均无误；2^(1/12) ≈ 1.0595、C4≈261.63 Hz、A3=220 Hz 均经数值验证无误。
- **存根设计合理**：两道练习都用 `raise NotImplementedError("TODO")` 而非 `return None`，阻止了学生错过实现。
- **色度轮可视化清晰**：极坐标 bar + HSV 颜色映射直观展示 12 音高类别，适合教学。
- **本课收束表格简洁**：用三行表格对 12-EDO / MIDI / Chroma 做了准确总结，L77 衔接方向正确。

## 问题列表

### 🔴 Critical

**[课程衔接/标题一致性]** — 标题副标题为"音高、**音程**、色度轮与十二平均律"，但"音程"（interval）在正文、学习目标和练习中完全缺失。学习目标 4 条均未包含音程，教学内容也没有涉及全音/半音距离、完全五度、大三度等概念。Fix：要么在 cell-1 之后补一节介绍音程（半音距离定义 + 常见音程的半音数）并加一道计算两音 MIDI 差的小练习；要么将标题中的"音程"删去，改为"音高、色度与十二平均律"。

### 🟠 Major

**[练习设计/可运行性 — Task 1]** — cell-4 对 `midi_to_freq` 的验证只有 print+三元运算符（`✅/❌`），没有 assert；唯一的 assert 只测了 `freq_to_midi(440.0)`。学生若实现错误（如差了八度），循环只打印 ❌，不抛异常，notebook 照常运行完毕。Fix：将 3 条 print 循环改为 assert：
```python
assert abs(midi_to_freq(midi) - expected) < 0.5, f"{name} failed"
```

**[练习设计/可运行性 — Task 2]** — cell-6 对 `chroma_from_freq` 的验证完全没有 assert，4 条全是 print-only。学生实现错误后 notebook 静默通过。Fix：在打印循环后添加：
```python
assert chroma_from_freq(freq) == expected_pc, f"{name} chroma wrong"
```

**[数学/代码准确性 — 取整陷阱]** — `chroma_from_freq` 的注释只写"用 freq_to_midi 换算后取 mod 12"，未说明如何取整。实际上 B4=493.88 Hz → `freq_to_midi` 返回约 70.9999，若学生用 `int()` 截断得 70，再 mod 12 = 10（A#），与预期 11（B）不符。验证用例恰好包含 B4 (expected_pc=11)，本可暴露此 bug，但因无 assert 而静默失败。Fix：注释中明确说明应使用 `round()`（或 `int(round(...))`），例：`n = round(freq_to_midi(freq)); return n % 12`。

### 🟡 Minor

**[教学质量 — 学习目标措辞]** — 目标 3 写"实现色度轮可视化"，但 cell-7 是完整给出的演示代码，学生不需要实现任何东西，只需运行。Fix：将目标 3 改为"读懂并运行色度轮可视化代码"，或将 cell-7 拆出其中一部分作为练习（如让学生填写 angles 的计算）。

**[可运行性 — 文件路径]** — cell-7 用 `Path.cwd()` 保存 `chroma_wheel.png`。在 Jupyter 中 `cwd` 取决于服务器启动目录，与 notebook 所在目录无关，可能将图片写入意想不到的位置。Fix：改用 `Path(__file__).parent` 或相对于 notebook 的固定路径：`Path("chroma_wheel.png")`（在 Jupyter 下等效于 kernel cwd）。

**[教学质量 — Aurora连接缺失]** — 本课是 music 模块（L76–L82）的入口，但没有任何对 `src/aurora/music/` 的引用。`aurora.music.features` 和 `aurora.music.similarity` 在 L77 中会直接使用，在此提前提及会降低突兀感。Fix：在"本课收束"或 cell-8 末尾加一段：`from aurora.music import features  # 下课 L77 将调用此模块`。

**[练习设计 — 边界测试缺失]** — 两道练习均未测试边界情况。`midi_to_freq(0)`（最低 MIDI）、`freq_to_midi` 传入极低频率，以及 `chroma_from_freq(0)` 均未验证。Fix：Task 1 末尾加 `assert abs(midi_to_freq(0) - 8.18) < 0.1`；Task 2 说明函数假定输入为可听频率（>0 Hz）。

**[教学质量 — 内容深度]** — 本课作为 L76–L82 七课 music 模块的基础课，篇幅偏短（仅 2 道练习，无对音阶结构、转调、音程的实质讲解）。若标题修正后不再承诺音程，则可在建议里加入大调 chroma 集合的简单练习（如"给定 chroma 集合判断是否是大调"），加深学生对"chroma 分布 → 调性"的直觉。

### 🟠 Major

**[代码可运行性 / 功能完整性]** — `import datetime` 在单元格 `671dd07e` 中导入，但全课从未使用。结合 `else` 分支中出现的"建议：把这封信存到 `docs/letter_to_self_DATE.md`"，可以判断最初的设计意图是用 `datetime.date.today()` 自动生成带日期的文件名，并将信件写入磁盘——这个功能被放弃了但 import 未清除，而 `else` 分支中的提示也只是打印一行字，并不实际保存文件。学生如果照做，需要手动复制粘贴，体验断裂。

Fix: 要么删除 `import datetime` 并把 `else` 分支中的"建议"改为纯文字说明；要么补全文件保存代码：
```python
import datetime
save_path = f"docs/letter_to_self_{datetime.date.today()}.md"
with open(save_path, "w", encoding="utf-8") as f:
    f.write(letter_to_future_self)
print(f"已保存到 {save_path}")
```

### 🟡 Minor

**[练习设计]** — `aurora_v2` 字典对 `chosen_direction` 的值没有验证：学生填写 `"X"` 或 `"方向B"` 同样被计为"已填"。一个简单的 assert 可以防止格式错误带来的混淆。

Fix: 在 `filled` 计算后加一行：
```python
assert aurora_v2["chosen_direction"] in (None, "A", "B", "C", "D"), \
    "chosen_direction 只能是 'A'/'B'/'C'/'D'"
```

---

**[代码可运行性]** — `letter_to_future_self` 的哨兵检测 `"在这里写" in letter_to_future_self` 是脆弱的：学生若在模板内继续写作而未删除该短语（例如写"在这里写下了很多..."),  `else` 分支（含字数统计）永远不会执行，静默给出错误反馈。

Fix: 改用更具体的哨兵，例如检测一整行 `"（在这里写）"` 是否仍存在：
```python
if "（在这里写）" in letter_to_future_self:
```

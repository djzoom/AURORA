## 概述

本课以 WER 计算为主线，搭配一段 HMM-GMM vs. 端到端架构的简要对比。数学公式和测试用例均正确，WER DP 推导引导得当，但存在三个结构性问题：stub 用 `pass` 而非 `raise NotImplementedError` 导致整本 notebook 无法顶到底运行；教学排序倒置（L67 才讲 Levenshtein DP 基础，L66 已要求学生独立实现）；以及标题承诺的 AM→LM→Decoder 内容在正文中严重欠缺。

## 亮点

- WER 的 (S+D+I)/N 公式清晰，"顺序互换=2次替换"的反直觉例子配了手工 Levenshtein 追踪，很有说明力。
- 四个 assert 测试覆盖了完全正确、全删除、换位、插入四种典型场景，数值均经验证为正确（参见 solutions/L66_asr_overview_solutions.md）。
- Whisper 参数（sr=16000, n_fft=400, hop=160, n_mels=80）与 L43-L47 保持一致，cross-reference 准确。
- `pairs` 实验区引导学生用真实 ASR 输出替换示例，是良好的开放式探索设计。

## 问题列表

### 🔴 Critical

**[Runability]** — Cell 10 (`compute_wer` stub) 以 `pass` 结尾，函数返回 `None`。Cell 11 随即执行 `abs(compute_wer(...) - 0.0)`，触发 `TypeError: unsupported operand type(s) for -: 'NoneType' and 'float'`，而非设计中的 `AssertionError`。学生看到的是误导性的 TypeError，且 notebook 无法从头到尾顺利执行。
**Fix**: 将 `pass` 改为 `raise NotImplementedError("请实现 compute_wer")`。

### 🟠 Major

**[Course Continuity / Pedagogical Ordering]** — "本课收束"明确写道："下一课 L67 将从零实现 Levenshtein 编辑距离动态规划算法，建立 WER 的数学基础。"然而 L66 的主练习已要求学生自行实现该 DP。学生在 L66 做练习时，数学基础尚未讲授；L67（edit_distance.ipynb）才是这个基础。顺序倒置使练习缺乏支撑。
**Fix**: 将 `compute_wer` 练习移到 L67 之后（例如放到 L68 或 L73 的 wer_eval 模块）；或在 L66 中提前给出完整 DP 推导，并将 L67 调整为更深的变体（带权重编辑距离等）。

**[Pedagogical Quality / Title-Content Mismatch]** — 标题承诺 "声学模型 → 语言模型 → 解码器，端到端 vs 经典流水线"，但实际上 AM/LM/Decoder 架构对比仅占 Section 1（1 个 markdown + 1 个演示代码块），而 WER 计算占据 Sections 3-5 及全部主练习。标题与内容重心严重不匹配，令学生预期错位。
**Fix**: 将标题改为 "ASR 评估指标 WER 与系统架构概览" 或类似准确描述；或者显著扩充 AM→LM→Decoder 的代码演示内容，使之与标题相符。

### 🟡 Minor

**[Accuracy]** — Section 2 文字说明 "T_frames ≈ 总样本数 / hop"，对 3 秒音频给出 ≈300 的印象，但 Cell 6 的正确公式 `1 + (48000 - 400) // 160` 打印 **298**，与文字近似相差 2 帧。虽然公式本身正确，但散文叙述会让学生记错。
**Fix**: 将文字改为 "T_frames = 1 + (N - n_fft) // hop，对 3 s 音频约为 298 帧"。

**[Exercise Design]** — `compute_wer` 的 spec 没有说明当 `reference` 为空时的预期行为（div-by-zero 风险）。注释 `# ✏️ TODO: 边界条件` 过于模糊，学生无法确定应返回 0.0、`float('inf')` 还是抛出异常（solutions 文件选择了 `float('inf')` 但正文未提及）。
**Fix**: 在 docstring 或 markdown 提示中明确："若 `reference` 为空且 `hypothesis` 亦为空，返回 0.0；若 `reference` 为空但 `hypothesis` 非空，返回 `float('inf')`。"

**[Exercise Design]** — DP 推导在 markdown 中使用二维记号 `dp[i][j]`，而 stub 注释直接引导学生实现一维滚动数组（`dp 数组（长度 R+1）`、`return dp[R] / R`）。两套记号并存，初学者容易将二维推导与一维实现对应错位。
**Fix**: 在 markdown 推导之后增加一段"滚动数组优化"说明，明确从 2D 推导到 1D 滚动的转换步骤，或统一使用 2D 记号并让学生先实现 2D 版本。

**[Pedagogical Quality]** — WER 性能基线仅给出英语 LibriSpeech 数据（Whisper-large-v3 ≈ 2–5%）。本课为中文教学且面向中文学生，提供普通话 ASR 基线（AISHELL-1: CER ≈ 4–6%，注意中文用 CER 而非 WER）更为贴切，同时可顺带介绍 CER 与 WER 的区别。
**Fix**: 补充 AISHELL-1/Common Voice 中文数据的 CER 基线，并一句说明中文 ASR 通常用字错率（CER）而非词错率。

### 💡 建议

- 增加一个明确的 **学习目标** 列表（3-4 条具体可测目标），以便学生明确本课期望掌握的能力边界。
- 第 5 节的 `pairs` 数据集注释里 "whether" 拼写错误（应为 "weather"），虽是故意构造的替换错误示例，可在注释中标注 `# 拼写错误（替换）：whether→weather` 以消除歧义。
- 考虑对 `pairs` 实验增加汇总统计（分别统计 S/D/I 总量），而非仅打印每句 WER，这样与 Section 5 的 "错误类型分析" 主题更一致。

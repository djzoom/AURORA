## 概述

L29 covers the three core distributions (uniform, Gaussian, Bernoulli) with sensible pedagogical sequencing: hook via sampling, implement the PDF from scratch, then validate via the empirical rule and numerical CDF. The `gaussian_pdf` exercise is well-scaffolded with a clear spec and a step-by-step reasoning block. However, a critical API compatibility bug (`np.trapezoid` requires NumPy 2.0+ while the project allows NumPy ≥1.24) means three cells crash on any NumPy 1.x installation. The test suite for the sole exercise is thin, and several cells contain dead or misleadingly-commented code.

## 亮点

- The Gaussian PDF reasoning block (cell `5feb5fbc`) is genuinely excellent: it breaks the formula into three named steps, gives concrete example input/output values (x=0 → 0.399, x=1 → 0.242), and explains the normalization denominator in plain language.
- The Aurora connection sections are accurate and specific: He init, VAE latent sampling, and diffusion noise scheduling are all directly derived from Gaussian theory.
- The `normal_cdf` implementation (cell `cell-18`) is self-contained — it does not depend on the student's (possibly unimplemented) `gaussian_pdf`, so the empirical-rule verification runs regardless of exercise state.
- Bernoulli PMF is clean: the pre-implemented cell (cell-16) validates PMF sums to 1 and cross-checks the sample mean and variance against theory in one pass.

## 问题列表

### 🔴 Critical

**[Runability]** — `np.trapezoid` is a NumPy 2.0+ API; `pyproject.toml` only requires `numpy>=1.24`. Cells `cell-12`, `cell-14`, and `cell-18` all call `np.trapezoid(...)` and will raise `AttributeError: module 'numpy' has no attribute 'trapezoid'` on any NumPy 1.x environment, breaking the CDF section entirely.

Fix: Replace every `np.trapezoid(y, x)` with `np.trapz(y, x)` (available since NumPy 1.0), or add `trapezoid = getattr(np, "trapezoid", np.trapz)` at the top of each affected cell. Alternatively, bump the minimum to `numpy>=2.0` in `pyproject.toml` if the project can afford it.

### 🟠 Major

**[Exercise Design]** — The `gaussian_pdf` test cell (cell `a6d604b7`) only asserts (1) the peak value at x=0 with sigma=1, and (2) left-right symmetry around mu=0. A student can pass both tests with a formula that is correct only for mu=0, sigma=1, for example by hard-coding `sigma=1` and forgetting the normalization denominator dependence on sigma. There is no assertion that `gaussian_pdf(np.array([0.0]), mu=1.0, sigma=2.0)` equals the correct value, and no assertion that `np.trapezoid(pdf, xs) ≈ 1` for various sigmas appears in the exercise test cell itself (it is only shown informally in cell-12).

Fix: Add two extra asserts to the test cell: one for a non-standard (mu, sigma) pair, and one that checks the integral equals 1 within tolerance.

```python
# Check non-trivial parameters
y2 = gaussian_pdf(np.array([2.0]), mu=2.0, sigma=0.5)
assert abs(y2[0] - 1/(0.5*np.sqrt(2*np.pi))) < 1e-9, 'mu=2, sigma=0.5 峰值错误'
# Check normalization
xs_dense = np.linspace(-10, 10, 4000)
integral = np.trapz(gaussian_pdf(xs_dense, 0.0, 2.0), xs_dense)
assert abs(integral - 1.0) < 0.001, 'sigma=2 时积分应≈1'
```

### 🟡 Minor

**[Runability]** — Cell `cell-14` computes `cdf = np.cumsum(pdf) * (xs[1] - xs[0])` but then never uses or prints `cdf` — the variable is immediately orphaned. The cell prints the `np.trapezoid` area for the 1-sigma interval instead. This dead assignment adds visual noise and may confuse students about whether they are expected to examine `cdf`.

Fix: Remove the `cdf = np.cumsum(...)` line, or actually display it (e.g., `ax.plot(xs, cdf)`).

**[Pedagogical Quality]** — There are no numbered learning objectives (学习目标). The intro cell states a single "目标" sentence but does not enumerate concrete, testable goals. This is inconsistent with good lesson hygiene and makes it hard for students to self-check mastery.

Fix: Add a numbered list: (1) implement gaussian_pdf from the formula; (2) verify the 68-95-99.7 empirical rule numerically; (3) compute a Bernoulli PMF and validate with sampling; (4) approximate a continuous CDF via trapezoid integration.

**[Runability]** — In cell `cell-22`, `rng = np.random.default_rng(42)` is placed inside the `for n in [30, 300, 3000]` loop, so each sample size gets a fresh generator starting from seed 42. While intentional for reproducibility, it means the n=300 sample is not a superset of the n=30 sample — an effect students may find surprising. A brief comment clarifying this choice would prevent confusion.

**[Exercise Design]** — No edge-case tests exist for `gaussian_pdf`: passing `sigma=0` causes division-by-zero and returns `nan` or `inf` silently. The spec says "sigma > 0" but there is no assertion to enforce it. For a research-engineer audience, defensive checks matter.

Fix: Add `assert sigma > 0, "sigma must be positive"` inside the implementation hint, or add an explicit test `try: gaussian_pdf(x, sigma=0); assert False except ZeroDivisionError: ...` to document expected behavior.

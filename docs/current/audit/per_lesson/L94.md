## 概述

L94 是课程的毕业 Demo 节，定位清晰：把六个月成果组织成可展示的证据链，并固化面试话术。框架设计（WWHR、四层证据链、简历 bullet 生成器）有实际指导价值，Q&A 答案也足够具体。但模块路径与真实代码库存在系统性偏差——"Aurora 连接"标头和能力矩阵引用的 7 个路径在真实 src/aurora/ 下均不存在或名称不符——直接损害了"证据可验证"这一核心卖点。WWHR 缩写漏掉 Limits（L），面试官一眼能发现。

## 亮点

- **四层证据链表格**（GitHub history / 博客 / 测试 / 录屏）以"不可伪造性递增"为轴，逻辑严密，是帮助学生建立面试心智模型的优质内容。
- **Q&A 10 道题**答案方向具体且诚实——主动承认 Limits（无流式解码、噪声鲁棒性差），符合"成熟研究工程师"定位。
- **能力矩阵用 os.path.exists() 动态检查**，未实现的模块输出"🔲 待实现"而非假装存在，设计思路合理。
- **HuggingFace 分工表格**（Aurora 自己实现 vs 通过 HF 使用）把"no API wrappers"原则与工程实用性的边界讲清楚，是课程中少见的"何时不手写"的明确说明。
- 简历 bullet 格式规范（动词精确 + 指标可查 + 对比锚点）是可直接复用的写作模板。

## 问题列表

### 🔴 Critical

**[准确性 / 模块路径]** — "Aurora 连接"标头和能力矩阵（cell 869c557c）引用的模块路径与真实代码库系统性不符：

| 矩阵路径 | 实际状态 |
|---|---|
| `src/aurora/train/kws.py` | `train/` 包不存在；实为 `mlops/` |
| `src/aurora/asr/ctc.py` | `asr/` 包不存在；实为 `speech/` |
| `src/aurora/asr/eval.py` | 同上 |
| `src/aurora/music/embed.py` | `music/` 下只有 `features.py`, `similarity.py` |
| `src/aurora/music/recommend.py` | 同上，无此文件 |
| `src/aurora/rag/retriever.py` | `rag/` 下只有 `__init__.py` + `README.md` |
| `src/aurora/pipeline/runner.py` | `pipeline/` 包不存在 |

标头同时列出 `src/aurora/train/`、`src/aurora/asr/`、`src/aurora/rec/`、`src/aurora/agent/` 为已实现模块，但这四个包均不存在。这会让首次打开 notebook 的面试官/同学看到全红矩阵，同时与标头的"已实现"描述矛盾。

Fix: 将路径更新为实际存在的包名（`speech/`、`mlops/`、`music/features.py`、`music/similarity.py`、`llm/`）；待实现模块保留占位但从标头移除"已实现"措辞。

---

**[准确性 / 术语]** — WWHR 框架缩写与内容不符：单元格中写的是 `WWHR（What / Why / How / Results / Limits）`，共 5 个组件对应 4 个字母，"Limits"（L）在缩写里没有对应字母。这在面试演示中会被追问。

Fix: 改为 `WWHRL（What / Why / How / Results / Limits）` 或将 Limits 合并入 Results 改写为 `Results & Limits`，使缩写与展开一一对应。

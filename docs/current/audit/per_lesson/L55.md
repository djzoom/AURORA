# L55 — 前向传播（forward pass）拆解 — 算子（operator）节点：`__pow__`、`relu`、`tanh` 的实现与计算图展开

**总体评级**: B- | **问题数**: 1 critical / 2 major / 5 minor

## 概述

本课以 Karpathy 风格的 `Value` 类为骨架，教学思路清晰：先概念、再演示、再练习、最后感知机综合实验，顺序合理。数学公式（tanh 导数、幂次法则、ReLU 次梯度）全部正确，反向传播闭包设计准确。然而，练习脚手架（cell-10）存在一处关键不一致——`__pow__` 桩代码缺少计算图边（`_prev` / `_op`）——学生若只填数据项会得到一个静默错误的反向传播。此外，综合演示（cell-13）复用了同一个 `x1`/`x2` 对象跨两次 `.backward()`，导致梯度累积，产生误导性输出。

## 亮点

- **梯度闭包机制的解释**（cell-3）简洁准确：一句话说清"前向 + 后向同时完成"，对初学者友好。
- **tanh 梯度复用技巧**（cell-5）是真实工程实践（避免重复调用昂贵函数），值得单独点出——本课做到了。
- **数值差分验证**（cell-6）在课程这个阶段出现是很好的习惯，让学生体验解析梯度与数值梯度的一致性。
- **感知机前向实验**（cell-13）概念上设计精良：`z=0` 时 `tanh'(z)=1`，梯度等于输入值本身，直觉上非常清晰。
- 断言覆盖了前向值和反向梯度两个维度，正负区域都有测试（ReLU）。

## 问题列表

### 🔴 Critical

**[Exercise Design / Runability]** — `__pow__` 脚手架缺少 `_prev` 和 `_op`，与其他算子不一致。

cell-10 中的 `relu` 和 `tanh` 桩代码已经预填了 `Value(t, (self,), 'relu')` 和 `Value(t, (self,), 'tanh')`，学生只需填 `t = ...` 即可。但 `__pow__` 桩写的是：

```python
out = Value(...)        # 计算 self.data**n
```

学生填入 `Value(self.data**n)` 后，`_prev` 为空集，`_op` 为空字符串，导致计算图中无边——`backward()` 不会把梯度回传到 `self`。验证断言 `assert abs(a.grad - 6.0) < 1e-9` 会失败，但错误信息是梯度为 `0.0`，学生很难定位到 `_prev` 缺失。

**Fix**：将 `__pow__` 桩改为与 relu/tanh 一致的形式：
```python
out = Value(self.data**n, (self,), f'**{n}')
```
只让学生填幂次计算，`_prev` 和 `_op` 已预填。

---

### 🟠 Major

**[Runability]** — cell-13 的 relu 对比演示复用 `x1`/`x2`，导致梯度累积污染。

cell-13 先对 `output = z.tanh()` 调用 `backward()`，此时 `x1.grad = 0.5`，`x2.grad = -0.3`。随后 relu 演示用同一个 `x1`、`x2` 对象：

```python
zr = w1r*x1 + w2r*x2 + br
out_r = zr.relu()
out_r.backward()
```

执行后 `x1.grad = 1.0`（0.5+0.5），`x2.grad = -0.6`（-0.3+-0.3）。虽然打印的是 `w1r.grad` 而非 `x1.grad`，但若学生在 REPL 中检查 `x1.grad` 会得到令人困惑的累积值。演示应使用新建的 `x1r = Value(1.0)` / `x2r = Value(2.0)`。

**Fix**：
```python
x1r, x2r = Value(1.0), Value(2.0)
zr = w1r*x1r + w2r*x2r + br
```

---

**[Runability]** — `__truediv__` / `__rtruediv__` 在 `__pow__` 实现之前调用会触发 `NotImplementedError`，错误信息无指向。

starter 类（cell-2）中 `__truediv__` 定义为 `self * other**-1`，而 `__pow__` 此时是 `raise NotImplementedError`。学生若在完成练习之前尝试除法运算，会看到裸 `NotImplementedError` 而不知道是要实现 `__pow__`。

**Fix**：在 `__truediv__` 和 `__rtruediv__` 上加注释说明依赖关系：
```python
def __truediv__(self, other):
    # 依赖 __pow__，需先实现
    return self * other**-1
```
或暂时用显式提示替代：
```python
def __truediv__(self, other):
    raise NotImplementedError("请先实现 __pow__，__truediv__ 依赖它")
```

### 🟡 Minor

**[Exercise Design]** — cell-10 的 `...` 桩会产生 `TypeError`（不是 `NotImplementedError`），信息不明确。

`out = Value(...)` 执行时抛出 `TypeError: float() argument must be a string or a real number, not 'ellipsis'`，学生难以理解是占位符问题还是类型错误。

**Fix**：改用字符串注释而非 `...`，或在 docstring 中明确标注 `# 👇 只修改这行`。

---

**[Exercise Design]** — 没有显式的编号学习目标（学习目标）章节。

第一格的"目标"一句话描述了任务但不列举具体的学习目标，与课程其他节格式可能不一致。

**Fix**：添加：
```
## 学习目标
1. 理解计算图节点 = 前向值 + 反向闭包的双重结构
2. 实现 __pow__、relu、tanh、exp 的 _backward 闭包
3. 验证链式法则在单节点层面的正确性
```

---

**[Accuracy]** — 断言使用舍入参考值 `0.7616` 而非 `math.tanh(1.0)`。

`assert abs(Value(1.0).tanh().data - 0.7616) < 1e-4` 在语义上是对的（误差 ~5.8e-6 远小于 1e-4），但不如直接用：
```python
assert abs(Value(1.0).tanh().data - math.tanh(1.0)) < 1e-9
```
这样更精确，且体现了"验证与标准库一致"的测试哲学。

---

**[Exercise Design]** — ReLU 反向断言未覆盖 `x=0` 边界。

当前测试覆盖正数（`Value(2.0)`）和负数（`Value(-1.0)`），但 `x=0` 的次梯度行为（取 0）是本课明确声明的教学点，未在断言中出现。

**Fix**：添加：
```python
a = Value(0.0)
r = a.relu()
r.backward()
assert a.grad == 0.0, "relu(0) 次梯度应为 0.0"
```

---

**[Pedagogical Quality]** — 概念格（cell-5、cell-7）已完整给出实现代码，练习实质变为抄写。

cell-5 已给出 `self.grad += (1 - out.data**2) * out.grad`，cell-7 已给出 `self.grad += (out.data > 0) * out.grad`，这与 cell-10 的 TODO 内容一模一样。这使练习失去了"独立推导"的挑战性，尤其是对进阶学生。

**Fix**（可选）：将概念格中的代码换成伪代码或数学公式，或明确定位为"guided implementation"而不是独立练习。

---

## 优先修复（前3项）

1. **[Critical] 补全 `__pow__` 脚手架的 `_prev=(self,)` 和 `_op`**：这是唯一会让学生产生静默错误（梯度为 0 但无报错）的问题，修复成本极低，影响极大。

2. **[Major] cell-13 relu 对比演示改用新建 `x1r`/`x2r` 对象**：梯度累积是自动微分初学者最常见的混淆点，演示代码本身就产生这种混淆，需要修复。

3. **[Major] `__truediv__` 在 `__pow__` 未实现时的错误提示**：在 starter 类中加注释或临时改成有意义的 `NotImplementedError` 消息，避免学生在实现 `__pow__` 之前误用除法时看到神秘报错。

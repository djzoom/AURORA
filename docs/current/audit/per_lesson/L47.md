## 问题列表

### 🔴 Critical

**[Accuracy / Math]** — Cell 3 states "前 40 个 bin 集中在 0–2700 Hz（语音共振峰最密集的区间），后 40 个 bin 平摊 2700–8000 Hz". This is factually wrong by ~970 Hz. The actual `mel_filterbank` implementation (`src/aurora/audio/mel.py` lines 57–58) uses `np.linspace(mel(0), mel(8000), 82)` — 82 equally-spaced mel points covering 0–2840 mels. The 40th filter center sits at mel_point[40] = 1402.5 mels = **1730 Hz**, not 2700 Hz. The value 2700 Hz corresponds to approximately mel bin 51, not 40. Fix: replace "0–2700 Hz … 2700–8000 Hz" with "0–1730 Hz … 1730–8000 Hz" (or compute exactly: `mel_to_hz(np.linspace(0, hz_to_mel(8000), 82)[40])`).

**[Runability]** — Cell 13 executes `feat = log_mel_spectrogram(x, sr, n_mels=n_mels, ...)` then immediately `feat.T` and `feat.shape`. When the function stub in cell 10 is unimplemented (`pass` returns `None`), this raises `AttributeError: 'NoneType' object has no attribute 'T'` — a confusing crash that gives no pedagogical guidance. Cell 11 guards against None with an explicit assertion, but cell 13 has no such guard. Fix: add `if feat is None: continue` inside the loop, or move cell 13 below a comment instructing students to complete cell 10 first, or add a sentinel skip: `if log_mel_spectrogram is None: pass  # implement first`.

### 🟡 Minor

**[Exercise Design]** — The function stub in cell 10 uses `pass` (returns `None` silently). **[Pedagogy / Accuracy]** — Cell 12 states `n_mels=40` causes "低频细节丢失". Mel bin spacing is non-linear in Hz: halving `n_mels` widens *all* mel bins proportionally on the mel scale. A more accurate description: "整体频率分辨率降低，尤其影响低频 Hz 分辨率，因为 Mel 刻度在低频比高频分辨更细". Fix: revise the description to reflect uniform mel-space degradation.

## 概述

本课实现了 `train_epoch` 和 `eval_accuracy` 两个核心函数，练习设计规范，数学公式（交叉熵、per-class accuracy、混淆矩阵方向）均正确。但课题标题承诺"过拟合诊断"，cell-16 的参数实验中 `val_loader` 与 `train_loader` 使用完全相同的数据，使得任何"验证准确率"都毫无意义，根本无法演示过拟合现象。这一结构性错误与课程的核心教学目标直接冲突。

## 亮点

- `train_epoch` 和 `eval_accuracy` 的推理路线（cell-9、cell-12）写得清晰具体：分步列出调用顺序，给出典型数值范围（随机初始化 ≈0.25，收敛后 0.75–0.90），帮助学生自验结果。
- 两个 TODO 函数均以 `raise NotImplementedError` 为桩，不会因 `return None` 静默通过测试。
- CrossEntropyLoss 的公式展示（cell-3）和演示（cell-4）互相印证，且明确指出"不要在模型末尾手动加 softmax"，这是初学者最常犯的错误。
- 混淆矩阵的行列约定 `C[i][j] = 真实 i、预测 j` 正确，与 sklearn `confusion_matrix` 的默认行为一致。
- cell-15 给出了混淆矩阵中深色非对角格的实际含义解读（两类 mel 特征相似 → 需要更多数据或更深网络），是有价值的定性分析。

## 问题列表

### 🔴 Critical

**[Runability / Pedagogical Quality]** — cell-16 中 `val_loader` 与 `train_loader` 使用完全相同的 NumPy 张量（`_x_big`, `_y_big`）。在这种设置下，"验证准确率"等于训练集准确率，模型永远不会表现出泛化间隔。本课课题明确承诺"过拟合诊断"，这个演示不仅无法教授该概念，还会让学生产生错误直觉（认为 val_acc 和 train_loss 同步下降是正常的）。

**Fix**: 用 `torch.utils.data.random_split` 把 `_x_big`/`_y_big` 按 80/20 切成训练集和验证集；或分别用不同的随机种子生成 `_x_train`、`_x_val`。配合加大 `num_epochs`（例如 20），可以让验证曲线与训练曲线自然分叉，真正演示过拟合。

### 🟠 Major

**[Accuracy]** — cell-9 写道"第 1 epoch 典型输出：`avg_loss ≈ 1.5`（4 类随机初始化）"，而 cell-15 正确地写了"≈log(num_classes)≈1.39"。4 类随机分类器的期望交叉熵是 ln(4) ≈ 1.386，cell-9 的 1.5 偏高且与同一 notebook 的其他表述矛盾。

**Fix**: 将 cell-9 的"≈1.5"改为"≈1.39（ln 4）"，与 cell-15 保持一致。

**[Pedagogical Quality]** — 课题标题和 "本课收束" 均承诺过拟合诊断，但 notebook 正文对此只有 cell-15 的一段描述性 Markdown，没有任何对应练习、可观察演示或量化工具（如 early stopping 判据、泛化间隔公式）。与之对比，L65 的第 2 节系统地实现了 train/val loss 对比曲线和 early stopping 可视化——这些内容实际属于 L64 的承诺范围。

**Fix**: 要么在 L64 增加一个具体的过拟合演示（见 Critical 项的修复），要么将标题和 "本课收束" 中的"过拟合诊断"改为"过拟合介绍"，并明确注明"深度诊断见 L65"，避免标题过度承诺。

**[Exercise Design]** — 两个验证测试都非常薄弱：cell-11 只断言 `isinstance(_loss, float)` 和 `_loss > 0`；cell-14 只断言 `0.0 <= _acc_before <= 1.0`。这些断言无法捕捉常见错误，例如忘记 `zero_grad()`（梯度累积导致损失异常）、忘记 `model.eval()`（eval 时 BN 使用 batch 统计）、或 `correct / total` 写成整数除法。

**Fix**: 在 cell-11 中，用固定随机种子运行两轮并断言第二轮损失小于第一轮（验证权重确实被更新）；在 cell-14 中增加 `assert _acc_before < 0.6`（随机 4 类模型准确率不应异常偏高）以及一个全正确输入的边界检查（`eval_accuracy` 应返回 1.0）。

### 🟡 Minor

**Fix**: 将所有 sklearn 导入合并到 cell-2 的 import 块中。

**[Pedagogical Quality]** — 没有明确编号的学习目标列表。"目标"段落是一个单句话，缺少结构化的目标编号（如"1. 实现…；2. 理解…；3. 可视化…"），与课程其他章节风格不一致。

**Fix**: 在 cell-0 的"目标"下用有序列表列出 3 个具体、可验证的学习目标。

**[Pedagogical Quality]** — 没有说明为何在"no API wrappers"原则下可以使用 `sklearn.metrics.confusion_matrix`。CLAUDE.md 明确规定核心算法须手写；混淆矩阵计算是一个简单的 `np.zeros` + 双重循环，完全可以手写，并且手写过程能帮助学生理解其内部结构。

**Fix**: 添加一句注释说明"混淆矩阵统计不是 audio/ML 核心算法，使用 sklearn 工具函数合理；若想从头实现，可用 `np.add.at(cm, (y_true, y_pred), 1)`"。或者将手写版本作为一个可选练习。

**[Exercise Design]** — `train_epoch` 的 smoke test（cell-11）用的是 `nn.Sequential(nn.Flatten(), nn.Linear(40*32, 4))`，但输入 `_x` 的形状是 `[16, 1, 40, 32]`，`Flatten()` 后维度是 1280（= 1×40×32），而 `40*32 = 1280`，两者恰好相等。这是一个意外的数值巧合：若输入通道数不为 1，测试会以 RuntimeError 失败，但原因不直观。建议在注释中明确写出 `nn.Linear(1 * 40 * 32, 4)` 或 `nn.Linear(1280, 4)` 以消除歧义。

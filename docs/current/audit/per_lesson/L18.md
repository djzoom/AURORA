### 🟡 Minor

1. **[Runability — 存根单元格缺少 import]** — cell 978af2b9 调用 `np.asarray` 但本单元格未 `import numpy as np`；若学生重启内核后单独运行该单元格会得到 `NameError: name 'np' is not defined`。  
   **Fix**: 在存根函数前加 `import numpy as np`。

2. **[Accuracy — det 显示精度不一致]** — Cell e2cc5b9f 用 `round(np.linalg.det(A))` 输出 `det = 2`（Python 内置 `round` 不传 ndigits 返回 `int`），而同一单元格的特征值用 `np.round(..., 3)` 输出浮点数，显示风格不统一；若 det 因浮点误差为 1.9999 也会被静默舍入为 2。  
   **Fix**: 改为 `np.round(np.linalg.det(A), 4)`。

3. **[Pedagogical Quality — 缺少结构化学习目标]** — 全课无独立的「## 学习目标」列表，目标散落在标题 cell 的散文中，学生难以在课末自检。课程其他单元（如 L14、L16）均有编号目标列表。  
   **Fix**: 在第一个 markdown 单元格后加「## 学习目标」块，列 3–4 条可验证目标。

4. **[Code Quality — Jacobi 循环内重复构造对角矩阵]** — `jacobi` 函数在每次迭代调用 `np.diag(np.diag(A))`（O(n²) 分配），循环 30 次即构造 30 次全零矩阵再填对角，与课程「研究工程」风格不符。  
   **Fix**: 在循环前加 `D = np.diag(np.diag(A))`，循环内改用 `A @ x - D @ x`。

5. **[Accuracy — null space"应为全零"表述模糊]** — Cell-8 注释「应为全零」，实际结果因浮点误差约在 1e-15 量级，`np.round(..., 10)` 掩盖了这一事实。对于初学者，"全零"与"数值近似零"的区别值得明确。  
   **Fix**: 注释改为「（浮点误差约 1e-15，视为数值零）」。

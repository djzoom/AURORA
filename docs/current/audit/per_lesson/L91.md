## 概述

L91 是 LLM 模块的收尾可视化课，目标合理：用热力图展示 attention 权重、用柱状图展示 LoRA 参数节省。代码逻辑本身无误，softmax 实现稳定，LoRA 数学正确。但存在一个严重的标题-内容不符问题（标题声称"多头注意力"而实现仅为单头），以及若干影响正确性的事实错误和可运行性风险。

## 亮点

- `scaled_dot_product_attention` 的 softmax 使用了数值稳定写法（`scores - scores.max(axis=-1, keepdims=True)`），是好示范。
- LoRA 的参数量公式 `2·d·r`、节省比例、以及对角线高说明 token 主要关注自身这些叙述都准确。
- 参数实验小节（Section 3）对 `d_k` 大小影响 softmax 分布峰度的解释清晰到位，直觉培养有效。
- 热力图在格子内显示数值，颜色与阈值切换白/黑字，视觉细节用心。

## 问题列表

### 🔴 Critical

**[内容-标题不符]** — 标题声称"多头注意力（Multi-Head Attention，MHA）权重热力图"，但整个 notebook 仅实现单头 `scaled_dot_product_attention`，完全没有多头机制（按 head 分割 d_model、并行运行 h 个头、concat 后线性投影）。L83 明确说"multi-head 的投影矩阵将在 L2 封装进 MultiHeadAttention 类"——也就是说 MHA 在本课程内并未正式实现，L91 却在标题和 Aurora 连接处直接使用"MHA"和"多头注意力"，会让学生误认为他们已看到了 MHA 的热力图。
Fix：将标题改为"Self-Attention 权重热力图"，删去标题中的"多头注意力（MHA）"，或者新增一个演示小节，正式实现 2 头 attention 并并排显示两张热力图对比。

### 🟠 Major

**[可运行性]** — `import torch`（cell-2）是全文唯一的 torch 引用，notebook 其余所有计算均使用 numpy。若学生在未安装 torch 的纯 CPU 环境（或轻量 venv）中运行，将在 cell-2 直接抛出 `ModuleNotFoundError`，导致后续所有单元格无法执行。
Fix：删除 `import torch` 这一行。

**[事实错误]** — cell-5 注释"`(GPT-2 规模)`"对应 `d=4096`，但 GPT-2 最大变体（XL）的 `d_model=1600`；d=4096 对应的是 LLaMA-7B 或 GPT-3（175B）的子模块，不是 GPT-2。该错误会在学生对照 GPT-2 论文时造成混淆。
Fix：将注释改为"（LLaMA-7B / GPT-3 规模）"，或直接删去括号中的具体型号。

**[Aurora 连接不准确]** — 标题格子说"Aurora 连接：`aurora.llm`（自注意力（Self-Attention）机制实现）"，但实际的 `aurora.llm` 模块（`kvcache.py`, `sample.py`, `retrieve.py`）中根本没有 attention 实现；L83 的收束明确说 attention 核心将进入 `aurora/whisper/attention.py`。
Fix：将 Aurora 连接改为"`aurora.llm`（KVCache、采样、TF-IDF 检索） · `aurora/whisper/attention.py`（自注意力核心）"。

**[缺少学习目标]** — 本课程其他课（如 L83、L84）均有明确编号的"学习目标"部分，L91 仅在标题格的"目标"一行一句话描述，没有可量化的 3-5 条学习目标（例如"能写出稳定的 softmax，能计算任意 rank 的 LoRA 参数量"），与模块风格不一致。
Fix：在 cell-0 后添加"## 学习目标"格，列出 3 条具体、可验证的目标。

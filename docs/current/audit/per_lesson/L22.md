## 概述

The core central-difference explanation is mathematically sound and the error analysis (O(h²) truncation, catastrophic cancellation below h≈1e-12) is correct. However, the notebook has a systematic mismatch between its markdown narrative and its code cells throughout Section 1: the text describes sin/cos experiments while every code cell uses polynomials. Additionally, the title promises both a formal limit definition and tangent-line visualization that never appear, and the sole exercise stub returns `None` instead of raising `NotImplementedError`, producing a cryptic `TypeError` rather than a clean failure signal. The Aurora connection references a file that does not exist.

## 亮点

- The explanation of *why* central differences outperform forward differences (Taylor cancellation of O(h) terms leaving O(h²)) is clear, accurate, and appropriately rigorous for lesson 22.
- The catastrophic cancellation explanation ("有效位数从 16 位降到 4 位") is correct and provides genuine insight into floating-point limits.
- The h-sweep experiment (cell-16) is a well-designed empirical investigation that makes the abstract U-shaped error curve concrete and computable.
- The scaffolding cell (cell-11) before the TODO cleanly lists inputs, key step, and output type — a good pattern for beginner notebooks.

### 🟠 Major

**[Pedagogical Quality]** — Title promises a formal limit definition (极限定义) that never appears.
The notebook goes directly to the central-difference approximation formula without ever writing `f'(x) = lim_{h→0} (f(x+h)-f(x))/h`. The limit definition is in the title and subtitle but absent from the body.
**Fix**: Add a markdown cell in Section 1 that writes the limit definition, explains that a computer cannot take h→0 (floating-point limitation), and motivates the finite-h approximation.

**[Pedagogical Quality]** — Title and section header promise tangent-line (切线) content but no plots exist.
"切线斜率" is central to derivative intuition. The notebook has zero figures. A simple matplotlib figure (curve + tangent line at one point) would make the concept concrete and align with the title.
**Fix**: Add a visualization cell after the `f=x**2` demo that draws the parabola and the tangent line at x=3.

**[Exercise Design]** — Assertion threshold (1e-3) is four to five orders of magnitude too loose.
Central differences with h=1e-5 achieve accuracy of ~1e-10. The test `abs(...) < 1e-3` would pass even a forward-difference implementation with h=1e-3, defeating the purpose of the test. The threshold also contradicts the notebook's own claim that "误差已在 1e-10 量级".
**Fix**: Change both assertions to `< 1e-8` to distinguish correct central-difference implementations from naïve alternatives.

**[Pedagogical Quality]** — No numbered 学习目标 (learning objectives) section.
The notebook opens with a one-sentence "目标" blurb that is not enumerated. Other lessons in this course use a proper numbered list.
**Fix**: Add a "## 学习目标" section with 3–4 numbered items: (1) state the limit definition, (2) derive O(h²) error for central differences, (3) implement `numeric_derivative`, (4) identify optimal h range empirically.

## 问题列表

### 🔴 Critical

**[数学准确性]** — Chroma 音高类别公式缺少 MIDI 偏置项，导致音高类别映射完全错误。  
Markdown（cell 3）写的是：
```
p = round(12 * log2(f / 440)) mod 12
```
对 A4（440 Hz）代入：`round(0) mod 12 = 0`，即 C——但 A4 应该是音高类 9（A）。  
实际代码（`src/aurora/music/features.py`，`hz_to_midi`）做的是：  
`midi = 12 * log2(f/440) + 69`，然后 `pitch_class = round(midi) % 12`，结果正确。  
正确公式应为：`p = round(12 * log2(f / 440) + 69) mod 12`，或等价地 `p = round(12 * log2(f / C4_freq)) mod 12`（C4 ≈ 261.63 Hz）。  
**Fix**：在 cell 3 的公式中补上 `+ 69`，并加注"69 = A4 的 MIDI 编号，mod 12 将 C 规范为 0"。
